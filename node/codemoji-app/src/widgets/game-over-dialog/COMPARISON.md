# 📊 Сравнение игровых модальных окон

## Обзор всех игровых диалогов

В приложении есть три основных модальных окна для различных игровых ситуаций:

| Диалог | Когда показывается | Основная цель |
|--------|-------------------|---------------|
| **VictoryDialog** 🎉 | Игрок победил в игре | Поздравить с победой |
| **FirstPlaceDialog** 🥇 | Игрок вышел на 1 место во время игры | Подписка на уведомления |
| **GameOverDialog** 😔 | Игрок не успел победить | Утешить и мотивировать |
| **ErrorDialog** ❌ | Техническая ошибка | Информация об ошибке |

## Детальное сравнение

### VictoryDialog 🎉

**Назначение:** Финальная победа в игре

**Когда показывается:**
- Игрок первым разгадал код на 100%
- Игрок набрал максимальный балл
- Игрок выиграл игру

**Визуал:**
- Медаль победителя (`/images/game/medal.png`)
- Конфетти в фоне
- Зеленая кнопка (#34C759)
- Эмодзи: 🎉 🎊

**Основная кнопка:** "Отлично!" (зеленая)

**Управление:**
```typescript
import { useVictoryDialog } from '@/widgets/victory-dialog'
const { show } = useVictoryDialog()

show({
  playerName: 'Player',
  prizeAmount: 2500,
  earnedPoints: 150,
  leaderboardPosition: 1
})
```

---

### FirstPlaceDialog 🥇

**Назначение:** Выход на первое место во время игры

**Когда показывается:**
- Игрок впервые выходит на 1 место в лидерборде
- Во время активной игры (не финал)
- Промежуточное достижение

**Визуал:**
- Медаль первого места (`/images/game/medal.png`)
- Конфетти в фоне
- Синяя кнопка (#0050FF)
- Эмодзи: 🔔

**Основная кнопка:** "Подписаться на уведомления" (синяя)

**Особенность:** Предлагает подписку на уведомления о других игроках

---

### GameOverDialog 😔

**Назначение:** Игрок не успел победить

**Когда показывается:**
- Другой игрок первым разгадал код
- Истекло время на разгадывание
- Закончились попытки
- Игрок был элиминирован

**Визуал:**
- Изображение ошибки (`/images/error-img.png`)
- Красная кнопка (#FF2F00) или синяя (#0050FF)
- Блок с утешительными наградами
- Эмодзи: 💪 (при наличии попыток)

**Основные кнопки:**
- Если есть попытки: "Попробовать еще раз" (синяя) + "Закрыть"
- Если нет попыток: "Закрыть" (красная)

**Управление:**
```typescript
import { useGameOverDialog } from '@/widgets/game-over-dialog'
const { show } = useGameOverDialog()

show({
  winnerName: 'Winner',
  playerPosition: 2,
  earnedPoints: 50,
  reason: 'winner_found'
})
```

**Причины проигрыша:**
- `winner_found` - найден победитель
- `time_expired` - время истекло
- `attempts_exceeded` - попытки закончились

---

### ErrorDialog ❌

**Назначение:** Техническая ошибка

**Когда показывается:**
- Ошибка сервера
- Ошибка сети
- Невозможность выполнить действие
- Системные проблемы

**Визуал:**
- Изображение ошибки (`/images/error-img.png`)
- Красная кнопка (#FF2F00)
- Ссылка на чат поддержки

**Основная кнопка:** "Закрыть" (красная)

**Особенность:** Содержит ссылку на чат поддержки

---

## Цветовая схема

| Диалог | Основной цвет | Ассоциация | Эмоция |
|--------|---------------|------------|--------|
| VictoryDialog | Зеленый #34C759 | Успех | Радость 🎉 |
| FirstPlaceDialog | Синий #0050FF | Действие | Мотивация 🔔 |
| GameOverDialog | Красный #FF2F00 или Синий #0050FF | Завершение/Возможность | Поддержка 💪 |
| ErrorDialog | Красный #FF2F00 | Предупреждение | Внимание ⚠️ |

## Логика выбора диалога

```typescript
// Псевдокод для выбора правильного диалога

if (gameEnded) {
  if (player.isWinner) {
    showVictoryDialog() // 🎉 Победа!
  } else {
    showGameOverDialog({ // 😔 Не успел
      reason: determineReason(),
      winnerName: game.winner.name
    })
  }
} else if (player.position === 1 && !player.wasFirstBefore) {
  showFirstPlaceDialog() // 🥇 Первое место!
}

if (technicalError) {
  showErrorDialog() // ❌ Ошибка
}
```

## Порядок приоритета

При одновременных событиях приоритет следующий:

1. **ErrorDialog** (самый высокий) - технические проблемы
2. **VictoryDialog** - победа в игре
3. **GameOverDialog** - проигрыш в игре
4. **FirstPlaceDialog** (самый низкий) - промежуточное достижение

## Совместное использование

Все диалоги используют Jotai для управления состоянием и интегрированы глобально в `BaseLayout`.

```typescript
// Все диалоги доступны одновременно
import { useVictoryDialog } from '@/widgets/victory-dialog'
import { useGameOverDialog } from '@/widgets/game-over-dialog'
import { useFirstPlaceDialog } from '@/widgets/first-place-dialog' // если есть хук

function GameLogic() {
  const { show: showVictory } = useVictoryDialog()
  const { show: showGameOver } = useGameOverDialog()

  const handleGameEnd = (result) => {
    if (result.isWinner) {
      showVictory({ /* данные */ })
    } else {
      showGameOver({ /* данные */ })
    }
  }
}
```

## Рекомендации по использованию

### ✅ Правильно

- Использовать **VictoryDialog** только для финальной победы
- Использовать **GameOverDialog** для всех случаев проигрыша
- Использовать **FirstPlaceDialog** только во время активной игры
- Использовать **ErrorDialog** только для технических ошибок

### ❌ Неправильно

- Показывать **VictoryDialog** при промежуточных достижениях
- Показывать **ErrorDialog** при проигрыше в игре
- Показывать **FirstPlaceDialog** при финальной победе
- Одновременно показывать несколько игровых диалогов
