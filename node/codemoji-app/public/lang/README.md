# Переводы / Translations

## Структура

```
public/lang/
├── ru/
│   └── translation.json   ← русский язык
├── en/
│   └── translation.json   ← английский язык
└── README.md               ← этот файл
```

Каждый файл `translation.json` — это JSON с вложенными ключами и текстовыми значениями.

## Как редактировать

Файлы переводов — это обычный JSON. Каждая строка имеет вид:

```json
"ключ": "значение"
```

**Менять нужно только значение (текст справа от двоеточия), ключ менять нельзя.**

### Пример

```json
{
  "common": {
    "error": "Произошла ошибка",
    "tryAgain": "Попробовать снова"
  }
}
```

Если нужно изменить текст ошибки, меняем только значение:

```json
"error": "Упс, что-то пошло не так"
```

Ключ `"error"` остаётся без изменений — он используется в коде для поиска перевода.

## Переменные `{{name}}`

Некоторые значения содержат переменные в двойных фигурных скобках — `{{переменная}}`. Они подставляются автоматически из кода.

```json
"title": "Вы выиграли {{prizePool}}!"
```

- `{{prizePool}}` — будет заменено на реальное значение (например, `"$100"`)
- Переменную `{{prizePool}}` удалять или переименовывать нельзя
- Можно менять текст вокруг неё и перемещать переменную в другое место строки

### Пример с переменной

Русский:
```json
"title": "Вы выиграли {{prizePool}}!"
```

Английский:
```json
"title": "You won {{prizePool}}!"
```

## Спецсимволы

| Символ | Значение        | Пример                              |
|--------|-----------------|--------------------------------------|
| `\n`   | Перенос строки  | `"Строка 1\nСтрока 2"`              |
| `\"`   | Кавычка внутри  | `"Нажмите \"Далее\""`               |

## Правила

1. **Не менять ключи** — только значения
2. **Не менять `{{переменные}}`** — только текст вокруг них
3. **Не удалять и не добавлять ключи** — это делают разработчики
4. **Сохранять валидный JSON** — проверяйте запятые и кавычки
5. **Оба языка должны иметь одинаковый набор ключей** — если ключ есть в `ru`, он должен быть и в `en`

## Проверка JSON

После редактирования убедитесь, что файл — валидный JSON. Можно проверить на [jsonlint.com](https://jsonlint.com) или в любом редакторе с подсветкой JSON.

Частые ошибки:
- Пропущенная запятая между строками
- Лишняя запятая после последнего элемента
- Незакрытые кавычки
