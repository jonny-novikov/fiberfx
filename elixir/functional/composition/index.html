F2 · Functional Programming

Composition & pipelines

Compose: f after g The pipe A pipeline of stages Three deep dives

Functional programs are built by joining small functions into larger ones. Composition wires the output of one function into the input of the next; the pipe operator writes that wiring left to right. A pipeline is a run of stages, each transforming the value on its way through.

Two notations describe the same idea. f(g(x)) composes f after g — g runs first, even though it is written second. The pipe rewrites this as x |> g() |> f(), reading in the order things happen. Most Elixir is pipelines: take a value, thread it through map, filter, and reduce, and read it top to bottom.

Compose: f after g

Composing two functions feeds the result of the first into the second. With inc then double, the value gains one, then doubles. The order is fixed by the nesting: the inner function runs first.

The pipe

The pipe operator writes the same chain in execution order. x |> inc() |> double() reads “take x, add one, then double” — the value moves left to right, each stage receiving it as its first argument. It computes exactly what double(inc(x)) does.

Idea

The pipe reorders the text to match the order of operations.
