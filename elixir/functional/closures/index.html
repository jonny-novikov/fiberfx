F2 · Functional Programming

Closures & partial application

A closure is a function bundled with the environment it was defined in. It can read the variables that were in scope when it was created — carrying them along after that scope is gone. That is what lets one function build another, tailored to a value decided earlier.

The folds in F2.05 took small functions as their combiners. Those functions usually need a value from their surroundings — a factor to multiply by, a threshold to compare against. A closure captures that value. Partial application is the same idea pointed at arguments: fix some of a function’s inputs now and supply the rest later. Both are built from one mechanism — a function remembering its environment.

A function that remembers

A function defined inside another can use the outer function’s variables. Return that inner function and it keeps them. make_adder takes n and returns a function that adds n to whatever it is later given — the n is captured.

Partial application

Partial application fixes some of a function’s arguments and leaves the rest open. From a two-argument add, fixing the first gives a one-argument function waiting for the second. The fixed value is captured, exactly as before.

The & shorthand

Anonymous functions are common enough that Elixir has a shorthand: the capture operator &. Inside &(…), &1 and &2 stand for the first and second arguments. It is the same function as the fn form, written shorter.

What this lands

A closure is a function carrying the environment it was defined in, so it can use a value decided earlier. That single mechanism gives you function factories like make_adder and partial application, where fixing an argument is itself a capture. The & operator is a short way to write these small functions.
