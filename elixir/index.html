How to read this

This course makes one claim and then earns it: functional programming is the algebra of functions, and Elixir is a place to run that algebra at scale. Each chapter takes an idea you already trust from mathematics and shows the line that connects it to working code.

The shape is deliberate. Algebra establishes the mindset; functional programming studies the tools on their own terms; the Elixir language gives them a syntax; algorithms put them under load; pragmatic engineering ships them; and Phoenix carries them onto the web and into real-time. Each chapter reuses the one before it, so nothing arrives without its foundation.

Pick a chapter from the arc below to see what it covers and what it stands on. The full module directory follows.

The course is one curve, not eight disconnected topics. Read it in order and each chapter is a consequence of the last.

One idea, early: composition

The thread that runs through every chapter is composition. In algebra, applying f and then g is the single function g∘f. In Elixir, that same act has a syntax — the pipe |> — which reads left to right in the order the work happens.

Three pure functions are wired below: double maps x to 2x, increment adds one, and square maps x to x². Set the input, toggle which functions take part, and watch the pipeline and its result rebuild.

A pipeline reads left to right: x |> f |> g is g(f(x)). Turn every function off and the pipeline is the identity — it returns its input untouched. F1.03 makes f∘g and its associativity precise.
