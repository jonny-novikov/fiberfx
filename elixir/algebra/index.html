Why start with algebra

You already think functionally

Functional programming didn't invent its ideas — it inherited them from algebra. Map the vocabulary once and Elixir stops looking foreign.

algebraElixir / FP
a function f : A → B→a function (first‑class)
substitution→referential transparency
composition f ∘ g→the pipe |>
x is a fixed value→immutable data
Σ, Π over a set→reduce / fold
recursive definition→recursion (no loops)
solving by structure→pattern matching

None of this is a metaphor. An Elixir pipeline is literally a composition of functions; an Enum.reduce is a finite sum written generally; pattern matching is solving an equation by its shape. F1 makes each correspondence explicit, so the rest of the course feels like notation you already own.

Chapter F1 · nine lessons

The foundation, in nine steps

Each lesson is one algebraic idea and its functional twin — concept first, then the Elixir it becomes.

01

What a function really is

Mappings, domain and range: a function sends each input to exactly one output. → first‑class functions
02

The substitution model

Replace equals with equals; an expression simply is its value. → referential transparency & purity
03

Composition, f ∘ g

Chain small functions into bigger ones; composition is associative. → the pipe |>
04

Immutability & binding

A symbol names a fixed value — you rewrite expressions, you never mutate. → immutable data
05

Sets, sequences & mappings

Collections, and applying one function across all of them at once. → lists, maps, Enum.map
06

Recursion & induction

Define a thing in terms of itself: base case and step; prove it by induction. → recursion, no loops
07

Higher‑order operators — Σ and Π

Operators that take functions; sums and products as a single fold. → map / filter / reduce
08

Equations & pattern matching

Identities, solving, and matching a value by its structure. → pattern matching & destructuring
F1.9 · lab

Functions on the plane — a plotting lab

Plot and compose functions, and watch f ∘ g take shape as curves you can drag. → an interactive plotting lab

Try it · lesson 01

A function is a mapping

Pick a function and watch where it sends each input. That's all a function is — a rule from inputs to outputs. In Elixir it's the same idea, written fn x -> ... end.

f : A → Bchoose a ruleinputs ↦ outputs↦
—

Notice each input has exactly one arrow — that's what makes it a function. Switch the rule and the whole mapping redraws at once, with nothing mutated. Hold this picture: in F4 we'll map functions over thousands of keys at once with persistent CHAMP maps, and it's the same idea at scale.
