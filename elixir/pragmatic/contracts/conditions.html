Preconditions, postconditions & invariants

A contract is an agreement between a caller and a function, and like any contract it assigns obligations. The precondition is the caller’s obligation — what must hold before the call. The postcondition is the function’s obligation — what it guarantees on return, given the precondition held. The invariant is a shared obligation on the data itself — what every operation must leave true. The value of naming them is precise blame: when something breaks, the term that broke says exactly whose fault it is, and that tells you whether you have a bad request or a bug.

Three terms, three owners. Select one to see who must guarantee it and what that buys the other side.

Who owns each term In code

Who owns each term

The asymmetry is the point. Because the caller owns the precondition, the function may assume it and need not defend against what it forbids. Because the function owns the postcondition, the caller may trust the result blindly. The invariant binds both: no operation may leave the entity in a shape its type forbids. Select a term to see its owner.

Name the owner and you name the fault. A broken precondition is the caller’s; a broken postcondition is the function’s; a broken invariant is a defect in the type itself.

In code

Write the contract down, even before you enforce it. As documentation, the three terms tell the next reader the rules; in the body they map onto a guard or check for the precondition, the returned shape for the postcondition, and a constraint every operation respects for the invariant.

# @precondition  user_id is a USR, course_id is a CRS, learner not already enrolled  (caller)
# @postcondition {:ok, %Enrollment{progress: 0}}, stored                              (function)
# @invariant     0 <= progress <= 100                                                  (always)
def enroll(user_id, course_id) do
  with :ok <- check_precondition(user_id, course_id) do
    enrollment = %Enrollment{id: Portal.ID.new("ENR"), user_id: user_id, course_id: course_id}
    Portal.Store.put(enrollment)
    {:ok, enrollment}            # postcondition holds: progress defaults to 0
  end
end

Caller and function each owe the other something; the invariant binds the data they share.
