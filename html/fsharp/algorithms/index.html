Algorithms & Data Structures

The structures a message queue is built from — re-derived in F#.

This chapter walks the same twelve rungs as the Elixir course's E4 · Algorithms & Data Structures — lists, trees, sorting, maps, HAMT, CHAMP, branded ids, persistence, a CHAMP behind an actor, recipes, dynamic programming, a lab — re-derived in F# with cross-links to each E4 sibling and notes on the F# implementation and its efficiency. The queue is made of these

A message queue is not a special structure. It is a few ordinary ones, assembled: a sorted set of time-ordered ids, a map of rows, a recurrence for the retry delay, and one writer to keep mints from racing. That is the whole chapter — each rung builds one of those pieces, and each dive shows the piece at work in EchoMQ.

EchoMQ keeps its pending jobs in a sorted set keyed by the branded id. Because the id is a time-ordered Snowflake and the Base62 encoding is fixed-width, the id order is the mint order — so the oldest job is always the smallest id, and a claim is a single pop-minimum with no second index to maintain. Claim the next job below and watch it leave oldest-first. The numbers come from the real echo_fs port.

A queue is a sorted set of time-ordered ids, a map of rows, and one writer — the same algorithms this chapter builds, assembled into a bus.
