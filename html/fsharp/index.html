How to read this

F# is a functional-first language in the ML family, running on the same .NET runtime as C#. This course takes it seriously — not as a curiosity beside C#, but as a complete way to build software: types that make illegal states unrepresentable, immutability by default, and composition as the primary verb.

The shape is deliberate. History shows where the ideas come from; the language chapter teaches F# in depth on its own terms; the C# and Elixir bridges translate the habits you already have from each community; algorithms put the language under load — the same twelve-module arc the Elixir course runs, re-derived in F#; and pragmatic programming ships a real DevOps tool. Each chapter reuses the one before it.

Pick a chapter from the arc below to see what it covers and what it stands on. The full module directory follows.

The course is one curve, not six disconnected topics. Read it in order and each chapter is a consequence of the last.

One idea, early: composition

The thread that runs through every chapter is composition. F# gives it two operators you will reach for constantly: the pipe |>, which feeds a value left-to-right into the next function, and forward composition >>, which glues two functions into one. Both read in the order the work happens.

Three pure functions are wired below: double maps x to 2x, increment adds one, and square maps x to x². Set the input, toggle which functions take part, and watch the F# pipeline and its result rebuild.

A pipeline reads left to right: x |> f |> g is g(f(x)). Turn every function off and the pipeline is the identity — it returns its input untouched. C1.04 makes |>, >>, and their laws precise.
