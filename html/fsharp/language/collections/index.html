Module C1.07

Collections list, array, seq

A small family of containers, each with one cost it is best at — pick by the cost, not by habit.

A program holds many values, not one, so F# supplies a small family of immutable-first containers. The list is the functional default — a singly-linked chain that prepends in constant time. The array is mutable and contiguous, the one to reach for when you index by position. The seq is lazy — it produces elements on demand and need never be fully in memory. And Map and Set are immutable trees for key lookup and set algebra. The containers differ in their cost: choosing the right one is choosing which operation should be cheap. This module covers the family in three dives — the three sequence types and their cost model, the module functions that transform a whole collection in a pipeline, and Map and Set for lookup and membership.

The family, in three dives

You have shaped values from the records and unions of the previous module. Programs hold many of them, and the container you store them in is a cost decision. The three dives below build that decision. Select one to read its summary and the one idea it adds.

The first dive sets the cost model — why a list prepends in O(1) but indexes in O(n), why an array indexes in O(1), and what it means for a seq to be lazy. The second covers transforming: the module functions List.map, List.filter, and List.fold that each return a new collection and chain with |>. The third is Map and Set — an immutable key→value tree and an immutable set, for lookup and for union, intersection, and membership.

One family, three cost profiles. The container you reach for is the one whose cheap operation is the one you do most.
