Chapter C2

F# for C# developers

You already know .NET. This chapter ports the habits, not the runtime.

A working C# team has been handed an F# project. Nine modules translate every C# reflex into its idiomatic F# form so the team is productive on day one: classes become modules and functions, mutable fields become immutable values and expressions, null and exceptions become Option and Result, inheritance becomes discriminated unions, loops become recursion and fold, LINQ becomes seq pipelines — and the two languages meet cleanly across one solution. Every C# and F# snippet is real and runnable.

The nine modules → Course home

How to read this chapter

F# runs on the same .NET runtime as C#: the same types, the same libraries, the same garbage collector. What changes is the defaults. Where C# makes you opt in to immutability, F# makes you opt out. Where C# uses null for "missing" and exceptions for failure, F# puts both in the type, so the compiler forces you to handle them.

So this is not a language you learn from zero — it is a set of translations. Each module takes one C# habit and shows its idiomatic F# form, with the C# you know beside the F# you are learning. Select a habit below to see the pair.

F# is not a new platform — it is new defaults on the platform you have. Learn the translations and the rest is the .NET you already know.

The nine modules

The chapter in order, the final module a hands-on lab. The arc runs from the object habits — classes, mutation, null and exceptions — through the functional core — unions, recursion, functions, sequences — to interop and a mixing-paradigms lab. Each module carries three dives, C# beside F#.
