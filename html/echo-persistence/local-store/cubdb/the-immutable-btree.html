The immutable B-tree.

Inside the file is a B-tree, and it is copy-on-write. Changing a key does not edit a node in place; it writes new copies of the nodes along the path from the root to that leaf, and a new root. Everything off the path — and the entire old tree — stays exactly as it was. Update a key and watch which nodes are new and which are shared.

Why this is the whole game

store.design.md

Copy-on-write is what links Dive 4.1 to the next module. Because an update only ever adds nodes — the path copies plus a root — it appends cleanly to the file, and because the old root is undisturbed, the version it names is still a complete, readable tree. A snapshot, then, is nothing but a retained root: no copy, no freeze. The work per write is the tree’s height, not its size, so it stays cheap as the data grows. Module 5 takes this single fact — old roots stay valid — and builds MVCC and time travel out of it.
