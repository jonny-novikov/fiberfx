The Proof · module 01

The contract is run

The bus contract is not asserted in prose — it is run against a live server, scenario by scenario.

EchoMQ.Conformance is the protocol written as a set of runnable scenarios. Each is a name and a one-line contract that drives the public surface — or raw commands where the contract is the wire — against a live server and asserts the externally visible verdict: the fence, the row shape, idempotent admission, the kind law, the lex law, the token discipline, the schedule, the morgue, the reaper, the lanes, the read / operator / watch planes, the flow family. The suite is the polyglot promise made testable: any runtime that speaks the wire is conformant exactly when it drives the same server to the same verdicts. It is all real code.

Run, don’t assert

A scenario is a name plus a one-line contract. scenarios/0 returns the whole set as a keyword list in run order; run(conn, queue) walks it, drives each one against a live server on its own sub-queue, and records the verdict. Pick a scenario below to read its contract, the verb (or raw command) it drives, and the externally visible verdict it asserts — the value an outside caller, or a port in another language, must reproduce.

A scenario is one honest row: a named contract, a live drive, an externally visible verdict. The set is the protocol — and the set is run.

The set is not a fixed number: it grows by additive minor — a new clause is appended, the prior set stays byte-frozen, the count is re-pinned. This page teaches the shape (names + contracts in run order → run on sub-queues → one CONF line each → {:ok, n}), never a count that would go stale.
