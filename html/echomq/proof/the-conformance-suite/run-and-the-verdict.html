The conformance suite · dive 02

Run, and the verdict

Walk the list once, isolate each clause on its own sub-queue, clean up behind it, and answer one verdict.

run(conn, queue) walks scenarios/0 once. For each {name, contract} it derives a per-scenario sub-queue — queue <> "." <> name — runs the clause there, purges what it minted, prints one CONF line, and records pass or fail. After the loop it prints a closing tally and returns the verdict: {:ok, n} when every clause passed, {:error, failed_names} otherwise — naming exactly which clause broke. The set grows, so n is the live total, re-pinned, never a literal on the page.

One loop, one verdict

run/2 is a comprehension over scenarios(). Each iteration runs one clause and records a {name, ok} pair; the verdict is a fold over those pairs. A clause that raises is caught and turned into a named failure — the run never crashes, it reports.

# echo_mq — EchoMQ.Conformance
# Run every scenario on a per-scenario sub-queue of `queue`, purge what it
# mints, print one CONF line each + a closing tally. Returns {:ok, n} when all
# pass, {:error, failed_names} otherwise. The set grows by additive minor; the
# count n is the LIVE total, re-pinned in the two pinning tests, never hardcoded here.
def run(conn, queue) when is_binary(queue) do
  results =
    for {name, contract} <- scenarios() do
      q = queue <> "." <> Atom.to_string(name)   # a disjoint braced keyspace per clause

      verdict =
        try do
          apply_scenario(name, conn, q)            # drive the public surface / raw wire
        rescue
          e -> {:fail, Exception.message(e)}       # a raise becomes a NAMED failure, never a crash
        end

      purge(conn, q)                               # KEYS emq:{q}:* then DEL — leave no residue
      ok = verdict == :ok
      IO.puts("CONF #{name} #{if ok, do: "ok", else: "FAIL #{inspect(verdict)}"} -- #{contract}")
      {name, ok}
    end

  failed = for {name, false} <- results, do: name
  IO.puts("CONFORMANCE #{length(results) - length(failed)}/#{length(results)}")
  if failed == [], do: {:ok, length(results)}, else: {:error, failed}
end

The verdict is a fold: pass every clause and the answer is {:ok, n} for the live total; fail any and it is {:error, failed_names} — the contract names what broke.

Sub-queues & the purge

Each clause gets its own queue suffix, so the braced keyspace emq:{queue.name}: is disjoint from every other clause's. Two clauses never collide, and a failure is named, not smeared across the run. Because the queue name carries the {…} hashtag, all of a clause's keys still hash to one slot — the suite respects the same placement law the bus does.

After each clause, run/2 deletes everything under that sub-queue's prefix — KEYS emq:{q}:* then DEL the matches. The suite leaves the server as it found it, so you can run it against a live system and it cleans up behind itself.

# echo_mq — EchoMQ.Conformance
# Delete everything under this sub-queue's braced prefix. The suite leaves the
# server as it found it, so it can run against a live system without residue.
defp purge(conn, q) do
  {:ok, keys} = Connector.command(conn, ["KEYS", "emq:{" <> q <> "}:*"])
  if keys != [], do: {:ok, _} = Connector.command(conn, ["DEL" | keys])
  :ok
end

Isolation by name plus the purge is what makes the suite safe to run against a live server: no clause sees another's keys, and none leaves any behind. The braced suffix is the same hashtag discipline the whole bus uses, so the cleanup sweep is a single-slot KEYS over one queue's keyspace, not a scan of the server.

Reading the verdict

The output is one CONF <name> ok -- <contract> line per clause, then CONFORMANCE <passed>/<total>. A green run ends {:ok, n}; a single failing clause yields {:error, [name]} — naming exactly which clause of the contract a port (or a regression) broke, with the rest of the run already cleaned up. Pattern & implementation

the pattern (Redis Patterns Applied)

A coordination test must isolate each case and clean up after itself, or one case's residue poisons the next. Coordination teaches the atomic primitives the run asserts.

→
the implementation (echo_mq)

run/2 gives every clause a disjoint braced sub-queue and purges it — isolation by name, no residue, one named verdict — and a raise becomes a named failure, so the run reports rather than crashes.

The pattern asks for isolation and cleanup; the run gives both by name, and answers one verdict the contract can stand behind.
