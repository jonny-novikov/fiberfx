The Lua layer

Every state change is one atomic script. The wire is not a list of commands — it is the set of scripts.

To enqueue a job is to run the @enqueue script; to claim one is to run @claim; to finish one is to run @complete. Each is the whole transition — read, decide, write — in a single server-side step no other client can interleave. That is why the scripts are the protocol: the transitions live below the language line, every key declared in KEYS[], loaded once and run by SHA. This module reads the @enqueue script in two beats, the declared-keys law, and the EVALSHA dispatch.
