R0.2.1 · Valkey under codemojex — dive 1

The facade seam.

One seam holds the whole system together: every surface calls the same boundary — the echo data layer — and Valkey lives below it, never on a surface’s path. The one owned client is EchoWire; EchoStore and the EchoMQ bus ride it, both over Valkey.

Grounding: echo/apps/echo_wire (the connector + the version fence) · content/bcs3.1.md (the fence live) · bcs4.1.md (the EchoStore read path) — every figure verbatim from a committed record.

§1

One boundary, one return set

echo/apps/echo_wire · the master invariant

The codemojex reaches Valkey through one boundary, the echo data layer. The master invariant is a rule about every surface above it: a surface calls only that boundary and reads only a typed result from a closed set. A surface never opens a socket, never holds a key, never branches on a raw store error. EchoWire — the one owned Valkey client — is below it.

The closed return set is the other half of the seam. Every call returns either a success or one of a fixed, small set of typed errors — :disconnected when the socket is gone, :overloaded when in-flight is full, {:version_fence, got} when the boot fence fails — never a raw exception. Because the set of outcomes is closed, the surface’s handling is finite. How the connector resolves a call, pipelines, and reconnects is the subject of the EchoMQ course, not repeated here.

§2

Every outcome is one of a closed set

content/bcs4.1.md

A condition below the boundary — an EchoStore L1 hit, a cold read that runs the loader once, a dropped socket, a full in-flight queue — never escapes as a raw store condition. The data layer folds each one into the closed return set, and the surface reads the typed result. The committed EchoStore walk shows the read path the seam hides:

A failure below the boundary never reaches a surface as a raw store error. The data layer returns a success or one of a closed set of typed errors — and the surface handles exactly that.

§3

The seam at the edge — and at the wire

echo/apps/codemojex · content/bcs3.1.md F3

Take one call: the play API admits a guess. The Telegram webhook hands each decoded update to Codemojex.EchoBot.ingest/1, which mints a branded CMD and enqueues the command; the play surface itself reads a typed result from a closed set — never a raw exception:

# A surface (the play API) calls ONLY the consumer’s seam and matches ONLY the closed result set.
case Codemojex.Guesses.submit(round, player, emojis) do
  {:ok, :enqueued} -> await_score()
  # one of a closed set: :no_round | :closed | :expired | :bad_guess | :insufficient
  {:error, reason} -> reject(reason)
end

Nowhere in the play API is there a socket, a key, or a parse branch on a raw field. And the discipline reaches all the way down: even the wire’s boot is typed at its own seam. At every connect EchoWire’s connector reads {emq}:version and refuses any other value with a typed error — the same shape as the consumer’s closed set, applied at the wire.

the application · the typed seam, twice

The edge above, the version fence below

The guess’s outcomes are typed by the consumer’s seam; the connector’s boot is typed by the fence. The committed line reads the fence back through the fenced connection itself — on the emq:{q}: keyspace the bus owns.

Notes on ValkeyThe EchoStore L2 row is written with SET … PX, so the second layer expires itself on the server’s own clock even when no node sweeps it — PTTL 300 ms of 300 in the committed record — valkey.io/commands/set.

the invariant

A caller depends on one stable interface and a closed set of outcomes; the store behind it is an implementation detail that can change without the caller noticing.

The seam is the master invariant the whole course respects: the surface above the boundary, EchoWire and Valkey below it, and a closed set of outcomes between them.
