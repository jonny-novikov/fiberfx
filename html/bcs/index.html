B0 · Orientation — the Branded Component System course

Boundaries around systems.
Identity as a contract.

Encapsulation boundaries are drawn around systems, not objects. The only values that cross are identities, and messages about identities. And identity is a contract — the 14-byte branded snowflake: typed by its namespace, ordered by its mint time, placed by its hash, identical in meaning across five runtimes against one canon.

The course teaches the BCS manuscript Part for Part. Every figure on every page is quoted verbatim from a committed output — a frozen transcript, a normative vector, a bench record. The worked project is a trading system; the bus between its systems is EchoMQ, backed by Valkey.

§1

The law, in three clauses

content/bcs1.md

Clause one

Systems own their state and behavior

A system is the unit that encapsulates: a property store, the code allowed to touch it, and a boundary everything else stays behind. Ownership is total or it is not ownership; a sweep over somebody else’s table is a data race wearing a method’s clothing.

failure retired · the reach-through — a change to one system’s representation breaks code in another system’s file.

Clause two

Only identities, and messages about identities, cross

A message may say generate an image for this id with this prompt; it may never carry the entity itself, a pointer to it, or a live structure entangled with one. An identity is the one value that copies freely without forking truth, because it names state rather than carrying it.

failure retired · the traveling object — the entity, handle, or pointer that escapes its owner.

Clause three

Identity is a typed, ordered, placed contract

The value everything crosses on cannot be the weakest value in the program. The contract gives identity four load-bearing properties — namespace, order, placement, canon — and each one retires a failure that otherwise reappears as bespoke machinery, system by system.

failures retired · the silent join · the second clock · the routing table · the dialect

The composition: clause one makes the crossings enumerable → clause two makes every crossing typeable → clause three makes the typing checkable before anything runs. Three clauses, one law — the inter-system architecture verified by the type checker, the schema compiler, and the storage domain.

§2

The contract, property by property

content/contract.md

typed

The namespace rides in the first three bytes and in the type parameter. The wrong-table join dies at compile time, at the schema layer, and at the storage domain — before runtime.

ordered

String order is mint order — the order theorem. A table keyed by branded id is already a timeline; a feed is a descending sort; a time window is two synthetic cursors.

placed

hash32 is placement any holder computes identically: the same function routes a CHAMP slot, a cache shard, and a hash partition — arithmetic, not a directory service.

canonical

One Rust source, one C reference, one vector file, a conformance suite per runtime. Which language a system is in becomes a deployment detail.
