B1.4 · dive — the three deaths

Three boundaries, three deaths.

The handle’s validity is scoped to one process’s current memory layout, and each scope violation has a name: the save file, the socket, the foreign store. Each death traces to a missing property of the identity contract — and each of the contract’s answers is already a committed measurement.

Source: content/bcs1.4.md · What and the diagnosis; the contract’s answers are on stage in bcs_rung_1_1_check.out.

§1

The deaths, named

content/bcs1.4.md · What
DEATH 1

The save file

An index is an address into an array arrangement that no longer exists after restart, so persistence becomes a swizzling pass — every table rewritten from handles to something durable and back — and the generation pattern serializes into noise.

DEATH 2

The socket

Entity 4117 on machine A names nothing on machine B, so replication mints a second identity scheme with bidirectional mapping tables at both ends — the dialect failure, self-inflicted before any foreign runtime is even involved.

DEATH 3

The foreign store

A handle cannot be a foreign key, so the database grows its own serials; created_at columns sprout beside them — the second clock; shard keys are invented per table — the routing table; and nothing anywhere checks that the integer arriving in the assets table was ever an asset — the silent join, at last, with no compiler in reach.

Three boundaries, one pathology: the handle’s name embeds the process’s private circumstances, so the name dies wherever those circumstances end.

§2

The diagnosis, in contract terms

content/bcs1.4.md · quoting bcs_rung_1_1_check.out

A handle is placement and liveness wearing identity’s clothes. The index is placement, leaked into the name — which is why the name dies whenever the layout does. The generation counter is a liveness check, and its own literature is candid that the check “isn’t waterproof” — collision detection over a reused slot, probabilistic by construction. The contract separates what the handle fused, and two of its answers are already in the committed transcript:

What remains in the name is exactly what a name should carry — kind and instant — at 65 bytes per key on the measured table, cheaper at rest than the handle’s own decimal shadow.

§3

What the handle fused, separated

content/bcs1.4.md · the diagnosis

Generation counters are rejected by subsumption: detection of reuse is replaced by prevention of reuse, and a probabilistic guard for an impossible event is deleted weight. The store-side half of the third death — keys, serials, and shard schemes as a design surface — is taught applied in the Redis Patterns course.
