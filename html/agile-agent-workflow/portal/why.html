Why an end-to-end exemplar

A6 makes each rung production-grade. A7 exists because a method is proven only when the whole loop runs end to end on a real system — a sequence of hardened rungs is not a demonstrated, composed run.

The failure A7 prevents is the one every teaching course risks: techniques proven in isolation that never compose into a shipped system. Seven correct lessons do not add up to a deployed Portal; one composed run does. Why the run is its own Part

By the end of A6, every move the course teaches has been proven: A1 made the case for thin slices, A2 decomposed a vision, A3 planned delivery, A4 defined and proved a rung, A5 briefed an agent to build it, and A6 hardened the result to production quality. Each was demonstrated on a worked fragment. None of that, on its own, is a shipped system.

A7 is its own Part because composition is a separate claim from correctness. A technique can be correct in isolation and still never assemble into a deployed product — the seams between moves are where real systems fail, and seams only appear when the moves run in sequence on one codebase. The exemplar makes that claim concrete: it runs decompose, roadmap, spec, brief, build, harden, and accept once, uninterrupted, from an empty repository to a deployed Portal. The proof is the whole run, not the sum of the lessons.

The A6→A7 seam

Read the chain backward and the seam is exact. A6 delivers hardened rungs. A7's job is the one thing A6 does not produce: a run that assembles those rungs into a shipped system. Selecting a state below reports what each layer holds — what exists after A6, what is still missing, and what the composed run produces.

The seam is the load-bearing reason A7 follows A6: hardened rungs are the input; the composed run that ships them is the work A7 does.

On the Portal

The Portal is what makes the claim testable. It is one real, multi-surface system — a branded store, an event-sourced engine behind one facade, a Phoenix web app, a Telegram bot, and a student dashboard — carried from an empty repository to a deployed platform. Running the loop on the Portal means the seams between the moves are exercised on a system that actually has to run, not on seven disconnected fragments.

The companion Functional Programming in Elixir course builds the Portal's surfaces for real, with the OTP internals it cites rather than re-teaches; its F6 Phoenix chapter is the web surface, shipped end to end from a real roadmap. A7 is the agile-side mirror: the same system, run as one composed loop, to show the method holds whole.

The principle

A sequence of correct techniques is not a demonstrated method; the method is the whole loop run once, from nothing to production, with the seams between moves exercised.
