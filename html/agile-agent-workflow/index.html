Agile Agent Workflow · course overview

The agile agent workflow

Pragmatic Programming with Claude Agents — a way of building software in which a human supplies judgement, taste, and acceptance, and a Claude agent supplies fast, exact implementation. Paired over thin, provable increments, the two ship reliable systems neither would reach alone. The course opens with the framework you will build — two layers, four artifacts, one loop — then teaches each part in depth and proves it on one running project.

The practical project is the Portal: a learning platform carried from an empty repository to a deployed, multi-surface system — a branded store, an event-sourced engine behind one facade, a Phoenix web application, a Telegram bot, and a student dashboard. Every idea in the six modules is practised on it, and the exemplar runs the whole loop, from zero to production.

The course arc

Start with A0 — Foundations: the framework the rest of the course fills in. Its three pages are the whole method in miniature — the two layers, the four artifacts, and the loop — and they are built and readable now. A1 makes the case for that framework. A2 and A3 teach decomposition into user stories and the roadmap that sequences them; A4 and A5 define the spec and the agent brief that turn a rung into running code; A6 hardens every increment for reliability and correctness. The exemplar, A7, then runs the whole loop on Portal. The through line: a Claude agent is a fast, tireless implementer of well-specified thin slices, and the human is the source of decomposition, judgement, and acceptance.

A workflow earns its keep at the boundary: A6 keeps each increment correct, and A7 proves the method ships.

Portal Exemplar · A7

Everything the six chapters teach converges here. Portal is built the way the course prescribes — one value ladder per chapter, each rung decomposed into stories, planned in a roadmap, defined in a spec, briefed to a Claude agent, and accepted only when its invariants hold — and carried from an empty repository to a clustered, deployed platform. The exemplar is seven steps, each run through the same loop.

the workflow

Decompose into stories, plan a roadmap, write the spec and the agent brief, build, and prove — correct by definition.
