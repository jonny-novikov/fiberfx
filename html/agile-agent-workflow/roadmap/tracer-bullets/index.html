A3.7 · The roadmap layer

The tracer bullet

Ship one thin thread through every layer first, kept and grown — not a layer built fully, and not a prototype to discard.

A roadmap stays honest about uncertainty when it threads one thin slice end to end before deepening any layer. The exemplar is the Portal's real web chapter — F6 (Phoenix): its tracer is rung f6.1, the engine served as a web app (request → facade → render), shipped before persistence (f6.3) or the rendered catalog (f6.5). Walk the ladder on the right to find it.

The tracer thread The three dives References

Thread first, then depth

A backlog can be ordered two ways. Breadth-first builds each layer fully — the whole data layer, then the whole facade binding, then the whole view — and integrates last, when the most risk is left to discover. Depth-first ships one thin thread through every layer first: a tracer bullet. The thread proves the seams on day one, and every later rung deepens the slice that already works.

On the Portal's web chapter F6, the tracer is rung f6.1: the engine served as a web app — a request enters the endpoint, calls the Portal facade, and renders a page. Its demo is “hit the root, see a page”; its harness is a single GET smoke. It is shipped before persistence (f6.3) or the rendered catalog (f6.5), and it is a real thin feature that is kept and grown — not a prototype to throw away. Every rung holds the master invariant: “The web layer calls only the Portal facade and renders only the closed %Portal.Error{} set.”

the principle

Ship a thin end-to-end thread first — a tracer bullet — to de-risk integration early; keep it and grow it. A prototype is built to learn, then discarded.
