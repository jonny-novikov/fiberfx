The reference module

A3.9 is the chapter's — and the course's — appendix. The teaching modules build the argument; this one indexes it. A glossary is a closed, data-driven list: a fixed set of terms where each entry carries a term, a one-line abstract, its Source, and a link to the /elixir/phoenix chapter where the framework implements the idea. It is the place to look a concept up and jump to its real code — not the place to learn it for the first time.

Closed means complete: no entry without a real abstract, a real Source, and a real, resolving framework link. That is what makes the reference module a tool rather than a summary. Every term resolves three ways — to a sentence that defines it, to a book that grounds it, and to a route you can open to see it built.

the principle

A glossary is an index, not a tutorial — a fixed list of terms, each pointing outward to where it is defined and where it is used.
