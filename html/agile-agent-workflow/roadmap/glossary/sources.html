The bibliography

Ten vetted sources, each a real external link. A source entry carries a title, an author, an abstract that says what the source gives the workflow, and the modules that cite it. The sources are the canon the course stands on — not decoration, but the set of ideas the framework was built to.

The Pragmatic Programmer — Hunt & Thomas

Tracer bullets, the single source of truth, DRY, and design by contract.

cited by A1 · A3 · A3.9

Extreme Programming Explained — Beck

Small batches, iterations, continuous feedback, and the inspect-and-adapt loop.

cited by A1 · A3 · A3.1 · A3.2

Specification by Example — Adzic

The spec as executable, shared truth; the four artifacts a rung is built from.

cited by A1.04 · A3.3 · A3.9.1

User Stories Applied — Cohn

The Connextra story form, value over tasks, and the value ladder.

cited by A2 · A3.9.1

Continuous Delivery — Humble & Farley

Keeping the system releasable at every increment; the roadmap as the delivery plan.

cited by A3 · A3.9 · A3.9.1

INVEST in Good Stories — Wake

The six tests of a good story: Independent, Negotiable, Valuable, Estimable, Small, Testable.

cited by A2

Gherkin reference — Cucumber

The Given/When/Then form of an acceptance criterion.

cited by A2 · A3.9.1

User-story template (Connextra) — Agile Alliance

"As a … I want … so that …" — the role-need-value shape of a story.

cited by A2

The llms.txt convention — llmstxt.org

A machine-readable map of a site or course for agents.

cited by A0 · A3.9

Building effective agents — Anthropic

The agent as an implementer of well-specified, decomposed work; the Author/Operator division.

cited by A0 · A1.03 · A3.9.1

Every Source above is a real, vetted external link, reused across the course so the same idea always points at the same reference.
