Vision to stories

The vision, run through four moves

The Portal vision is one line: learners enrol in a course and study it. A vision names a destination, not a slice. To turn it into work the loop can build, run the first four A2 moves on it in order — each one a technique from an earlier module, now applied to a single input.

Value, not tasks (A2.01) comes first. The vision implies chores — “add a courses table”, “build a Phoenix route” — and those are not stories. Restate each as a change in what a role can do: a learner browses the catalogue, enrols, opens a lesson, tracks progress. The Connextra form (A2.02) writes each as role, want, reason, so the value is on the card. INVEST (A2.03) scores each against the six readiness tests. Given/When/Then (A2.04) pins each ready story with one concrete scenario — the executable definition of done an agent and a test both read.

value
A unit of work is a change a role can use, not a technical chore. The vision's tasks are restated as four learner-facing capabilities.
form
Each capability is written “as a learner, I want …, so that …” — the value stated on the card before anything is built.
INVEST
Six yes/no tests read before hand-off. Three candidates pass 6/6; “track progress” fails Small (and the Estimable that follows).
acceptance
One Given/When/Then scenario per ready story — the concrete confirmation that closes it, the same line the Author builds to and the test accepts.

A2.04 — Given/When/Then acceptance — is applied here as the fourth move; the A2.03 INVEST gate is the third. Both are read before any code is generated, because a sentence is cheaper to repair than a rung.

The first story set

Running the four moves on the vision yields a first set of four stories. Three — browse, enrol, open a lesson — pass all six INVEST tests and carry a Given/When/Then; they are ready to hand to the Author. The fourth, “track progress through a course,” is valuable and testable in principle but fails Small: it spans every lesson and three surfaces at once, so it cannot fit one rung and cannot be estimated. It is kept as a story and flagged — the input to the next dive.

Switch the story below to read its INVEST score and where it stands. The readout reports the pass count, the first failing letter when there is one, and how many of the set are rung-ready.

The first set, written out

On the Portal — the learning platform this course builds — the four moves produce these stories. Each ready one carries a role-want-reason card and a Given/When/Then scenario; the too-big one carries its INVEST verdict and a note that it splits next. The Portal’s internals — the branded store and the event-sourced engine behind one facade — are built in the companion course; here the work is the stories, not the code.

# Vision: learners enrol in a course and study it.
# Run value → form → INVEST → acceptance on each candidate.

browse the catalogue   # INVEST 6/6 · ready
  As a learner, I want to browse the catalogue, so that I can find a course to study.
  Given a non-empty catalogue, when a learner opens it, then every published course is listed.

enrol in a course      # INVEST 6/6 · ready
  As a learner, I want to enrol in a course, so that it is added to my courses.
  Given a learner viewing a course, when they enrol, then it appears in their courses.

open a lesson          # INVEST 6/6 · ready
  As a learner, I want to open a lesson, so that I can study it.
  Given an enrolled learner, when they open a lesson, then its content is shown.

track progress         # INVEST 4/6 · fails S, E — too big, splits in A2.07.2
  As a learner, I want to track progress, so that I can see how far I have come.
  # spans every lesson and three surfaces — kept, flagged for splitting.
the principle

A vision is decomposed by applying the story disciplines in order — value, form, readiness, acceptance — not by guessing at tasks.

Recap — four moves, a first set

The vision became a first story set by running value-not-tasks, the Connextra form, INVEST, and Given/When/Then on it in order. Three stories are ready — each with a card and an acceptance scenario; one, “track progress,” is kept but flagged too-big. The set is not the backlog yet: it has a story that cannot fit a rung, and an unordered list. The next dive takes the flagged story and splits it.
