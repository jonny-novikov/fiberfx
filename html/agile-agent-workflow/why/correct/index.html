A1.05 · Why an Agile Agent Workflow

Correct by definition

There is one word left open: done. In this workflow it is not a feeling and not a sign-off — it is a closure over traced, executed checks. A rung is done when its closure is complete.

A1.01 named the unit a rung should be; A1.02 named the principles that keep it ownable; A1.03 gave the loop its motion; A1.04 split the two layers. This module fixes done: every deliverable realized by a story, every story accepted, every invariant proven — and the mechanical quality gates that turn “A+” from an opinion into a repeatable result. Three dives: what done is, how it is established, and what holds it.

What done means The three dives References

Done is a closed set

Across this chapter a rung has been the unit of work: a thin slice of value, specified only enough, built and proven before the next begins. This module names the test that says it is proven. A rung carries a closure — a finite, named set of acceptance checks, fixed up front from the spec. The rung is done when the set is closed: every check has executed and passed, and nothing is outstanding.

That is a stricter claim than “looks done.” “Looks done” is a statement about appearance; “closure complete” is a statement about a set you can enumerate and re-run. And a majority is not a closure — four checks of five green is not done, because the one outstanding check is usually the sad path everyone wanted to skip. Toggle the five checks for one Portal rung below; the closure ring closes only when all five are proven.

the principle

Done is a closed set of executed, passing checks — defined up front and complete only when every member is green. A majority is not a closure.

on the Portal

The “enrol” rung is done only when all five acceptance checks — including “enrol twice → no duplicate” — have executed green; four of five leaves it not done.

A rung is done when its closure is complete — every check in the set executed and green, nothing outstanding — not when it looks done.
