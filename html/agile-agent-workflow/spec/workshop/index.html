Six moves, one spec

The spec layer defines what we build and prove for each rung the roadmap orders. The workshop is that layer applied once, end to end, on the Portal's engine — F5.1 — Portal's engine-facade rung, the engine chapter's first rung and the seed of the F6 master invariant. It runs in six moves, and each produces a named part of the spec.

By-example replaces an abstract requirement with worked cases — F5.1-US1's three: the app boots on port 4000, POST /enroll returns 422 from the stubbed boundary, an unknown path returns 404. The triad splits the work across f5.1.md, f5.1.stories.md, and f5.1.llms.md. The anatomy fills the six sections of the spec — Goal, Rationale, Scope, Deliverables (D1–D7), Invariants, Definition of Done. The stories derives five Connextra stories (US1–US5) from the deliverables. The invariants states the five (INV1–INV5). Traceability closes the chain with a Coverage line that maps every deliverable to a story.

Over the sequence sits F5.1-INV1, the master-invariant seed, quoted verbatim: “the web layer holds no domain logic … so F6 can replace Bandit with Phoenix untouched.” The output is not invented — it reproduces the real f5.1.md and f5.1.stories.md, including the Coverage line that closes them.

the principle

The full spec-layer sequence run once on a chapter reproduces a closed, acceptable spec: examples become checks, the triad splits the work, the anatomy is filled, stories derive from deliverables, invariants are stated, and the chain closes.

The three dives

Each dive carries one pass of the workshop on the real F5.1 rung, in the arc read the deliverables → run the sequence → check the spec closes: first read F5.1's seven deliverables and the replaceable-seam invariant; then step the six A4 moves over the triad and watch the spec take shape; then check the produced Coverage line against the real one and confirm it closes. Read them in order — A4.7.1 starts.
