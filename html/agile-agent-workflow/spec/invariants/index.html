A4.5 · The spec layer

Invariants that must always hold

An invariant holds for every value, always. An acceptance check holds for one scenario. Naming the two apart is how a spec states the rules a slice must never break, separately from the examples that prove it behaves.

An invariant is universally quantified — true for all values, at all times, named once and exercised by checks. An acceptance check is one named scenario — a single Given/When/Then that proves one example. The Portal's master invariant is the canonical example. An invariant, or a check

A spec carries two kinds of claim about a slice. An invariant is a rule that holds for every value the increment can take — it is named once, in the spec's Invariants section, and quantifies over all inputs. An acceptance check is one scenario — a single Given/When/Then on a story that proves one example behaves. The first says for all; the second says for this case.

Select a statement below to classify it. The master invariant and “the error set is closed” are invariants — universal rules. A single Given/When/Then is a check — one scenario. The readout names the kind and why.

An invariant is a universal rule; a check is one scenario — the spec names them apart, and a rung is done only when every invariant holds and every check passes.
