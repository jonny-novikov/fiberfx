A4.2 · The spec layer

The triad

A rung is not specified by one document. It is specified by three — the spec, the stories, and the agent brief — each answering exactly one question.

The spec answers what we build, why, and when it is done. The stories answer who wants what, and how we will know. The agent brief answers how to build it, with proof gates. Cram all three into one file and each reader is failed; keep them apart and each file reads for one audience. The real triad is f6.1.{md,stories.md,llms.md}; the second, fittingly, is the course's own a4.*. Three dives: which question, the real triad, and the course itself.

The three files The three dives References

One question per file

A specification of a rung must say three different things, and each thing has a different reader. What we build and why and when it is done is the contract, read by an author and a reviewer. Who wants the rung and how we will know it works is the user's side, read by product and QA. How an agent builds it, with the gates that prove it is the brief, read by a coding agent. Three questions, three readers, three files.

The discipline is keeping them apart. One file per question means each reads cleanly: the contract is not buried in build steps, the stories are not buried in references.
