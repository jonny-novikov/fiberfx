Introduction

Mercury is a UI-kit that stays out of the way.

Mercury ships a set of framework-agnostic CSS primitives, a React component library, and a Figma library that all speak the same token vocabulary. Product teams install one package and get consistent typography, color, spacing, focus rings and motion across every surface — from marketing pages to authenticated dashboards.

It's designed to be boring in the right places. Defaults are opinionated so you rarely need overrides; tokens are exposed when you do. The goal is a system that serves your product, not a design system that serves itself.

Installation

Install in one line.

Mercury is distributed as a single package on npm. It has zero runtime dependencies beyond React 18+ and works with Vite, Next.js, Remix, and plain HTML.

Terminal 
# Install the package
npm install @mercury/ui @mercury/tokens

# Peer dependency
npm install react@^18 react-dom@^18
Terminal 
pnpm add @mercury/ui @mercury/tokens
Terminal 
yarn add @mercury/ui @mercury/tokens
HTML 
<link rel="stylesheet"
      href="https://cdn.mercury.dev/v2/mercury.css">
<script src="https://cdn.mercury.dev/v2/mercury.js"></script>

Import the stylesheet

Mercury ships as pure CSS so Server Components and static HTML work without a runtime. Import it once at the root of your application:

app/layout.tsx 
import '@mercury/ui/styles.css';
import '@mercury/tokens/light.css';  // or dark.css

export default function RootLayout({ children }) {
  return <html lang="en"><body>{children}</body></html>;
}
Quickstart

Your first screen, in 30 lines.

Here's a complete settings card using the primitives you'll use most: Stack, Card, TextField, Switch, and Button. Everything is themeable via tokens and fully accessible out of the box.

TSX 
import { Card, Stack, TextField, Switch, Button, Heading } from '@mercury/ui';

export function NotificationSettings() {
  return (
    <Card padding="lg">
      <Stack gap="md">
        <Heading level={2}>Notifications</Heading>
        <TextField label="Reply-to address" placeholder="team@acme.co" />
        <Switch label="Weekly digest" defaultChecked />
        <Switch label="Product updates" />
        <Button variant="fill">Save changes</Button>
      </Stack>
    </Card>
  );
}

What you get

  • Keyboard navigation across every interactive element, including focus traps inside dialogs.
  • Screen-reader labels inferred from label props — no extra aria-* required.
  • Dark mode automatically, driven by prefers-color-scheme or a data-theme override.
  • Type-safe props with autocomplete for every token value.
Project setup

Wire it into your build.

Mercury works without any bundler configuration, but the following optional steps unlock tree-shaking, CSS-variable tooling, and theme switching.

Root provider (optional)

Wrap your app in MercuryProvider to enable portals (for dialogs and toasts), global toasts, and theme switching:

TSX 
import { MercuryProvider } from '@mercury/ui';

<MercuryProvider theme="auto" brand="acme">
  {children}
</MercuryProvider>

TypeScript

Types ship inline with the package. No extra @types/* install required.

Browser support

Mercury supports the last 2 major versions of Chrome, Safari, Firefox and Edge. IE11 is not supported.

Foundations

Color.

Mercury uses a dual-layered color system: primitives (the raw palette) and semantics (what a color means in context). Use semantics in product code. Primitives are escape hatches for marketing and illustration.

Brand primitives

TokenValueUse
--color-piggyrgb(78 70 180)Primary brand — CTAs, focus rings, key moments.
--color-roshirgb(255 78 100)Destructive and error states.
--color-krillinrgb(255 179 25)Warnings, pending states, highlights.
--color-bulmargb(138 207 222)Informational surfaces and illustrations.
--color-dodoriargb(162 164 48)Success and positive confirmation.
!
Don't hard-code hex values.

Always reference a token. color: #4e46b4 will silently break in dark mode and when a brand is rethemed. Use color: var(--color-piggy) instead.

Foundations

Motion.

Motion is meaningful, not decorative. Durations are capped — most UI transitions land between 120ms and 240ms. Longer animations are reserved for celebratory moments and onboarding.

  • Duration tokens: --dur-snap (80), --dur-fast (120), --dur-base (200), --dur-slow (320), --dur-languid (520).
  • Easing tokens: --ease-out-quad for entry, --ease-in-quad for exit, --ease-spring for overshoot on meaningful actions.
  • Reduced motion is respected automatically via prefers-reduced-motion — components fall back to opacity-only transitions.

Components · Inputs

Button.

Buttons trigger actions. Use one fill button per view to represent the primary action. Outline and ghost variants carry secondary and tertiary actions.

Components · Feedback

Toast.

Toasts confirm that an action completed. Use them sparingly — reserve toasts for actions the user initiated, not background events.

TSX 
import { useToast } from '@mercury/ui';

const toast = useToast();

toast.success('Changes saved', { description: 'Your settings are live.' });
toast.error('Upload failed', { action: { label: 'Retry', onClick: retry } });
Components · Navigation

CommandPalette.

Open with ⌘ K (or Ctrl K). A single registry of actions, searchable and navigable entirely from the keyboard.

  • Scoring is fuzzy and prefix-weighted — typing new ranks New file above Renew subscription.
  • Groups render as sticky section headings inside the palette.
  • Recents persist per-user in localStorage and surface when the palette is empty.
Usage

Accessibility.

Mercury targets WCAG 2.2 AA. Every component ships with a regression-tested a11y baseline — keyboard, screen reader, and color contrast are verified in CI.

  • Focus rings never disappear. You cannot disable them from props.
  • Color is never the only channel. Status uses color + icon + text.
  • Motion respects prefers-reduced-motion. Parallax and shake animations are suppressed.
  • Every label prop becomes a visible label and an aria-label when visually hidden.
Usage

Theming.

Mercury supports 12 shipped brand themes, plus a token-override API for custom brands. Switch themes at runtime by changing a single attribute on the root:

HTML 
<html data-theme="dark" data-brand="acme">
  <!-- every component retints automatically -->
</html>

Contributing.

Mercury is maintained by the Platform Design team and accepts external contributions. New components require a proposal (an ADR) before implementation. Token additions are reviewed weekly.

  • Read the contributor guide before opening a PR.
  • All PRs require one design reviewer and one engineering reviewer.
  • Release cadence is weekly, on Tuesdays.
