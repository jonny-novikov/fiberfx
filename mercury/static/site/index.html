01principles

A system serving products, not itself.

Mercury is built around three principles. Each rules out a kind of waste — of time, of effort, of quality — and together they keep the system honest.

01 · efficiency

Efficiency, not consistency.

Consistency is a result of efficiency, not the other way around. When the system is intuitive to use, consistency becomes an automatic by-product of adoption.

"A design system isn't a project. It's a product serving products."— Nathan Curtis, EightShapes
02 · agility

Dynamic capability + stability.

Agility needs two things: a dynamic capability — speed, nimbleness — and a stable foundation that doesn't change. Mercury is both.

"The longer you delay decisions, the more expensive they become."— Craig Villamor, Google Maps
03 · quality

Quality over quantity.

Focus on the best solutions. Quality emerges as the consolidation of agility and efficiency — and compounds as each component iterates.

"Good design is a language, not a style."— Massimo Vignelli
02foundations

Tokens first. Components second.

Every style — colour, type, radius, elevation — lives in a token. Themes swap tokens; components never change. Iris is the brand; Indigo drives interaction; Slate is the neutral spine.

04getting started

Install. Import. Ship.

Mercury ships as a TypeScript + Vite library with an optional Effector adapter. No CLI, no config matrix.

Install
pnpm add @mercury/ui
pnpm add @mercury/effector
# peers: react, react-dom, effector
Use
import { Button } from "@mercury/ui";
import "@mercury/ui/styles.css";

<Button variant="primary">
  Ship it
</Button>
