## {emq-4-4-learnings} Learnings

### L-1 — A terminal-state assertion is a weak no-op-defeater for a fair-rotation liveness property

When a conformance scenario proves a FAIRNESS / anti-starvation property, asserting the terminal state ("every lane eventually drains to depth 0") is gate-invisible to the very bug it should catch: a no-rotation / FIFO / serve-heavy-to-exhaustion-first mechanism ALSO reaches the terminal state, because the re-ring guard advances the ring head as each lane empties. Terminal liveness ≠ fair interleaving.

The load-bearing shape is a **bounded-early-window interleaving witness**: assert that every light lane is served at least once within the first few ring cycles WHILE the heavy lane is still deep (far from exhaustion). A no-fair-share rotation cannot satisfy it (the light lanes are stuck behind the heavy backlog in the early window); fair round-robin reaches them in the first cycle. Proven both ways here: the no-rotation mutation (LMOVE→LINDEX) goes RED on the interleaving witness and GREEN on a terminal check.

Corollary (anti-redundancy): pair the interleaving drill with a separate proportion-band scenario — the Director's two mutations showed each catches a DISTINCT fault (rotation vs. weight), so neither is vacuous. For Apollo to fold into the echo-mq-implementor / echo-mq-architect conformance craft: "for a fairness property, witness the interleaving in a bounded early window, do not assert only the terminal drain."
