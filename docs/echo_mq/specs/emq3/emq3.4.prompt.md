## The rung in one paragraph

The destructive blast radius is bounded by the declared window — a trim can NEVER delete inside it (INV4, the load-bearing invariant: the conformance scenario POSITIVELY proves in-window survival + below-window deletion, never a no-op). ## Risk tier — HIGH (a DESTRUCTIVE at-rest op — `XTRIM` permanently removes entries, the emq.4.1 `drain/3` class)

emq3.4 crosses ONE HIGH-risk trigger — a destructive at-rest op — and the gate is shaped to THAT trigger, not emq3.3's process triggers:

- **A DESTRUCTIVE at-rest op** — `XTRIM` PERMANENTLY removes stream entries (the emq.4.1 `drain/3` precedent: permanent entry removal). The hazard is **over-deletion**: a trim that removes an entry INSIDE the declared window is silent data loss. So the gate is the **blast-radius mutation battery** (mutate the trim bound / the approx-exact flag / the `MINID` floor — each mutation a CAUGHT over- or under-deletion) + a POSITIVE conformance proof (INV4 — the scenario appends inside AND below the window and asserts in-window survival + below-window deletion; a no-op fails INV4's letter). **The destructive-op gate is the load-bearing one this rung.**
- **NO new process, NO lease** — unlike emq3.3, emq3.4 adds NO supervised process holding a blocking lane and NO lease key. The `trim/_` verb is a stateless host fn; the driver is an opt-in `:transient` child whose decision core is a PURE function of the injected clock (no same-ms mint race to flake across runs). So the **≥100 determinism loop is NOT run** — the rung mints NO branded ids (it consumes the writer's; the `MINID` floor derives from a caller `DateTime`) and opens NO lease, so running the loop would FORGE a same-millisecond-mint load this rung does not introduce (a false rigor signal). The posture is a **MULTI-SEED sweep + an honest determinism-posture statement** (the program law for a no-mint/no-lease rung).

But the rung touches **NO frozen line** — ZERO `echo_wire` edit (the trim rides the shipped connector `command/3`; `@wire_version` frozen `echomq:2.4.2`) — adds **NO new script** (`XTRIM` is issued direct, not via Lua — `grep -c redis.call` on the `lib/` diff = 0; `append/4` + every shipped `Script.new/2` byte-identical), edits **NO shipped script** (the byte-freeze), adds **NO keyspace subkey** (D-3 — the policy is BEAM-side; `keyspace.ex` unedited, no `emq:{q}:stream:<name>:policy`), does **NOT touch the frozen emq3.3 reader** (D-2 — `git diff stream_consumer.ex` EMPTY), and adds **NO new wire class** (the closed registry `{EMQKIND, EMQSTALE}` stands — `XTRIM` is a stock verb).

**The gate is the blast-radius mutation battery + the POSITIVE INV4 conformance proof + the multi-seed sweep + Apollo mandatory** — NOT the ≥100 determinism loop (no process, no mint, no lease) and NOT a frozen-touch HIGH (no frozen line edited).
