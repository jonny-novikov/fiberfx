## The rung in one paragraph

emq3.6 builds **time-travel + hydration** — the tier's last two stated needs ([`../../emq.streams.md`](../../emq.streams.md) §The needs), both already cheap because the substrate is in place. TIME-TRAVEL (bus-side, `echo_mq`): an ADDITIVE read surface on `EchoMQ.Stream` that takes a `DateTime` window and returns the entries whose branded `EVT` mint-instant falls in it — *"entry ids are millisecond-prefixed, so branded mint instants map straight to `XRANGE` bounds"*. The `from` reuses the SHIPPED `minid_floor/1` (`stream.ex:239`, the half-open floor `trim/4` MINID already proves); the `to` is the NEW upper-bound inverse (the largest id at-or-before `t1` — the bound `trim/4` never needed); the read DELEGATES to the byte-frozen `read/6` (`stream.ex:270`); ZERO new Lua. The load-bearing property: a window read EQUALS reading the full stream and filtering by the id's mint instant (INV-TT). HYDRATION (store-side, `echo_store`): a NEW surface (Arm 3 — a one-shot `Table.hydrate_from_stream/_` or a supervised hydrator) that reads a stream tail (Arm 4 — `read/6` live or `StreamArchive.merge_read/5` deep) and folds it into an `EchoStore.Table` (`table.ex`) via `put/4` versioned by the branded `EVT` id, holding per key the value with the NEWEST mint id (newer-wins by mint order, `Coherence.newer?/2`) — *"changelog semantics without a compactor"*; then the `:tracking` staleness fence (`table.ex:256`) keeps it fresh — *hydrate-then-fence equals loader truth* (INV-HYDRATE + INV-FENCE). The net-new bus code is the additive time-travel read (NO Lua, NO wire edit); the net-new store code is the hydration (through the PUBLIC `EchoStore.Table` API — NO engine edit). All under the v2 master invariant (binding every surface UNCHANGED — braced keys, branded ids the writer minted, declared keys [vacuous — no new Lua], the server clock [vacuous — no lease], inline `Script.new/2` [none added], `@wire_version` frozen `echomq:2.4.2`).
