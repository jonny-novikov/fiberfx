# EchoMQ command registry ŌĆö the v1ŌåÆv3 capability surface, by feature

> **Dev-support reference, NOT the catalogue.** All **50** v1 BullMQ Lua commands mapped v1ŌåÆv3, sorted by the **12-feature taxonomy** ([`emq.epic.1.md`](../emq.epic.1.md) ┬¦The feature index). This is the feature-sorted, Ōēż300-line-per-file successor to the former 1290-line `emq.1.specs.md` matrix ŌĆö *why* each v3 form re-derives its v1 analogue, never the forward-only DSL catalogue itself.

**How to read.** This file is the **summary matrix** (one table per feature). A command's **full detail + the v1ŌåÆv3 side-by-side** lives exactly once in its feature slice under [`emq.commands/features/`](./emq.commands/features/), anchored `#{command}`; the Command cell links there. A command serving two features is filed under its **primary** feature and cross-referenced (`ŌåÆ feature`) from the secondary. The raw v1 sources are [`emq.commands/registry/*.lua`](./emq.commands/registry/); the flat agent index is [`emq.commands/llms.txt`](./emq.commands/llms.txt).

**Columns:** Command ┬Ę v1 (purpose) ┬Ę v2 status (grounded ŌĆö token ┬Ę rung `sha` ┬Ę module) ┬Ę v3 decision (PROPOSED) ┬Ę BCS ┬Ę EchoMesh. **Voice:** v2 is as-shipped present tense, traced to the rung that shipped it; every v3 claim is forward / PROPOSED; `[RECONCILE]` marks a claim ahead of as-built code. Status tokens: **PORTED** (shipped) ┬Ę **PARTIAL** (shipped in part) ┬Ę **NOT YET** (proposed) ┬Ę **RETIRED** (dropped by design) ┬Ę **[RECONCILE]** (building).

Per-rung commits: emq.0 `a2d599c8` ┬Ę emq.1 `e0fa9b03` ┬Ę emq.2.1 `7d98ef86` ┬Ę emq.2.2 `76fc947c` ┬Ę emq.2.3 `3c6461ff` ┬Ę emq.2.4 `3298e4bc` ┬Ę emq.3.4 `4c401479` ┬Ę emq.3.5 building.
