The bridge · four crossings

From algebra to LiveView

You don't start from zero. Functional programming is the algebra you've practiced for years — functions, substitution, composition — made executable. We build that bridge one crossing at a time.

01

Algebra

where you already are

A function maps inputs to outputs. You substitute, you compose f∘g, and a value never mutates — x stays x. That isn't only math: it's the entire mindset of functional programming.

02

Functional Programming

algebra, made executable

Pure functions, immutability, higher‑order functions, recursion and pattern matching, composition. The same rules you know from algebra — only now they run.

03

Elixir

a functional language for production

Pattern matching, the pipe |>, enumerables and recursion, processes, OTP and concurrency on the battle‑tested BEAM. Functional — and fast, and fault‑tolerant.

04

Phoenix LiveView

real‑time apps, functionally

Rich, interactive, server‑rendered UIs that update live over WebSockets — with little to no JavaScript. The same functional model, all the way to the browser.

Each crossing reuses the last. The substitution you do in algebra becomes a pure function; that function becomes an Elixir pipeline; that pipeline drives a live, interactive screen. Nothing is thrown away — your math intuition carries you to the end.

A taste of Elixir

Read it once — you already get it

Here's a pipeline. Data flows left to right through small pure functions. If you can read a composition of functions, you can read Elixir.

1..10
|> Enum.map(&(&1 * &1))
|> Enum.filter(&(rem(&1, 2) == 0))
|> Enum.sum()
# => 220

Square every number from 1 to 10, keep the even squares, add them up. No loops, no mutable state — just values flowing through functions. Try the same idea below.

n3

3 |> double |> add(1) |> square => 49

Move the slider: the value flows through each function and every stage updates at once. This is exactly how the course teaches — every concept comes with runnable code you can change and re‑run, not just read.

This course is for developers curious about functional programming, for students with a strong feel for math who want to write real code, and for anyone tired of fighting mutable state and tangled side effects.
