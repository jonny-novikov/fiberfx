# Вотчер консистентности курса «Право» — `watch_law.sh`

Локальный страж курса `/law`: держит его согласованным и **проактивно** ловит проблемы,
пока ты правишь страницы. Аналог elixir-вотчера (`docs/elixir/references/watch_refs.sh`),
но для law и без зависимости от манифеста/cms.

## Что он делает

Опрашивает (polling, без fswatch) **живое дерево `law/`** и `docs/law/law-status.md`.
На каждое изменение (новый/изменённый `*.html` или правка статуса):

1. **Полный аудит консистентности** (`audit_law.py`) — проактивный слой:
   честность/дисклеймеры, кросс-курсовые протечки (`/logic/`, `lg-c…`, крошки «Логика»),
   тема главы (`--accent` = свой hex), целостность внутренних ссылок, **дрифт `status ↔ fs`**.
2. **Гейты затронутых страниц** (`checks.sh <page>`): `preflight.py` (KaTeX/структура)
   → `verify_numbers.py` (арифметика) → постраничный `audit_law.py` → `suite.law.js`
   (headless DOM, **0 скриншотов**).
3. *(только при `LAW_AI=1`)* на странице с красными гейтами — **scoped `claude -p`**,
   который чинит ТОЛЬКО эту страницу и гоняет `checks.sh` до зелёного (доверяем
   артефакту, не коду выхода; почасовой троттл; очередь на переполнении).

**Сепарация от elixir-вотчера:** тот сторожит `elixir/`, этот — `law/`. Пересечение лишь
в коммитах в `master` — **этот вотчер ничего не коммитит** (только проверяет/логирует).

## Команды

```bash
W=docs/law/toolkit/watch_law.sh
bash $W audit       # разовый ПОЛНЫЙ аудит консистентности (печатает находки)
bash $W once        # один цикл опроса (детект изменений + гейты затронутого)
bash $W baseline    # запомнить текущее состояние (ничего не считать изменённым)
LAW_AI=0 INTERVAL=30 bash $W start   # запустить демон (детерминированный режим)
bash $W status      # RUNNING/stopped + счётчики + итог последнего аудита
bash $W stop        # остановить
bash $W queue       # страницы, ждущие починки
```

**Env:** `INTERVAL=20` · `LAW_AI=0|1` · `MAX_AI_PER_HOUR=8` · `AI_TIMEOUT=1200`.

> **AI-режим оператор включает САМ.** `LAW_AI=1` спавнит агентов `claude -p` — это
> запускает только человек: `LAW_AI=1 bash $W start`. В режиме `LAW_AI=0` вотчер
> агентов не запускает (только python/node-проверки) — безопасно оставлять висеть.

Состояние и лог — в `docs/law/toolkit/.watch/` (`watch.log`, `findings.txt`, `state`,
`queue`, `pid`). Каталог `.watch/` и `.gen/` — рантайм-артефакты (стоит .gitignore'ить).

## Разовый прогон без демона

```bash
docs/law/toolkit/checks.sh                       # все гейты по всему law/
docs/law/toolkit/checks.sh law/trud/index.html   # только указанные страницы
```

Резолвится от расположения скрипта — работает из любой CWD.
