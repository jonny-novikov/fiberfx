ЖЁСТКИЕ ПРАВИЛА (A+ Apollo gates)
- Контент на русском, лаконично; документы — Writerside-friendly (md, KaTeX-чистые) для публикации.
- KaTeX strict: НИКОГДА не помещай  № ₽ « » “ ” „ … • — – · × ÷ ≤ ≥ ≠  и любые
  нелатинские/типографские символы внутрь $...$ или $$...$$. Единицы (В, А, Ом, Вт,
  кВт·ч, Вт·ч) — простым HTML-текстом ВНЕ формул; в формулах — \cdot, \times, \frac.
  Перед выдачей — regex-скан \$[^$\n]+?\$ на запрещённые символы.
- 0 изображений (бюджет валидатора 0; только DOM/computedStyle).
- Все ссылки root-relative (/physics/...); веб-хост не хардкодить.
- Никаких вложенных <a> внутри <a> (карточка-ссылка = внешний <a> без внутренних <a>;
  если нужны внутренние ссылки — внешний контейнер делать <div>).
- Каждый модуль ОБЯЗАН заканчиваться квизом. Ключи localStorage: модуль
  phys-<глава>-<модуль>-quiz, квиз главы phys-<глава>-kviz, финал phys-final-test.
- Один осмысленный интерактив на модуль (калькулятор / переключатель-визуализация /
  чек-лист) строго по директивам §8; интерактивы анонимны (эфемерны или localStorage)
  и с дисклеймером.
- Дизайн ТОЛЬКО через style.py: head(..., slug=<ключ палитры>) + NAV_JS; цвет главы —
  ключ PALETTE (tok / moshchnost / schet / bezopasnost / ustroystva / final).
- Навигация: полоса прогресса + section-nav (scroll-spy), карточки пред./след.,
  «↑ Наверх» (<body id="top">), хлебные крошки.
- Безопасность: материал образовательный, на каждой странице safety-banner; в модулях
  о безопасности — ТОЛЬКО модель/объяснение, без призывов к опасным опытам с сетью.
