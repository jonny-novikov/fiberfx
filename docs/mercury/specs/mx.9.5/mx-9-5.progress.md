## {mx-9-5-learnings} Learnings

### L-1

L — CSS multi-background layer ORDER is a gate-invisible trap of the same class as the bare type-scale var: in `background: A, B, C` the FIRST layer paints on TOP, so an opaque `linear-gradient(rgb(var(--bg-primary)), …)` listed first (as the mx.9.5 donor app.css:181–186 literally has it) silently hides every layer after it — the crosshatch never renders, no tool flags it, and a brief that quotes the donor literal propagates the no-op. The cheap lint: any multi-layer background whose FIRST layer is a full-opacity fill is dead code below layer 1 — reorder (decorative transparent-gap layers first, the solid base LAST) and flag the donor deviation with its file:line rather than copying the broken literal. Same law as `--text-heading-300` needing the -size/-lh pair and the unspaced clamp(): valid-parsing CSS that renders the wrong thing is caught only by a targeted read-time check, never by typecheck/build.
