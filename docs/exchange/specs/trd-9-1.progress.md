## {trd-9-1-decisions} Decisions

### D-1 — R-1 settled: accept the codegen-derived namespace, alias it as Proto inside investex modules

protoc-gen-elixir derives the Elixir module namespace from the proto `package tinkoff.public.invest.api.contract.v1` [common.proto:3, users.proto:3, sandbox.proto:3] — it emits `Tinkoff.Public.Invest.Api.Contract.V1.{Quotation,MoneyValue,GetAccountsRequest,...}`, NOT the literal `Investex.Proto.*` the chapter §Surface/§money-vocabulary names. DECISION: accept the generated names; inside each investex module that references a generated struct, `alias Tinkoff.Public.Invest.Api.Contract.V1, as: Proto` so call sites read `%Proto.Quotation{}` exactly as the chapter spec's prose intends. No rename of the generated tree, no `gen_descriptors`/`package_prefix` fragility. The chapter spec's `Investex.Proto.*` is thus a SPEC-LITERAL realized as the `Proto` alias — recorded as a realization-over-literal (L-1), not a STALE. The slice spec pins this contract: Mars realizes the alias; the generated module names are the namespace of record. Grounding: proto package line; the elixir-protobuf codegen names modules by package, capitalizing each dotted segment.

### D-2 — R-2 settled: Investex.Money LANDS in 9.1 (foundational, pure, G2 in-scope)

The chapter §Surface lists `Investex.Money.{from_quotation/1, to_quotation/1, from_money_value/1}` [trd.9.specs.md §money-vocabulary] but the 9.1 decomposition bullet does not name it, and the 7 implemented RPCs' responses carry no Quotation/MoneyValue field (GetAccounts/GetUserTariff/GetInfo/the sandbox trio return accounts/tariffs/ids; only GetMarginAttributes — out of the live round-trip — carries MoneyValue). DECISION: land Investex.Money + its pure round-trip property (G2) in 9.1. Reasoning: (1) the generated Quotation/MoneyValue structs exist the moment the 9.1 codegen runs, so Money has its inputs; (2) it is pure and network-free — it fits Tier 1 and the --no-start gate with zero risk; (3) it is the integer-{units,nano} contract the WHOLE platform speaks (INV-3) and de-risks 9.2, where Instruments/MarketData responses are money-dense; (4) it makes the 9.1 vertical carry the money invariant from the first rung rather than introducing a foundational helper mid-ladder. Consequence: INV-3 and gate G2 are IN-SCOPE for 9.1 (scoped to the pure codec; no decoded-money assertion is made against the live UsersService responses, which carry none). The Go float bridge (converters.go ToFloat/FloatToQuotation) is NOT ported — that is the deliberate INV-3 divergence.

### D-3 — R-3 settled: G1 at 9.1 = a GROWING parity scaffold (assert the implemented set maps; carry the rest as an explicit pending list)

### L-2 — The Go SDK does NOT read x-ratelimit-reset for the RE wait; the Elixir decide/3 "honoring x-ratelimit-reset" is a deliberate refinement over the Go source

(2) BUT L-7's fix is INEFFECTIVE.
