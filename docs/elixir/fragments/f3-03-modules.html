Functions, modules & the pipe

Three ideas, closely related. A function is the unit of work; a module is a named group of functions; and the pipe threads a value through a series of them. Together they are how an Elixir program is organised.

This is where the portal grows its first real modules. The magic-link result you learned to match in F3.02 is now returned by a function in Portal.Auth; the learner it identifies lives in Portal.Accounts; their completions live in Portal.Progress. Below is the portal's module map — the surface the rest of the course builds against.

Three things, one idea

You define a function with def inside a defmodule block. Related functions share a module, and the module name is part of how you call them: Portal.Auth.verify(token). The pipe then lets one function's result flow into the next as its first argument, so a sequence of calls reads top to bottom rather than inside out.
