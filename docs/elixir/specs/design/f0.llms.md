## Requirements

- **F0-R1** — The `:root` palette matches the authoritative values: surfaces `--ink/--ink-2/--ink-3`, text
  `--cream/--cream-soft/--cream-dim`, accents `--gold/--blue/--sage/--burgundy/--elixir` (each with a `-bright`
  variant except `--burgundy`), `--line`, the four stacks `--serif-display/--serif/--sans/--mono`, and
  `--measure: 68ch`; inside any `<svg>` a presentation attribute uses the literal hex mapped by role. [US: F0-US1]
- **F0-R2** — The builder emits exactly `<!doctype html>\n<html lang="en">\n` + the filled head + `\n<body>\n` + the
  fragment + `\n` + the bootstrap + `\n</body>\n</html>\n`; the head carries the meta, the Google Fonts link, the
  HTML-escaped `{{TITLE}}`/`{{DESC}}`, and one inline `<style>` declaring the tokens once. [US: F0-US2]
- **F0-R3** — A body fragment is authored top to bottom as the anatomy: skip link; sticky `.site` header whose
  `.route-tag` mirrors the page's clean route; `main#main.wrap`; a `.hero` (`.crumbs`, `.eyebrow`, one `<h1>`,
  `.lede`, `.kicker`, `.toc-mini`); content `<section>`s (`.prose` ≤ `--measure`, `.deflist`, a `.fig` with `<svg>` /
  `pre.code` / `.geo-readout`, a `.bridge`, a `.take`); a synthesis section with a `.note`; a `.pager`; the
  `.site-foot` stamp. Every block container is balanced. [US: F0-US3]
- **F0-R4** — The builder appends exactly one progressive-enhancement bootstrap: it adds `html.js`, then on
  `DOMContentLoaded` reveals `.reveal` via `IntersectionObserver`, short-circuiting to "reveal all" under
  `prefers-reduced-motion: reduce` or when `IntersectionObserver` is absent; the fragment carries no reveal script of
  its own. [US: F0-US4]
- **F0-R5** — Every page passes all nine Apollo gates — `containers`, `svg`, `no-future`, `voice`, `storage`,
  `motion`, `degrade`, `links`, `pager` — in that order; A+ is all nine passing and any FAIL is a hard stop; the
  Python `check` and `cms check` agree gate-for-gate. [US: F0-US5, F0-US9]
- **F0-R6** — Content degrades without JavaScript (`.reveal` is JS-gated via `html.js .reveal`), reduced motion is
  honoured, and neither `localStorage` nor `sessionStorage` appears anywhere in the document. [US: F0-US6]
- **F0-R7** — Visible text and code comments carry none of `revolutionary`, `blazing-fast`, `magical`, `simply`,
  `just`, `obviously`, `effortless` (word-boundary, case-insensitive), no first-person narration, and no gendered or
  perceptual verb applied to a tool or an agent. [US: F0-US9]
- **F0-R8** — Each page carries exactly one 14-char branded stamp (`TSK` + base62 Snowflake,
  `timestamp(41) << 22 | node(10) << 12 | seq(12)`, epoch `1704067200000`); an on-page decoder expands it client-side
  to the same namespace/snowflake/node/seq/timestamp the tool decodes; the canonical vector `TSK0KHTOWnGLuC` decodes
  to snowflake `274557032793636864`, node `0`, seq `0`, `2026-01-27 15:11:37 UTC` both ways. [US: F0-US7]
- **F0-R9** — Assembly is a pure function of (fragment, head, title, desc, stamp): the same inputs yield byte-identical
  output, and the only per-build-varying region of the bytes is the stamp. [US: F0-US7, F0-US8]
- **F0-R10** — `jonnify-cms` decomposes every published page into a deduplicated head template, a stamp-re-placeholded
  body-fragment template, and per-page data, and recomposes it byte-identically; `cms build --verify` and the offline
  parity test report every page reproducing byte-for-byte, with the loaded count equal to the on-disk `*.html` count
  and an empty skip list. [US: F0-US8]

## Execution topology

**Runtime topology (the assemble pipeline, and the store→builder→gates flow).**

```text
ASSEMBLE one page  (pure: fragment + head + title + desc + stamp → bytes)
  fragment(body) ──┐
  head(template) ──┼─▶ fill {{TITLE}}/{{DESC}} (Esc) into head
  title, desc   ───┘        │
  Snowflake.Mint("TSK") ─▶ {{BUILD_ID}} + decode ─▶ {{BUILD_TS}}  (the only per-build-varying region)
                            ▼
   <!doctype html>\n<html lang="en">\n + head + \n<body>\n + fragment + \n + BOOTSTRAP + \n</body>\n</html>\n
                            ▼
   nine Apollo gates  →  A+ (all nine) | FAIL (hard stop)

jonnify-cms STORE → BUILDER → GATES  (reproduce the published corpus)
  /elixir/**/*.html ──load──▶ store: head template (dedup by sha256) + fragment template (stamp re-placeholded)
                                   + per-page {title, desc, build_id, build_ts}
        │
        ├─ build --route R ─▶ Assemble(head, page) ─▶ bytes
        └─ build --verify   ─▶ for every page: Assemble == published file  (byte-for-byte)
                                   └─▶ parity test: loaded count == on-disk count; skip list empty
```

The stamp is the single per-build-varying span; every other byte of the envelope is fixed by (fragment, head, title,
desc), so the store round-trips a published page by replaying its pinned stamp.

**Task topology (build order; each step leaves a checkable artifact).**

```text
T1 lay down the shared head: meta + Google Fonts link + {{TITLE}}/{{DESC}} + one <style> with the :root tokens   [R1, R2]
      └─▶ T2 author the body fragment top to bottom in the anatomy vocabulary (hero → sections → synthesis → pager → foot) [R3]
            └─▶ T3 place exactly one build-stamp footer with the {{BUILD_ID}}/{{BUILD_TS}} placeholders + on-page decoder [R8]
                  └─▶ T4 assemble: fill head, mint the stamp, compose the exact envelope, append the bootstrap          [R2, R4, R9]
                        └─▶ T5 run the nine Apollo gates; A+ or stop; confirm Python check and cms check agree          [R5, R6, R7]
                              └─▶ T6 load into the content store, build --verify, run the parity test (byte-for-byte)   [R9, R10]
```

## Agent stories

Each agent story implements one user story and closes on a gate. Build them in task-topology order.

### F0-AS1 — Declare the palette once in the shared head [implements F0-US1]
Directive: lay down the shared head (T1) with the meta, the Google Fonts link, the `{{TITLE}}`/`{{DESC}}` placeholders,
and one inline `<style>` whose `:root` block carries the authoritative tokens — surfaces, text, the five accents with
their `-bright` variants (no `-bright` for `--burgundy`), `--line`, the four font stacks, and `--measure: 68ch` —
copied from `design-tokens.md`. Do not redeclare the palette per page; inside any `<svg>` use the literal hex by role.
Acceptance gate: the `:root` block equals the authoritative values; every SVG presentation attribute is a role-correct
literal hex; the `containers` and `svg` gates pass.

### F0-AS2 — Build the page from the anatomy and wrap it in the exact envelope [implements F0-US2, F0-US3]
Directive: author the body fragment top to bottom in the anatomy vocabulary (T2) — skip link, `.site` header with a
`.route-tag` mirroring the clean route, `main#main.wrap`, a `.hero`, content `<section>`s (`.prose`, `.deflist`,
`.fig`, `.bridge`, `.take`), a synthesis section with a `.note`, and a `.pager` — then assemble (T4) so the builder
fills the head's `{{TITLE}}`/`{{DESC}}` and emits exactly `<!doctype html>\n<html lang="en">\n` + head + `\n<body>\n` +
fragment + `\n` + bootstrap + `\n</body>\n</html>\n`. The fragment carries no `<head>` and no bootstrap.
Acceptance gate: the emitted bytes equal the exact envelope; every block container is balanced (`containers` passes);
the `.pager` and every internal href resolve to a live/built route (`pager` and `links` pass).

### F0-AS3 — Append the progressive-enhancement bootstrap [implements F0-US4, F0-US6]
Directive: have the builder append exactly one bootstrap (T4) that adds `html.js` and, on `DOMContentLoaded`, reveals
`.reveal` via `IntersectionObserver`, short-circuiting to "reveal all" under `prefers-reduced-motion: reduce` or when
`IntersectionObserver` is absent. Author no reveal script in the fragment; use no web storage.
Acceptance gate: with JavaScript off all content is visible (`.reveal` is JS-gated); reduced motion is honoured; the
`degrade`, `motion`, and `storage` gates pass.

### F0-AS4 — Mint one decodable branded stamp [implements F0-US7]
Directive: place exactly one build-stamp footer (T3) with the `{{BUILD_ID}}`/`{{BUILD_TS}}` placeholders and the
on-page decoder; at assembly mint `snowflake.Mint("TSK")` into `{{BUILD_ID}}` and its decoded UTC timestamp into
`{{BUILD_TS}}`. The page decoder, the Python source, and the tool are three encoders of one scheme using the same
epoch (`1704067200000`), base62 alphabet, shifts/masks, and UTC format.
Acceptance gate: the page carries exactly one 14-char stamp; the on-page decoder and `cms stamp decode` yield the same
fields; the canonical vector `TSK0KHTOWnGLuC` round-trips to snowflake `274557032793636864`, node `0`, seq `0`,
`2026-01-27 15:11:37 UTC`.

### F0-AS5 — Hold the voice and the A+ bar [implements F0-US9, F0-US5]
Directive: keep visible text and code comments free of the forbidden hype/dismissive words, of first-person narration,
and of gendered or perceptual verbs applied to tools; then run the nine Apollo gates (T5) and confirm the Python
`check` and `cms check` agree gate-for-gate.
Acceptance gate: the `voice` gate passes; all nine gates pass (grade A+); both checkers agree; a deliberately
introduced single-gate violation flips the grade off A+ and is then reverted.

### F0-AS6 — Reproduce the page byte-for-byte through the store [implements F0-US8]
Directive: load the page (and the published corpus) into the content store (T6), decomposing into a deduplicated head
template, a stamp-re-placeholded body-fragment template, and per-page data; then `cms build --verify` and the offline
parity test reassemble every page and compare to the published file.
Acceptance gate: `cms build --verify` reports every page reproducing byte-for-byte; the parity test asserts the loaded
count equals the on-disk `*.html` count, the skip list is empty, and each page round-trips byte-identically.

## Execution plan — first two stories

A worked trace of how the system turns a user story into a verified page, to confirm the path leaves no decision to the
agent that the spec has not already fixed. Each plan is *story → agent story → tasks → command → gate*; an agent runs
the steps top to bottom and stops at the gate.

### Plan A — F0-US1 (author against the fixed palette) · via F0-AS1
1. Tasks T1 (lay down the shared head with the `:root` tokens).
2. Source: copy the `:root` block and the SVG-by-role hex map from `design-tokens.md`; do not re-derive values.
3. Command: assemble the page, then `cms check <out.html>`.
4. Gate (closes US1): the `:root` block equals the authoritative values; every SVG attribute is a role-correct literal
   hex; `containers` and `svg` pass. Requirements satisfied: F0-R1, and the head half of F0-R2.

### Plan B — F0-US7 (one decodable stamp) · via F0-AS4
1. Tasks T3 → T4 (place the stamp footer + decoder, then mint at assembly).
2. Source: the stamp format, constants, and canonical vector from `06-snowflake-stamp.md`; the footer HTML from
   `page-anatomy.md`.
3. Command: assemble (mint `{{BUILD_ID}}`/`{{BUILD_TS}}`), then `cms stamp decode TSK0KHTOWnGLuC` and compare to the
   on-page decoder.
4. Gate (closes US7): exactly one 14-char stamp; the page decoder and `cms stamp decode` agree; the canonical vector
   round-trips. Requirements satisfied: F0-R8, and the stamp half of F0-R9.

Efficiency check: each story is one head/fragment region behind one assemble command and one gate, and every branch
(reduced motion, no-JS, empty state) is already specified — so the agent builds rather than decides.

```text
You are building and validating a course page to the F0 design-system bar. Read specs/design/f0.md,
specs/design/f0.stories.md, the skill references design-tokens.md and page-anatomy.md, and the jonnify-cms specs
05-build-validate.md, 06-snowflake-stamp.md, and 07-content-store.md first. You write only a body fragment; the builder
supplies the head, the envelope, and the bootstrap. Hold the voice: no hype/dismissive words, no first-person

Build in this order, keeping a checkable artifact after each step:

1) Head (T1 -> R1, R2). Use the shared head: meta charset + viewport, the Google Fonts link (Cormorant Garamond, PT
   Serif, Manrope, JetBrains Mono), the {{TITLE}}/{{DESC}} placeholders, and one inline <style> whose :root block
   carries the authoritative tokens copied from design-tokens.md — surfaces --ink/--ink-2/--ink-3, text
   --cream/--cream-soft/--cream-dim, accents --gold/--blue/--sage/--burgundy/--elixir (each with a -bright variant
   except --burgundy), --line, the four stacks --serif-display/--serif/--sans/--mono, and --measure: 68ch. Declare the
   palette once; never redeclare it per page.

2) Fragment (T2 -> R3). Author the body fragment top to bottom in the anatomy vocabulary: a skip link; a sticky .site
   header whose .route-tag mirrors this page's clean route; main#main.wrap (the single 1080px column); a .hero
   (.crumbs, .eyebrow, one <h1>, .lede, .kicker, .toc-mini); content <section>s, each with an <h2>, a .prose block
   capped at var(--measure), a .deflist, a .fig figure carrying its <svg> / pre.code / .geo-readout (inside the SVG use
   literal hex by role, never a CSS variable), a .bridge pairing the idea with its Elixir form, and a closing .take; a
   synthesis section with a .note pointing at the next module; and a .pager whose hrefs point at real built routes. Keep
   every .fig, .bridge, and block <div> balanced.

3) Stamp footer (T3 -> R8). Add the .site-foot with exactly one build-stamp widget carrying the {{BUILD_ID}} and
   {{BUILD_TS}} placeholders and the on-page decoder (base62 alphabet 0-9 A-Z a-z, epoch 1704067200000, layout
   timestamp(41) << 22 | node(10) << 12 | seq(12)). Exactly one stamp per page.

4) Assemble (T4 -> R2, R4, R9). Fill the head's {{TITLE}}/{{DESC}} (HTML-escaped), mint snowflake.Mint("TSK") into
   {{BUILD_ID}} and its decoded UTC timestamp into {{BUILD_TS}}, then compose the document as exactly
   <!doctype html>\n<html lang="en">\n + head + \n<body>\n + fragment + \n + BOOTSTRAP + \n</body>\n</html>\n. The
   appended BOOTSTRAP adds html.js, then on DOMContentLoaded reveals .reveal via IntersectionObserver, short-circuiting

5) Gate (T5 -> R5, R6, R7). Run the nine Apollo gates in order — containers, svg, no-future, voice, storage, motion,
   degrade, links, pager — via `cms check <out.html>` (and the Python check). A+ requires all nine; any FAIL is a hard
   stop. Confirm: content is visible with JavaScript off (.reveal is JS-gated), reduced motion is honoured, no web
   storage is used, no /future link exists, and the two checkers agree gate-for-gate. The on-page stamp decoder and
   `cms stamp decode` must yield the same fields; the canonical vector TSK0KHTOWnGLuC must round-trip to snowflake
   274557032793636864, node 0, seq 0, 2026-01-27 15:11:37 UTC.

6) Byte-parity (T6 -> R9, R10). Load the page (and the published /elixir tree) into the content store, then run
   `cms build --verify` and the offline parity test. Confirm every stored page reassembles equal to its published file
   byte-for-byte, the loaded count equals the on-disk *.html count, and the skip list is empty.

Deliverables to produce: a page that is the exact F0 envelope, A+ across the nine gates, carrying one decodable stamp,
degrading without JS, and reproducing byte-for-byte through jonnify-cms. Stop when the Definition of Done in
specs/design/f0.md is fully checked.
```
