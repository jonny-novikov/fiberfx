## Goal

After F5.1, `mix run` boots a supervised application that serves HTTP on port 4000. A request travels through a thin
`Plug.Router` to a stubbed `Portal.Engine` boundary (`dispatch/1`, `query/2`) and back. `Portal.ID` mints and decodes
branded Snowflake ids. The web layer holds no domain logic, so F6 can swap `Bandit` for Phoenix without touching
anything below it.

## Deliverables

  `jason ~> 1.4` in `mix.exs`; no Phoenix, no Ecto; `mix deps.get` and `mix compile` clean.
- **F5.1-D2** â€” `Portal.ID` (lib/portal/id.ex), a thin wrapper over the `echo_data` umbrella app's `EchoData.Snowflake`
  + `EchoData.Base62`: a snowflake is a 64-bit integer `timestamp(42 bits, ms since epoch
  1704067200000) <<22 | node(10 bits) <<12 | seq(12 bits)`; a branded id is a 3-letter uppercase namespace + the
  width-11 Base62 (0-9A-Za-z) encoding of the snowflake (14 chars, e.g. `ENR0NepbCmCG3d`); public `new/1`,
  `snowflake/1`, `namespace/1`, `at/1`, and `valid?/1` (the full 14-char format check â€” namespace + 11-char Base62 â€” the
  guard F5.4 builds its precondition on); pure except the clock and a monotonic per-ms sequence.

  > **Build note (F5.4 Â· latent id-uniqueness defect).** `Portal.ID.new/1` mints with a hardcoded `worker_id: 1`, and
  > `EchoData.Snowflake` holds its 12-bit sequence in the process dictionary (per-process), so two processes minting in
  > the same millisecond produce identical branded ids. This is safe in production because all mints serialize through
  > the single-writer `Portal.Engine`; it is a hazard only for multi-process tests (mitigated per-suite by
  > `Portal.Store.reset/0`). A later rung should make `worker_id` per-process / scheduler-derived, or move the sequence
  > out of the process dictionary.
- **F5.1-D3** â€” `Portal.Engine` (lib/portal/engine.ex): a GenServer named `__MODULE__` exposing `dispatch/1` and

- **F5.1-D4** â€” `Portal.Web.Router` (Plug.Router): `plug :match`; `Plug.Parsers` (`:urlencoded`, `:json` via `Jason`);

- **F5.1-D5** â€” `Portal.Application` supervises `[Portal.Engine, {Bandit, plug: Portal.Web.Router, port: 4000}]` with

- **F5.1-D6** â€” the web layer is a thin, replaceable adapter: the router only parses a request, calls the engine
