> swapped), the append-before-evolve command path, the `Portal` facade, and a compile-only `EnrollmentLive` sketch.

## Requirements

Each Requirement names whether the file is **created** (only `enrollment_live.ex`) or **changed/confirmed** (the rest
already exist at their F5.8 shape), so the implementer assembles rather than re-derives.

- **F5.9-R1** (change `lib/portal/application.ex`) — the assembled tree keeps the as-built four children
  (application.ex:17–22) and swaps only `Portal.EventLog → Portal.EventStore.adapter()`:
  `[Portal.Store, Portal.EventStore.adapter(), Portal.Engine, {Bandit, plug: Portal.Web.Router, port: port()}]`,
  `:one_for_one`. `Portal.Store` stays (the dual-write projection reads/writes it — `reproject_enrollments/1`,
  engine.ex:161–176); Bandit stays (the HTTP front door, replaced by `PortalWeb.Endpoint` only at F6.1). The adapter
  is started before the engine, whose `init/1` reads its stream via `read_stream/1` and folds with `replay/1`. [US:
  F5.9-US1]
- **F5.9-R2** (confirm `lib/portal/engine.ex`) — the command path runs the F5.4 contract guard, then `decide →
  append/2 (port) → evolve`, appending **before** folding. This is the F5.8 inversion of the as-built `decide → evolve
  → EventLog.append` (engine.ex:75–77), holding because `append/2` is fallible: a failed append aborts and leaves
  state unchanged. [US: F5.9-US2]
- **F5.9-R3** (confirm `lib/portal.ex`) — `Portal` (`enroll/2`, `deliver_lesson/2`, `progress_of/1`) delegates to the
  engine wrappers `command/1`/`query/1` (the F5.8 rename of `dispatch/1`/`query/2`), the only `GenServer.call`
  callers. [US: F5.9-US3]
- **F5.9-R4** (confirm `lib/portal/error.ex`) — `Portal.Error` has the closed four-code union; `from/1` has no
  catch-all. Two codes have producers today (`:course_not_found`, `:already_enrolled`); `:lesson_locked` and
  `:invalid_progress` remain reserved. [US: F5.9-US4]
- **F5.9-R5** (create `lib/portal_web/live/enrollment_live.ex`) — a `PortalWeb.EnrollmentLive` sketch:
  `mount/3` queries, `handle_event/3` commands and branches on `%Portal.Error{}`, `render/1` draws from `assigns`; it
  names only `Portal`. The sketch is **compile-only** — there is no endpoint or socket until F6.1, so it cannot mount
  or serve; the build target is that it compiles, not that it renders. `mount/3` reads `Portal.progress_of/1`, which
  returns structurally-`0` progress for **two independent reasons**: (1) no progress-advancing event exists — the core
  has no `progress` field and `LessonDelivered` folds into a `delivered` MapSet (core.ex:64–68), never touching
  enrollment progress; and (2) the Store projection hardcodes `progress: 0` at every mint (engine.ex:130) and reproject
  (engine.ex:171). So it is an honest read of `0`, not a calculation, and moving it off `0` would need both a
  progress-advancing command **and** a projection change (an F6 concern). The sketch assigns and renders that `0`. [US:
  F5.9-US3, F5.9-US4]
- **F5.9-R6** (module docs) — the F6 handoff adds only `PortalWeb.Endpoint`; everything below the boundary is
  unchanged. The concrete consumer is F6.1's `PortalWeb.CourseController.index/2`, which calls only
  `Portal.courses_of/1`; F6.1's tree adds `PortalWeb.Endpoint` and leaves the store and engine unmoved. [US:
  F5.9-US5]
- **F5.9-R7** — `mix run` boots, an enroll persists through the port, an **engine-process** restart replays to the
  same state (a full-VM restart empties the in-memory adapter — durability across that is F6.3/Postgres), and the
  LiveView sketch compiles. [US: F5.9-US1, F5.9-US2]

## Execution topology

```text
The assembled tree and the UI path:

   Portal.Application (Supervisor, :one_for_one)   # the as-built 4 children; only EventLog → adapter() swapped
     ├─ Portal.Store                       # RETAINED — the dual-write %Enrollment{} projection reads/writes it
     ├─ Portal.EventStore.adapter()        # was Portal.EventLog; started before the engine, read on boot
     ├─ Portal.Engine                      # init/1: read_stream -> replay -> {:ok, state}
     └─ {Bandit, plug: Portal.Web.Router, port: port()}   # RETAINED — HTTP front door; → PortalWeb.Endpoint at F6.1
            │
            ▼
   PortalWeb.EnrollmentLive (sketch, compile-only)   # no endpoint/socket until F6.1 — compiles, does not run
     ├─ mount/3        ─▶ Portal.progress_of/1 ─▶ assign :progress (structurally 0) | :error
     ├─ handle_event/3 ─▶ Portal.deliver_lesson/2 ─▶ :ok (re-query) | {:error, %Portal.Error{}} (assign :error)
     └─ render/1       ─▶ draws from @progress, @error          # touches only Portal

   command path:  contract guard ─▶ decide ─▶ append/2 (port) ─▶ evolve
                  # append BEFORE evolve — the F5.8 inversion of decide→evolve→EventLog.append,
                  # because append/2 is fallible: a failed append aborts, leaving state unchanged.
```

```text
Task DAG:
  T1 tree(store, adapter, engine, bandit) ─▶ T2 command append-before-evolve ─▶ T3 facade ─▶ T4 error+from/1 ─▶ T5 LiveView sketch (compile-only) ─▶ T6 handoff note ─▶ T7 verify(mix run, persist, engine-restart-replay, sketch compiles)
```
