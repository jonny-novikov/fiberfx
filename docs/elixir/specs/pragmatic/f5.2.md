# F5.2 ┬Ę Model the Portal domain

> Give the Portal a shape before any behaviour. Model the domain as plain structs grouped into the **Accounts**,
> **Catalog**, and **Learning** bounded contexts, each with a small public API and keyed by branded Snowflake ids ŌĆö
> entities whose enforced keys make incomplete states impossible to build, contexts that reference each other only by
> id, and a public surface the engine will call.

## Goal

After F5.2, the Portal's domain is modeled and callable. Seven entity structs (`User`, `Session`, `Course`, `Lesson`,
`Page`, `Enrollment`, `Progress`) carry `@enforce_keys` and a `@type t`; three context modules own them and expose a
small public API over the F4 `Portal.Store`; `Learning.enroll/2` mints an `ENR` id and stores a fresh enrollment.
Contexts reference one another only by branded id, and the web layer is unchanged.

## Rationale (5W)

- **Why** ŌĆö Before behaviour, the domain needs a clear, explicit shape. Plain structs make the model legible, let the
  compiler enforce required fields, and bounded contexts keep coupling at the boundary instead of spreading through
  the system.
- **What** ŌĆö seven entity structs, three context modules owning them, and a small typed public API over the F4 store.
- **Who** ŌĆö Developers (a legible model and APIs to call), Architects (bounded contexts and clear ownership), Learners
  (the courses and enrollments that will hold their learning, modeled here).
- **When** ŌĆö the second rung; depends on F5.1 (`Portal.ID`) and F4 (`Portal.Store`). It is the model that F5.3 wires
  end to end and F5.4 then hardens with a contract.
- **Where** ŌĆö `lib/portal/<context>/<entity>.ex` (structs) and `lib/portal/<context>.ex` (contexts). No web changes;
  no validation yet ŌĆö that is F5.4.

## Scope

- **In** ŌĆö the entity structs (`@enforce_keys`, `defstruct`, `@type t`); the three context modules owning their
  entities; the public APIs (`Accounts.user/1`, `Catalog.course/1`, `Catalog.lesson/1`, `Learning.enroll/2`,
  `Learning.courses_of/1`); reads delegating to `Portal.Store`; `Learning.enroll/2` minting an `ENR` id, building
  `%Enrollment{progress: 0}`, and storing it; `@spec` on every public function.
- **Out** ŌĆö input validation and the design-by-contract guard (ŌåÆ F5.4); commands/queries/events (ŌåÆ F5.5); the
  supervised state home (ŌåÆ F5.6); the `EventStore` port, the `Portal` facade, and the `%Portal.Error{}` contract
  (ŌåÆ F5.8); wiring `enroll` through the web layer end to end (ŌåÆ F5.3).

## Deliverables

- **F5.2-D1** ŌĆö seven entity struct modules, each with `@enforce_keys`, `defstruct`, and a `@type t`:
  `Portal.Accounts.User` (USR: `id`, `email`, `name`), `Portal.Accounts.Session` (SES: `id`, `user_id`, `token`),
  `Portal.Catalog.Course` (CRS: `id`, `title`, `slug`), `Portal.Catalog.Lesson` (LSN: `id`, `course_id`, `title`),
  `Portal.Catalog.Page` (PGE: `id`, `lesson_id`, `body`), `Portal.Learning.Enrollment` (ENR: `id`, `user_id`,
  `course_id`, `progress` default `0`, type `0..100`), `Portal.Learning.Progress` (PRG: `id`, `enrollment_id`,
  `lesson_id`, `percent`). All `id`/`*_id` fields are branded-id strings; `@enforce_keys` lists every field except
  `progress`.
- **F5.2-D2** ŌĆö three context modules (`Portal.Accounts`, `Portal.Catalog`, `Portal.Learning`), each the only module
  that builds or persists its entities, referencing other contexts' entities **by branded id only** (never embedding
  or aliasing another context's struct).
- **F5.2-D3** ŌĆö the public APIs, each with an `@spec`, everything else private (`defp`): `Accounts.user(user_id) ::
  {:ok, %User{}} | :error`; `Catalog.course/1` and `Catalog.lesson/1` likewise; `Learning.enroll(user_id, course_id)
  :: {:ok, %Enrollment{}} | {:error, atom}`; `Learning.courses_of(user_id) :: [%Enrollment{}]`.
- **F5.2-D4** ŌĆö reads delegate to the F4 store (`Portal.Store.get/2`, `Portal.Store.all/2`); `Learning.enroll/2` mints
  an `ENR` id with `Portal.ID.new("ENR")`, builds `%Enrollment{progress: 0}`, stores it with `Portal.Store.put/1`, and
  returns `{:ok, enrollment}`.
- **F5.2-D5** ŌĆö typespecs: a `@type t` on each entity and an `@spec` on each public function; `id`/`*_id` fields typed
  as branded-id strings; the types match the struct definitions.
- **F5.2-D6** ŌĆö verification: building a struct with all enforced keys succeeds and omitting one raises; from `iex`,
  `Learning.enroll("USR1", "CRS1")` returns `{:ok, %Enrollment{progress: 0}}` and the enrollment is retrievable; no
  context references another context's struct.

## Invariants

- **F5.2-INV1 (make illegal states unrepresentable ┬Ę seed)** ŌĆö every entity is a struct with `@enforce_keys`, so a
  struct missing a required field cannot be constructed ŌĆö it raises at build time rather than flowing on as a partial
  value.
- **F5.2-INV2 (bounded-context ownership)** ŌĆö a context references another context's entities only by branded id; no
  context embeds, aliases, or constructs another context's struct.
- **F5.2-INV3 (narrow public surface)** ŌĆö each context exposes exactly its documented public API; all other functions
  are private.
- **F5.2-INV4 (identity)** ŌĆö ids are branded-id strings at the struct boundary (the transport form), the integer
  snowflake is canonical, and `enroll` mints via `Portal.ID.new("ENR")`.
- **F5.2-INV5 (typed model)** ŌĆö every entity has a `@type t` and every public function an `@spec` that matches the
  structs, so the model is legible and Dialyzer-checkable. (Input validation is deferred to F5.4 ŌĆö F5.2 models shape,
  not rules.)

## Definition of Done

- [ ] Every entity has a struct with `@enforce_keys`, `defstruct`, and a `@type t`.
- [ ] Building a struct with all enforced keys succeeds; omitting one raises (F5.2-INV1).
- [ ] Three context modules exist; none references another context's struct directly ŌĆö only branded ids (F5.2-INV2).
- [ ] Each context exposes exactly its documented public API; helpers are private (F5.2-INV3).
- [ ] Reads delegate to `Portal.Store`; `Learning.enroll("USR1", "CRS1")` returns `{:ok, %Enrollment{progress: 0}}`
      from `iex` and the enrollment is retrievable.
- [ ] Every public function has an `@spec` and every entity a `@type t` matching the struct (F5.2-INV5).
- [ ] Every deliverable maps to a user story and every invariant is exercised by a check (see `./f5.2.stories.md`).
