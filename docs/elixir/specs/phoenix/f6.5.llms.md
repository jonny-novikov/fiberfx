# F6.5 · agent brief (llms)

> Implementation brief for an agent building the HEEx views. References, traced requirements, the execution topology,
> and a paste-ready prompt. Pairs with the spec [`f6.5.md`](f6.5.md) and the stories [`f6.5.stories.md`](f6.5.stories.md).

> **Read the body's two decision sections first** ([`f6.5.md`](f6.5.md) "Routing decision" + "Component-home
> decision"). They resolve the `/courses` collision (catalog `resources "/courses"` vs. the learner's
> `get "/my/courses"`), repurpose `CourseController` to the catalog and MOVE the enrolled read to
> `EnrollmentController.index`, add the `:show` route + template, and place a locally-defined `input/1` in
> `catalog_components.ex` (this umbrella has **no `CoreComponents`**). Build against those, not the pre-reconcile shape.

## References

- HEEx and `Phoenix.Component` — <https://hexdocs.pm/phoenix_live_view/Phoenix.Component.html>. Function components,
  `attr`, `slot`, `render_slot/1`, and the `~H` sigil.
- Phoenix templates / HEEx engine — <https://hexdocs.pm/phoenix/components.html>. `:for`, `:if`, escaped
  interpolation, and the rendering model.
- `Phoenix.Component.to_form/2` and `Phoenix.HTML.Form` — <https://hexdocs.pm/phoenix_live_view/Phoenix.Component.html#to_form/2>.
  Building `@form` from a changeset and rendering `<.input>`.
- Verified Routes (`~p`) — <https://hexdocs.pm/phoenix/Phoenix.VerifiedRoutes.html>. Compile-checked paths.
- Upstream: the `changeset/2` parse boundary and errors [`f6.3.md`](f6.3.md), the `Catalog` facade [`f6.4.md`](f6.4.md),
  and routing/`~p` [`f6.2.md`](f6.2.md).

## Requirements

- **F6.5-R0** — `router.ex`: catalog `resources "/courses", CourseController, only: [:index, :show, :new, :create]`
  (public `:browser` scope); enrolled read `get "/my/courses", EnrollmentController, :index` (scope
  `pipe_through [:browser, :require_auth]`); `get "/courses/:user_id"` and `scope "/learn"` removed;
  `EnrollmentController.create` redirect → `~p"/courses/#{course_id}"`. The enrolled markup + `:error` template +
  `render_outcome/2` MOVE `CourseController`/`CourseHTML` → `EnrollmentController`/`EnrollmentHTML`. [US: F6.5-US0]
- **F6.5-R1** — catalog `index.html.heex` renders `@courses` (= `Portal.list_courses/0` → `[%Course{}]`) with `:for`,
  `<.link navigate={~p"/courses/#{course.id}"}>` (to `:show`), escaped `{course.title}`, an `:if={course.published}`
  badge, and an `:if={@courses == []}` empty state; no `Repo` or context call in the template. [US: F6.5-US1]
- **F6.5-R2** — `course_card` (in `catalog_components.ex`) declares `attr :course` (required) and optional `:class` and
  renders an `<article>`; called with `:for`. [US: F6.5-US2]
- **F6.5-R3** — `panel` (in `catalog_components.ex`) uses `slot :inner_block` and `render_slot/1`. [US: F6.5-US3]
- **F6.5-R4** — the form is `to_form(Catalog.change_course(%Course{}))` rendered with `<.form>` and the
  **locally-defined** `<.input>` posting to `~p"/courses"`; backed by the new `Catalog.change_course/1` (delegating to
  the F6.3 `Course.changeset/2`) + a `Portal.change_course/1` facade delegate. [US: F6.5-US4]
- **F6.5-R5** — the create action redirects to `~p"/courses/#{course.id}"` on `{:ok, course}` and re-renders with
  `to_form(changeset)` on `{:error, %Ecto.Changeset{}}`, surfacing inline field errors. [US: F6.5-US5]
- **F6.5-R6** — the controller does all data access; templates render only from assigns. [US: F6.5-US6]
- **F6.5-R7** — the view adds no validation; parsing stays in `Catalog.create_course/1`'s changeset. [US: F6.5-US4]
- **F6.5-R8** — `show.html.heex` + `CourseController.show/2` (`course: Portal.get_course!(id)`) render one course, so
  the index link and the create redirect compile under Verified Routes. [US: F6.5-US1]
- **F6.5-R9** — `input/1` is defined locally in `catalog_components.ex` (text field + inline per-field changeset
  error); no `CoreComponents` and no unexported `Phoenix.Component` `input` is referenced. [US: F6.5-US4]

## Execution topology

```text
Final routes (router.ex):
   GET  /courses           CourseController :index    (:browser)        catalog list
   GET  /courses/new       CourseController :new      (:browser)        the create form
   GET  /courses/:id       CourseController :show     (:browser)        one catalog course
   POST /courses           CourseController :create   (:browser)        create
   GET  /my/courses        EnrollmentController :index ([:browser, :require_auth])  the learner's enrollments
   (removed: get "/courses/:user_id"; scope "/learn")
```

```text
The catalog render path (controller fetches, template renders):

   CourseController.index/2 ─▶ Portal.list_courses/0 (→ [%Course{}]) ─▶ assign :courses
                                                          │
                                                          ▼
   course_html/index.html.heex  ──renders──▶  <.course_card :for={c <- @courses} course={c} />
                                  (:for · ~p"/courses/#{c.id}" link to :show · {c.title} · :if badge · :if empty)
                                  optionally wrapped by  <.panel> … render_slot … </.panel>

   CourseController.show/2  ─▶ Portal.get_course!(id) (raises → 404) ─▶ assign :course ─▶ course_html/show.html.heex

The create round-trip:

   new.html.heex (<.form for={@form} action={~p"/courses"}> <.input field={@form[:title]} label="Title" />)
        │  submit params
        ▼
   CourseController.create/2 ─▶ Portal.create_course/1
        ├─ {:ok, course}             ─▶ redirect to ~p"/courses/#{course.id}"   # the catalog :show
        └─ {:error, %Ecto.Changeset{} = cs} ─▶ render new.html.heex, assign to_form(cs)  # inline errors (F6.3)

The enrolled read (MOVED from CourseController):

   EnrollmentController.index/2 ─▶ conn.assigns.current_user_id ─▶ Portal.courses_of/1 (→ {:ok, [%Enrolled{}]})
                                ─▶ render_outcome/2 ─▶ enrollment_html/index.html.heex ({:error,_} → :error @ 422)
```

```text
Task DAG:
  T0 router reconcile + enrolled-read MOVE ─▶ T1 catalog index template ─▶ T2 course_card component ─▶ T3 panel slot
     ─▶ T4 change_course wrapper (+ facade delegate) ─▶ T4b input/1 in catalog_components ─▶ T5 form (new.html.heex)
     ─▶ T6 create action (ok→show / error→inline) ─▶ T6b show action + show.html.heex ─▶ T7 discipline check ─▶ T8 verify
```

Touched files (the exact final set):

  `course_html/new.html.heex`; `lib/portal_web/controllers/enrollment_html.ex` + `enrollment_html/index.html.heex` +

- REWRITTEN: `lib/portal_web/controllers/course_controller.ex` (→ the catalog resource: `index`/`show`/`new`/`create`;
  `health/2` stays); `course_html/index.html.heex` (→ the catalog list; the old enrolled markup MOVES out).
- EDITED: `lib/portal_web/router.ex` (the routes above); `lib/portal_web/controllers/enrollment_controller.ex` (add
  `index/2` + `render_outcome/2`; fix the `:26` redirect to `course_id`).
- DOMAIN: `apps/portal/lib/portal/catalog.ex` (add `change_course/1`); `apps/portal/lib/portal.ex` (add

- REMOVED: `course_html/error.html.heex` is relocated to `enrollment_html/error.html.heex` (it renders a

## Agent stories

- **F6.5-AS0** [implements F6.5-US0] — Directive: edit `router.ex` to the final routes (catalog `resources "/courses"`
  `only: [:index, :show, :new, :create]`; `get "/my/courses" → EnrollmentController.index` under
  `[:browser, :require_auth]`; remove `get "/courses/:user_id"` and `scope "/learn"`); MOVE the enrolled markup +
  `:error` template + `render_outcome/2` from `CourseController`/`CourseHTML` to `EnrollmentController`/`EnrollmentHTML`
  (new `enrollment_html.ex` + templates); fix the `enrollment_controller.ex:26` redirect to `~p"/courses/#{course_id}"`.
  Acceptance gate: `/my/courses` renders the enrolled read under auth; `/courses` resolves to the catalog; the moved
  F6.2 tests pass (the `/my/courses` rename + the redirect).
- **F6.5-AS1** [implements F6.5-US1, F6.5-US6] — Directive: write the catalog `course_html/index.html.heex` over
  `@courses` with `:for`, `<.link navigate={~p"/courses/#{course.id}"}>`, escaped `{course.title}`, an `:if` badge, and
  an empty state; assign `@courses` in `CourseController.index/2` via `Portal.list_courses/0`. Acceptance gate: renders
  with and without courses; the template makes no data access.
- **F6.5-AS2** [implements F6.5-US2, F6.5-US3] — Directive: write the `course_card` component (`attr :course` required,
  optional `:class`) and a `panel` slot wrapper in `catalog_components.ex`; call `course_card` from the index with
  `:for`. Acceptance gate: both render; a missing required attr warns; `render_slot/1` frames inner content.
- **F6.5-AS3** [implements F6.5-US4] — Directive: add `Catalog.change_course/1` (+ the `Portal.change_course/1` facade
  delegate) and a local `input/1` in `catalog_components.ex`; build `@form` with `to_form/1` in
  `CourseController.new/2`; write `new.html.heex` with `<.form>`/`<.input>` posting to `~p"/courses"`. Acceptance gate:
  the form renders, `<.input>` resolves to the local component, and a valid submit reaches `create_course/1`.
- **F6.5-AS4** [implements F6.5-US5] — Directive: write `create/2` to redirect to `~p"/courses/#{course.id}"` on
  `{:ok, course}` and re-render with `to_form(cs)` on `{:error, %Ecto.Changeset{}}`. Acceptance gate: a valid create
  redirects to `:show`; an invalid create re-renders with inline field errors and no view-level validation.
- **F6.5-AS5** [implements F6.5-US1] — Directive: write `CourseController.show/2` (`course: Portal.get_course!(id)`)
  and a minimal `course_html/show.html.heex`. Acceptance gate: `:show` renders a course; the index link and the create
  redirect compile under Verified Routes.

## Comprehensive implementation prompt

```text
You are building the catalog's server-rendered views (rung F6.5). Read specs/phoenix/f6.5.md INCLUDING its "Routing
decision" and "Component-home decision" sections, the F6.4 Catalog facade (list_courses/0, get_course!/1,
create_course/1, plus a change_course/1 wrapper you will add), and the F6.3 changeset/2 (the parse boundary whose
errors must surface inline). Keep ALL data access in the controller; templates render only from assigns. This umbrella
has NO CoreComponents — define input/1 yourself.

0) Router reconcile + enrolled-read MOVE (T0 -> R0). In router.ex, declare the catalog as
   `resources "/courses", CourseController, only: [:index, :show, :new, :create]` (public :browser scope) and the
   enrolled read as `get "/my/courses", EnrollmentController, :index` in a scope `pipe_through [:browser, :require_auth]`.
   Remove `get "/courses/:user_id"` and the whole `scope "/learn"`. MOVE the enrolled markup
   (course_html/index.html.heex), the :error template, and render_outcome/2 from CourseController/CourseHTML to a NEW
   EnrollmentHTML (enrollment_html.ex + enrollment_html/index.html.heex + enrollment_html/error.html.heex), and add
   EnrollmentController.index/2 (reads conn.assigns.current_user_id, calls Portal.courses_of/1). Fix
   enrollment_controller.ex:26 redirect to ~p"/courses/#{course_id}". Update the moved F6.2 tests.

1) Catalog index template (T1 -> R1). In course_controller.ex, index/2 assigns courses: Portal.list_courses/0. Create
   lib/portal_web/controllers/course_html/index.html.heex rendering @courses with:
     :for={course <- @courses}, <.link navigate={~p"/courses/#{course.id}"}> (to :show), escaped {course.title},
     :if={course.published} for a badge, and :if={@courses == []} for an empty state.
   No Repo and no context call may appear in the template.

2) course_card component (T2 -> R2). In lib/portal_web/components/catalog_components.ex, declare:
     attr :course, :map, required: true
     attr :class, :string, default: ""
   with a ~H body returning an <article> with {@course.title} and an :if badge. Call it from the index as
   <.course_card :for={course <- @courses} course={course} />.

3) panel slot (T3 -> R3). Add a `panel` component using `slot :inner_block` and `render_slot(@inner_block)` to frame
   caller content.

4) change_course wrapper + local input/1 (T4/T4b -> R7/R9). Add Portal.Catalog.change_course/1 returning an unsaved
   %Ecto.Changeset{} for a %Course{} (delegating to the F6.3 Course.changeset/2), and add the facade delegate
   `defdelegate change_course(course), to: Portal.Catalog` in portal.ex. Define a local input/1 in
   catalog_components.ex (a text <input> bound to @field.value/@field.name/@field.id + a per-field inline error from
   @field.errors). The view performs NO validation of its own.

5) Form (T5 -> R4). In course_controller.ex, new/2 assigns form: to_form(Catalog.change_course(%Course{})). Write
   new.html.heex:
     <.form for={@form} action={~p"/courses"}> <.input field={@form[:title]} label="Title" /> ... </.form>

6) Create action (T6 -> R5). Write create/2: send params to Catalog.create_course/1; on {:ok, course} redirect to
   ~p"/courses/#{course.id}" (the catalog :show); on {:error, %Ecto.Changeset{} = cs} re-render new.html.heex with
   to_form(cs) so the F6.3 changeset errors appear inline per field.

6b) Show action + template (T6b -> R8). Write show/2 assigning course: Portal.get_course!(id) (raises
   Ecto.NoResultsError → 404 on a miss) and a minimal course_html/show.html.heex (escaped {@course.title} + :if badge).

7) Discipline (T7 -> R6). Confirm the controller does all data access and the templates read only from assigns; no
   template names a Repo or context. Every link is a verified ~p route.

8) Verify (T8). The catalog index renders courses and an empty state with no template data access; the component and
   slot render; show renders a course; a valid create redirects to :show; an invalid create re-renders with inline
   field errors; /my/courses renders the enrolled read under :require_auth and the moved F6.2 tests pass.

Constraints: HEEx-escaped interpolation for user data (never raw); verified ~p routes everywhere; declared attrs on
components; input/1 is local (no CoreComponents, no unexported Phoenix.Component input). Out of scope: LiveView and
live validation (F6.6), PubSub (F6.7), styling systems beyond minimal classes, pagination/search, and a generated
CoreComponents.
```
