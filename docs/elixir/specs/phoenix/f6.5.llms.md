# F6.5 · agent brief (llms)

> Implementation brief for an agent building the HEEx views. References, traced requirements, the execution topology,
> and a paste-ready prompt. Pairs with the spec [`f6.5.md`](f6.5.md) and the stories [`f6.5.stories.md`](f6.5.stories.md).

## References

- HEEx and `Phoenix.Component` — <https://hexdocs.pm/phoenix_live_view/Phoenix.Component.html>. Function components,
  `attr`, `slot`, `render_slot/1`, and the `~H` sigil.
- Phoenix templates / HEEx engine — <https://hexdocs.pm/phoenix/components.html>. `:for`, `:if`, escaped
  interpolation, and the rendering model.
- `Phoenix.Component.to_form/2` and `Phoenix.HTML.Form` — <https://hexdocs.pm/phoenix_live_view/Phoenix.Component.html#to_form/2>.
  Building `@form` from a changeset and rendering `<.input>`.
- Verified Routes (`~p`) — <https://hexdocs.pm/phoenix/Phoenix.VerifiedRoutes.html>. Compile-checked paths.
- Upstream: the `changeset/2` parse boundary and errors [`f6.3.md`](f6.3.md), the `Catalog` facade [`f6.4.md`](f6.4.md),
  and routing/`~p` [`f6.2.md`](f6.2.md).

## Requirements

- **F6.5-R1** — `index.html.heex` renders `@courses` with `:for`, a `~p` link, escaped title, an `:if` badge, and an
  `:if` empty state; no `Repo` or context call in the template. [US: F6.5-US1]
- **F6.5-R2** — `course_card` declares `attr :course` (required) and optional `:class` and renders an `<article>`;
  called with `:for`. [US: F6.5-US2]
- **F6.5-R3** — `panel` uses `slot :inner_block` and `render_slot/1`. [US: F6.5-US3]
- **F6.5-R4** — the form is `to_form(Catalog.change_course(%Course{}))` rendered with `<.form>`/`<.input>` posting to
  `~p"/courses"`; backed by `change_course/1`. [US: F6.5-US4]
- **F6.5-R5** — the create action redirects on `{:ok, _}` and re-renders with `to_form(changeset)` on `{:error, _}`,
  surfacing inline field errors. [US: F6.5-US5]
- **F6.5-R6** — the controller does all data access; templates render only from assigns. [US: F6.5-US6]
- **F6.5-R7** — the view adds no validation; parsing stays in `Catalog.create_course/1`'s changeset. [US: F6.5-US4]

## Execution topology

```text
The render path (controller fetches, template renders):

   CourseController.index/2 ─▶ Portal.list_courses/0 ─▶ assign :courses
                                                          │
                                                          ▼
   index.html.heex  ──renders──▶  <.course_card :for={c <- @courses} course={c} />
                                  (:for · ~p link · {c.title} · :if badge · :if empty state)
                                  optionally wrapped by  <.panel> … render_slot … </.panel>

The create round-trip:

   new.html.heex (<.form for={@form}> <.input field={@form[:title]} />)
        │  submit params
        ▼
   CourseController.create/2 ─▶ Portal.create_course/1
        ├─ {:ok, course}             ─▶ redirect to ~p"/courses/#{course.id}"
        └─ {:error, %Ecto.Changeset{} = cs} ─▶ render new.html.heex, assign to_form(cs)  # inline errors (F6.3)
```

```text
Task DAG:
  T1 index template ─▶ T2 course_card component ─▶ T3 panel slot ─▶ T4 change_course wrapper ─▶ T5 form (new.html.heex) ─▶ T6 create action (ok/error) ─▶ T7 discipline check ─▶ T8 verify
```

## Agent stories

- **F6.5-AS1** [implements F6.5-US1, F6.5-US6] — Directive: write `index.html.heex` over `@courses` with `:for`, a `~p`
  link, escaped `{course.title}`, an `:if` badge, and an empty state; assign `@courses` in `index/2` via
  `Portal.list_courses/0`. Acceptance gate: renders with and without courses; the template makes no data access.
- **F6.5-AS2** [implements F6.5-US2, F6.5-US3] — Directive: write the `course_card` component (`attr :course` required,
  optional `:class`) and a `panel` slot wrapper; call `course_card` from the index with `:for`. Acceptance gate: both
  render; a missing required attr warns; `render_slot/1` frames inner content.
- **F6.5-AS3** [implements F6.5-US4] — Directive: add `Catalog.change_course/1`, build `@form` with `to_form/1`, and
  write `new.html.heex` with `<.form>`/`<.input>` posting to `~p"/courses"`. Acceptance gate: the form renders and a
  valid submit reaches `create_course/1`.
- **F6.5-AS4** [implements F6.5-US5] — Directive: write `create/2` to redirect on `{:ok, _}` and re-render with
  `to_form(cs)` on `{:error, _}`. Acceptance gate: a valid create redirects; an invalid create re-renders with inline
  field errors and no view-level validation.

## Execution plan — first two stories

1. **F6.5-AS1 — the index template.** In `course_controller.ex`, `index/2` assigns `courses: Portal.list_courses/0`.
   Create `course_html/index.html.heex` rendering `:for={course <- @courses}` with `<.link navigate={~p"/courses/#{course.id}"}>{course.title}</.link>`,
   an `:if={course.published}` badge, and an `:if={@courses == []}` empty state. Confirm no `Repo`/context call in the
   template.
2. **F6.5-AS2 — the component and slot.** In `catalog_components.ex`, declare `attr :course, :map, required: true` and
   `attr :class, :string, default: ""` with a `~H` `<article>` body, plus a `panel` using `slot :inner_block` and
   `render_slot(@inner_block)`. Replace the index's per-row markup with `<.course_card :for={course <- @courses}
   course={course} />`.

## Comprehensive implementation prompt

```text
You are building the catalog's server-rendered views (rung F6.5). Read specs/phoenix/f6.5.md, the F6.4 Catalog facade
(list_courses/0, create_course/1, plus a change_course/1 wrapper you will add), and the F6.3 changeset/2 (the parse
boundary whose errors must surface inline). Keep ALL data access in the controller; templates render only from assigns.

Build the view half:

1) Index template (T1 -> R1). In course_controller.ex, index/2 assigns courses: Portal.list_courses/0. Create
   lib/portal_web/controllers/course_html/index.html.heex rendering @courses with:
     :for={course <- @courses}, <.link navigate={~p"/courses/#{course.id}"}>, escaped {course.title},
     :if={course.published} for a badge, and :if={@courses == []} for an empty state.
   No Repo and no context call may appear in the template.

2) course_card component (T2 -> R2). In lib/portal_web/components/catalog_components.ex, declare:
     attr :course, :map, required: true
     attr :class, :string, default: ""
   with a ~H body returning an <article> with {@course.title} and an :if badge. Call it from the index as
   <.course_card :for={course <- @courses} course={course} />.

3) panel slot (T3 -> R3). Add a `panel` component using `slot :inner_block` and `render_slot(@inner_block)` to frame
   caller content.

4) change_course wrapper (T4 -> R7). Add Portal.Catalog.change_course/1 returning an unsaved %Ecto.Changeset{} for a
   %Course{} (delegating to the F6.3 changeset/2). The view performs NO validation of its own.

5) Form (T5 -> R4). Build @form with to_form(Catalog.change_course(%Course{})) and write new.html.heex:
     <.form for={@form} action={~p"/courses"}> <.input field={@form[:title]} label="Title" /> ... </.form>

6) Create action (T6 -> R5). Write create/2: send params to Catalog.create_course/1; on {:ok, course} redirect to
   ~p"/courses/#{course.id}" (or the index); on {:error, %Ecto.Changeset{} = cs} re-render new.html.heex with
   to_form(cs) so the F6.3 changeset errors appear inline per field.

7) Discipline (T7 -> R6). Confirm the controller does all data access and the templates read only from assigns; no
   template names a Repo or context. Every link is a verified ~p route.

8) Verify (T8). The index renders courses and an empty state with no template data access; the component and slot
   render; a valid create redirects; an invalid create re-renders with inline field errors.

Constraints: HEEx-escaped interpolation for user data (never raw); verified ~p routes everywhere; declared attrs on
components. Out of scope: LiveView and live validation (F6.6), PubSub (F6.7), styling systems, and pagination/search.
```
