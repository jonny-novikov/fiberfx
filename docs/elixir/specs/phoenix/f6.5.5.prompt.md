# F6.5.5 ship prompt — Director's orchestration brief

> Ship **F6.5.5 (Apply the design system)** by fanning out the specialized lead-team — `venus`
> (reconcile + brief) → `mars` ×2 (build, then harden) → `apollo` (verify + mentor) — Director-in-loop at
> each gate, one LAW-4 commit at close. The *discipline* lives in the agent definitions
> (`.claude/agents/{venus,mars,apollo}.md`); the per-stage prompts below carry only the rung delta. This is
> the F6.7 template (`f6.7.prompt.md`) retargeted to a **presentation** rung: the determinism loop drops out
> (no id / process surface), the **liveness gate gains a visual dimension**, and a new **visual-proxy gate**
> stands in for the screenshots a headless build cannot take. Operator's field manual for this rung:
> [`../design/f0.operator.md`](../design/f0.operator.md).

## The rung in one paragraph

F6.5.5 makes the Portal **render in the F0 design system** — the same dark-editorial tokens the static
`/elixir` and `/course/agile-agent-workflow` pages use — executing F0's planned milestones 4 (Portal HEEx
tokens/anatomy) and 5 (static-vs-Portal parity). The Portal renders **unstyled today**: `portal_web.ex:30`
sets `layouts: []`, so no root layout exists, no `<head>`, no CSS reaches a page. The slice closes that: a
**root layout** (`PortalWeb.Layouts` + `lib/portal_web/components/layouts/root.html.heex`, replacing
`layouts: []`) supplying the F0-D3 `<!doctype><html><head>…</head><body>` envelope; a committed
**`priv/static/assets/app.css`** carrying the F0-D1 `:root` tokens + base CSS + the four Google-Fonts
families (Cormorant Garamond / PT Serif / Manrope / JetBrains Mono), served by the **existing** `Plug.Static`
`assets` mount (no endpoint or config change, no bundler); and the LOCAL **`PortalWeb.CatalogComponents`**
(`course_card/1` · `panel/1` · `input/1`) restyled to the F0 anatomy (`course_card`→`.mod`, `panel`→`.fig`,
the published badge→`.pill.live`, buttons→`.btn`/`.btn.ghost`). Over the facade; no `CoreComponents`. Triad:
[`f6.5.5.md`](f6.5.5.md) · [`f6.5.5.stories.md`](f6.5.5.stories.md) · [`f6.5.5.llms.md`](f6.5.5.llms.md).

## Mode — Flat-L2 lead-team, review gates

Real `Agent` spawns + SendMessage, Director-in-loop — not a `Workflow` fan-out. Risk is **moderate**: no new
supervision children, no write path below the facade, no new id mint — but the change touches **every
rendered page** (a regression is global), and it carries the **clamp-spacing footgun** that shipped
invisibly across 204 static pages. So the lead-team with a careful Apollo visual-proxy gate, not a solo
build. All peers Opus (LAW-2); the Director coordinates + ratifies (LAW-1a); one commit at close (LAW-4).

**The determinism loop drops out.** F6.5.5 adds no id mint, no GenServer, no supervision child — it is
presentation only. The ≥100-iteration loop (the same-millisecond branded-id collision guard,
`echo/CLAUDE.md` §4) is **not a gate for this rung**; `TMPDIR=/tmp mix test` green is the suite gate. The
rung's real acceptance is the **visual-proxy gate + liveness** (below) — the styling exception, stated so the
build neither skips a gate it owes nor runs the determinism loop as theatre.

## What reconcile settles — and the one surface to pin

`f6.5.5.md`'s contract is reconciled against the as-built F6.1–F6.6 surface; Venus does not re-open these:

| Settled | State |
|---|---|
| Delivery mechanism | a committed `priv/static/assets/app.css` over the EXISTING `Plug.Static` mount — NO esbuild/tailwind. Aligned with F0's inline-style / no-build model; the umbrella has no asset pipeline by design. |
| Components | the LOCAL `PortalWeb.CatalogComponents` (`course_card`/`panel`/`input`) are RESTYLED, not replaced. NO `CoreComponents` (stays F6.8). |
| Tokens | copied byte-for-byte from F0-D1 / the static `_head.html` — F0-INV2 (the palette declared once in the shared head); the build invents no value. |

The one genuinely-new surface this rung adds above the facade — **Venus pins it, the Director ratifies**:

- **A root layout does not exist yet.** `portal_web.ex:30` sets `layouts: []`. F6.5.5 adds `PortalWeb.Layouts`
  + `components/layouts/root.html.heex` and wires it in (replacing `layouts: []`). This is the rung's single
  new web surface — **presentation only**: it names only facade-rendered assigns + the `~p"/assets/app.css"`
  static route, no engine / repo / `GenServer.call`. Unlike F6.7's `update_course/2` it is **not** a write
  below the facade; it is flagged for ratification because it changes the `layouts: []` contract
  `portal_web.ex` set at F6.1.

## Acceptance — what "shipped" means (from `f6.5.5.stories.md`)

Done iff every story's Given/When/Then is a passing test or its named visual proxy, the suite is green, **and
the liveness criterion holds with its visual dimension** — the Portal boots and serves the **styled** catalog
on `:4000` (a real `<head>` + the `app.css` link, not a bare `<h1>`), parity-checked against the static
baseline by `apps/e2e/tests/elixir-hero.spec.ts` (computed style, not pixels). The determinism loop is **not**
part of this rung's gate (no id / process surface). Acceptance here is *visual*, proven by text-checkable
proxies — that is the whole reason this presentation rung earns its own triad rather than a footnote in F6.5.

## The invariant it must not break

Facade-only (no engine / repo / `GenServer.call` in the layout or the components); HEEx-escaped
interpolation; the `~p`-verified `/assets/app.css` route; the **clamp-spacing footgun** (every `clamp()`
spaced — the single highest-leverage check, since it shipped invisibly across 204 static pages); tokens
copied byte-for-byte from F0-D1, never re-invented (F0-INV2: the palette declared once in the shared head).
The **one new surface**: a root layout + a stylesheet — named, pinned, ratified.

## Why the prompts are short (the evolved process)

The discipline lives in the agent definitions; the spawn carries only the rung delta. F6.5.5 retargets the
F6.7 template for a presentation rung with two changes: the **determinism loop drops out** (no id / process
surface — running it would be theatre), and a **visual-proxy gate** (clamp-spacing parse, token-fidelity
diff, computed-style parity) stands in for the screenshots a headless build cannot take. Cheap acceptance —
the Given/When/Then stories plus the named proxies Mars + Apollo verify against — is the multiplier.
