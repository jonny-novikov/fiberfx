# F6.5 ┬Ę Views with HEEx

> Server-render the catalog. An index template over `@courses` (`:for`, a `~p` link per course, escaped interpolation,
> an `:if` empty state) with no data access in the markup; a `course_card` function component with declared `attr`s; a
> slot-based `panel` wrapper; and a changeset-backed form whose create action redirects on success and re-renders the
> F6.3 changeset errors inline on failure. Data access stays in the controller, rendering stays in the template.

## Goal

After F6.5, the catalog is rendered from the server. `course_html/index.html.heex` renders `@courses` with `:for`, a
`~p` verified link per course, escaped interpolation for the title, and an `:if` empty state ŌĆö with no `Repo` and no
context call in the template. A `course_card` function component (`attr :course, :map, required: true`, optional
`:class`) replaces the per-row markup, and a slot-based `panel` wrapper (`slot :inner_block`, `render_slot/1`) frames
caller content. A changeset-backed form ŌĆö `to_form/1` over `Catalog.change_course/1`, rendered with `<.form>` and
`<.input>` ŌĆö drives the create action: params go to `Catalog.create_course/1`, `{:ok, _}` redirects, and `{:error, _}`
re-renders with `to_form(changeset)` so the F6.3 changeset errors appear inline per field. Data access stays in the
controller; the templates only render from assigns.

## Rationale (5W)

- **Why** ŌĆö HEEx is where Phoenix renders safely and composably: templates are compile-checked, interpolation is
  escaped by default, `~p` routes are verified, and function components turn repeated markup into a declared, reusable
  unit. The discipline that keeps data access in the controller and rendering in the template is what makes views
  testable and the markup free of surprise queries. Without it, templates make context calls, routes drift, and
  validation errors vanish.
- **What** ŌĆö the index template over `@courses`, the `course_card` component, the slot-based `panel` wrapper, the
  changeset-backed form, the create action with inline errors, and the controller-fetches/template-renders discipline.
- **Who** ŌĆö Visitors and Learners (a real rendered catalog and a working create form with clear inline errors).
- **When** ŌĆö the fifth Phoenix rung; it renders what F6.3 and F6.4 make queryable and writable, and it is the static
  UI that F6.6 makes live.
- **Where** ŌĆö `lib/portal_web/controllers/course_html/index.html.heex` and `course_html/new.html.heex`,
  `lib/portal_web/controllers/course_controller.ex`, `lib/portal_web/components/catalog_components.ex`, and a
  `Portal.Catalog.change_course/1` changeset wrapper.

## Scope

- **In** ŌĆö the index template (`:for`, `~p` link, escaped interpolation, `:if` empty state, no data access); the
  `course_card` function component with declared `attr`s; the slot-based `panel` wrapper; the changeset-backed form
  with `<.form>`/`<.input>`; the create action that redirects on success and re-renders inline errors on failure; and
  the data-access-in-controller discipline.
- **Out** ŌĆö LiveView and live validation (ŌåÆ F6.6); PubSub (ŌåÆ F6.7); a styling or design system beyond minimal classes;
  authentication UI; and pagination or search.

## Deliverables

- **F6.5-D1** ŌĆö the index template `course_html/index.html.heex` rendering `@courses` with `:for={course <-
  @courses}`, `<.link navigate={~p"/courses/#{course.id}"}>`, escaped `{course.title}`, an `:if={course.published}`
  badge, and an `:if={@courses == []}` empty state; no `Repo` and no context call appears in the template.
- **F6.5-D2** ŌĆö a `course_card` function component: `attr :course, :map, required: true`, `attr :class, :string,
  default: ""`, and a `~H` body returning an `<article>` with `{@course.title}` and an `:if` badge; called as
  `<.course_card :for={course <- @courses} course={course} />`.
- **F6.5-D3** ŌĆö a slot-based `panel` wrapper using `slot :inner_block` and `render_slot(@inner_block)` to frame caller
  content.
- **F6.5-D4** ŌĆö a changeset-backed form: `to_form(Catalog.change_course(%Course{}))` assigned as `@form`, rendered
  `<.form for={@form} action={~p"/courses"}>` with `<.input field={@form[:title]} label="Title" />`; backed by a
  `Portal.Catalog.change_course/1` wrapper returning an unsaved changeset.
- **F6.5-D5** ŌĆö the create action: params ŌåÆ `Catalog.create_course/1`; on `{:ok, _}` redirect to the course (or
  index); on `{:error, %Ecto.Changeset{} = cs}` re-render `new.html.heex` with `to_form(cs)` so the F6.3 changeset
  errors show inline per field.
- **F6.5-D6** ŌĆö the discipline: the controller performs all data access (the `Catalog` calls) and the templates render
  only from assigns; no template performs a query or names a `Repo` or context.
- **F6.5-D7** ŌĆö verification: the index renders courses and an empty state with no template data access; the component
  and slot render; a valid create redirects; an invalid create re-renders with inline field errors; every route is a
  verified `~p`.

## Invariants

- **F6.5-INV1 (templates render, controllers fetch)** ŌĆö templates read only from assigns; all data access (the
  `Catalog` calls) is in the controller; no template names a `Repo` or a context.
- **F6.5-INV2 (safe by default)** ŌĆö interpolation is HEEx-escaped (`{...}`); raw output is never used for user data.
- **F6.5-INV3 (verified routes)** ŌĆö every link uses a `~p` verified route, so a path typo fails to compile.
- **F6.5-INV4 (declared components)** ŌĆö function components declare their `attr`s (required where needed), so a missing
  required attr is a compile warning; slots are rendered with `render_slot/1`.
- **F6.5-INV5 (errors surface inline)** ŌĆö a failed create re-renders the form with `to_form(changeset)`, so the F6.3
  changeset errors appear inline per field rather than being swallowed.
- **F6.5-INV6 (parse boundary unchanged)** ŌĆö the form posts params to `Catalog.create_course/1`, which parses through
  the F6.3 `changeset/2`; the view adds no validation of its own.

## Definition of Done

- [ ] The index renders `@courses` with `:for`, a `~p` link, escaped title, an `:if` badge, and an empty state ŌĆö no
      data access in the template (F6.5-INV1, INV2, INV3).
- [ ] `course_card` declares `attr :course` (required) and optional `:class` and renders an `<article>` (F6.5-INV4).
- [ ] `panel` frames content with `slot :inner_block` and `render_slot/1` (F6.5-D3).
- [ ] The form is `to_form(Catalog.change_course(...))` rendered with `<.form>`/`<.input>` posting to `~p"/courses"`
      (F6.5-D4).
- [ ] A valid create redirects; an invalid create re-renders with `to_form(changeset)` and inline field errors
      (F6.5-INV5).
- [ ] The view adds no validation; parsing stays in `Catalog.create_course/1`'s changeset (F6.5-INV6).
- [ ] Every deliverable maps to a user story and every invariant is checked (see `./f6.5.stories.md`).
