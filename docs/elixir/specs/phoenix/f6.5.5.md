## Rationale

The Operator named the reframe: **publish, real production, the routes a visitor expects**. A strangler-fig first slice ships
more value than a synthetic catalog skinned in the tokens ŌĆö the Portal serves the *actual* course-index pages at the *actual*
routes, byte-faithful to the originals, while the deeper tree stays on production Fiber. The acceptance is correspondingly
concrete: not "renders in the system" as a claim, but two two-origin computed-style comparisons against the live static pages.

### Why

An unstyled platform is not shippable, and a *synthetic* styled catalog proves less than *real* styled pages. The Portal's
root renders a bare `<h1>Portal</h1>` (`home.html.heex:6`); F0 already specified the Portal reproducing the static system as
planned work (`f0.roadmap.md` M4 "Portal HEEx components reproducing tokens/anatomy" + M5 "Parity gate: static vs Portal").
Reframing the proof onto two real course-index pages executes both milestones against pages that ALREADY EXIST in the system
and are independently servable for byte-for-byte comparison ŌĆö closing a foundation already drawn, against live witnesses.

**1 ┬Ę Each parity page is a COMPLETE-DOCUMENT HEEx template ŌĆö NO root layout.** Today the controller macro sets `layouts: []`
(`portal_web.ex:30`), so the current `page_html/home.html.heex` (`<h1>Portal</h1>`) already renders BARE ŌĆö no `<html>`/`<head>`
wrapper is injected. The two-page contract LEANS ON this: each template renders the entire document
(`<!doctype html>` ŌåÆ `<html lang="en">` ŌåÆ `<head>ŌĆ”</head>` ŌåÆ `<body>ŌĆ”</body>` ŌåÆ `</html>`), reproducing its OWN golden
master's `<head>`. A single shared root layout CANNOT serve both at byte-parity because the two heads are irreconcilable:
`html/courses.html`'s head (`:3-10`) has `<title>Courses ┬Ę jonnify</title>`, the `Cormorant Garamond:ŌĆ”1,400;1,500;1,600`
font subset, and two `<style>` blocks declaring `--surface`/`--surface-2`/`--body`/`--shadow-*`/`--rad`; `elixir/index.html`'s
head (`:3-10`) has `<title>Functional Programming in Elixir ŌĆö a jonnify course</title>`, a different font subset, and a
`<style>` declaring `--ink-2`/`--ink-3`/`--elixir`/`--measure:68ch` and the `.chap` system. They share token NAMES
(`--ink`/`--cream`/`--gold`/`--line`) but diverge in values, in the surface/measure tokens, and in every component rule.
Therefore: **two full-document templates, NO `PortalWeb.Layouts`, NO change to `portal_web.ex`.** This SUPERSEDES the prior
rung's "root layout replacing `layouts:[]`" mechanism ŌĆö that mechanism would have forced ONE head, which cannot hold both
pages at byte-parity.

> **The new web surface for ratification.** The `/` route already exists (`router.ex:62`, `get("/", PageController, :home)`).

**2 ┬Ę CSS + JS extract to committed static assets ŌĆö no bundler.** Each golden master carries its tokens + base CSS + every

delimiter, so inlining a CSS `<style>` full of `*{ŌĆ”}`/`a{ŌĆ”}` rules in HEEx is unviable (HEEx would mis-parse the braces); and
inlining the JS would hit the same `{` problem plus HEEx mis-parsing the `<` comparisons. So each page's CSS and JS extract to
committed static files under `priv/static/assets/`, `<link>`ed / `<script src>`'d by the verified `~p"/assets/ŌĆ”"` route. Four
files, each the master content relocated VERBATIM except the link remaps:
- `courses.css` ŌĆö the TWO `<style>` blocks of `html/courses.html` (`:12-110` the shared chrome + `:111-133` the courses-page
  chrome), concatenated, the CSS between the tags. Opens with the `:root` token block (`:13-28`).
- `courses.js` ŌĆö the `<script>` body of `html/courses.html` (`:174-189`: the reveal-on-scroll `IntersectionObserver` + the
  `.filter-bar` tag-filter).
- `elixir-index.css` ŌĆö the `<style>` body of `elixir/index.html` (`:12-308`, the CSS between `<style>` and `</style>`). Opens
  with the F0-D1 `:root` block (`:12-26`) byte-for-byte (F0-INV2).
- `elixir-index.js` ŌĆö the two `<script>` bodies of `elixir/index.html` (`:550-711` the arc chapter-selector + pipe builder
  IIFE; `:715-730` the reveal bootstrap), the arc `CH` dataset `"route"` fields (`:554`) remapped to production.
