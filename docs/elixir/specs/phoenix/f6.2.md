## Goal

After F6.2, the Portal has a navigable and protectable surface. The router declares the four route kinds (`get`,
`post`, `resources`, `live`), three pipelines (`:browser`, `:api`, `:require_auth`), and a protected scope that
stacks `:require_auth` after `:browser`. A `PortalWeb.RequireUser` module plug loads the session user and either
assigns it or halts with a redirect; a function plug shows the same contract without a module. Every URL the app
builds uses a verified `~p` path. Handlers still call only the `Portal` facade.

## Rationale (5W)

- **Why** — F6.1 served a single route. A platform needs many routes (reads, writes, REST sets, live pages), public
  and protected areas, and cross-cutting concerns (session, CSRF, headers, auth) expressed once rather than repeated
  in every action.
- **What** — a router with the four route kinds, the three pipelines, a protected scope, a module plug and a function
  plug, and compile-time-verified `~p` paths.
- **Who** — Visitors and Learners (reach public and protected pages), Developers (declare cross-cutting work once and
  get compile-time path safety), Operators (a clear public/protected boundary).
- **When** — the second rung of the value ladder; depends on F6.1's endpoint and `:browser` pipeline, nothing above.
- **Where** — `PortalWeb.Router`, a new `PortalWeb.RequireUser` plug, a thin public `PortalWeb.PageController`, and
  the F6.1 `CourseController` (reused). Nothing below the `Portal` facade changes.

## Scope

- **In** — the four route kinds; the `:browser`, `:api`, and `:require_auth` pipelines; a protected scope; a
  `RequireUser` module plug (`init/1` + `call/2`) that loads the session user and halts with a redirect when absent; a
  function plug; verified `~p` paths; a thin public landing route as a valid redirect target; verification of public
  vs protected access and path verification.
- **Out** — the auth backend itself (registration, login, password hashing) — deferred to F6.8 (`phx.gen.auth`), which
  replaces the placeholder login target. Write/live route *targets* are thin here: the enroll command is fleshed out
  with contexts in F6.4 and the LiveView in F6.6. Persistence remains the F5 store until F6.3.

## Deliverables

- **F6.2-D1** — route kinds in `PortalWeb.Router`: `get "/courses/:id", CourseController, :show`; `post "/enroll",
  EnrollmentController, :create`; `resources "/lessons", LessonController, only: [:show]`; `live "/enroll/:id",
  EnrollmentLive`; and a public landing `get "/", PageController, :home`.
  · **[RECONCILE] Here is why:** `/courses/:id, :show` ≠ F6.1's reconciled `/courses/:user_id, :index` (param + action
  differ) — F6.1-dependent; settle at F6.1's close against the built route and `Portal.courses_of/1 -> {:ok,
  [%Enrollment{}]}` (see the [RECONCILE] banner above).
- **F6.2-D2** — a `:browser` pipeline (`fetch_session`, `fetch_live_flash`, `protect_from_forgery`,
  `put_secure_browser_headers`) and an `:api` pipeline (`accepts ["json"]`).
- **F6.2-D3** — a `:require_auth` pipeline running `PortalWeb.RequireUser`, used in a protected `scope` that
  `pipe_through [:browser, :require_auth]`.
- **F6.2-D4** — `PortalWeb.RequireUser`: `init/1` returns the options once at compile time; `call/2` reads `:user_id`
  from the session, assigns `:current_user_id` and returns the conn, or `halt`s with a redirect to the public landing
  and a flash when no user is present.
- **F6.2-D5** — a function plug: a one-line `(conn, opts)` step (e.g. stamping a request attribute), wired into a
  pipeline to show the plug contract without a module.
- **F6.2-D6** — verified `~p` paths used for every internal URL the app builds (links, redirects); a deliberately
  wrong `~p` does not compile.
- **F6.2-D7** — verification: public routes load without a session; protected routes redirect when unauthenticated and
  render when a `:user_id` is in the session; an `:api` route negotiates JSON; a mis-typed `~p` fails compilation.

## Invariants

- **F6.2-INV1 (master)** — every route maps to a handler that calls only `Portal` and renders only `%Portal.Error{}`;
  no plug or handler reaches below the facade.
- **F6.2-INV2** — cross-cutting work (session, flash, CSRF, secure headers, auth) is declared once in a pipeline and
  never copied into an action.
- **F6.2-INV3** — `RequireUser.call/2` either assigns the user and returns the conn or `halt`s with a redirect; an
  unauthenticated request never reaches a protected action.
- **F6.2-INV4** — every internal URL is a verified `~p` path; there are no hand-built path strings for internal
  routes.
- **F6.2-INV5** — the protected scope pipes `:browser` before `:require_auth`, so the session is fetched before auth
  reads it.
- **F6.2-INV6 (parse, don't validate)** — `RequireUser.call/2` parses the session exactly once: it reads `:user_id`
  and either assigns a typed `:current_user_id` or halts with a redirect; no downstream action re-reads or re-checks
  the raw session.

## Definition of Done

- [ ] `mix compile` is clean (which also proves every `~p` path resolves — F6.2-INV4).
- [ ] Public routes (`/`, `/courses/:id`, `/lessons/:id`) load without a session.
- [ ] A protected route redirects to the public landing when unauthenticated and renders when a `:user_id` is in the
      session (F6.2-INV3).
- [ ] An `:api` route negotiates JSON via `accepts ["json"]`.
- [ ] The protected scope pipes `:browser` then `:require_auth` (F6.2-INV5); no action contains copied cross-cutting
      code (F6.2-INV2).
- [ ] A deliberately wrong `~p` path fails to compile; reverting it restores a clean compile.
- [ ] No plug or handler references `Portal.Engine`, a repo, or `GenServer.call` (F6.2-INV1).
