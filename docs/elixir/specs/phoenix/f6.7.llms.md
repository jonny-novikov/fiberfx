## Agent stories

Each story is a contract — precondition (what must hold before), postcondition (what the gate asserts after), invariant
(what stays true throughout). The Operator and Apollo accept at the boundary, not by re-reading the diff.

- **F6.7-AS0** [implements F6.7-US0] *(RATIFICATION-GATED — build only on Director approval)* — **Directive:** add
  `Portal.Catalog.update_course/2` (`course |> Course.changeset(attrs) |> Repo.update()`, the `create_course/1` mirror)
  + the `Portal.update_course/2` `defdelegate`. · **Precondition:** `Course.changeset/2` exists (`course.ex:47`); no
  `update_course` exists anywhere. · **Postcondition:** `update_course(course, attrs)` returns `{:ok, %Course{}}` on a
  valid change and `{:error, %Ecto.Changeset{}}` on an invalid one — the same shape as `create_course/1` (`catalog.ex:117`).
  · **Invariant (INV2):** the web names only `Portal.update_course/2`, never the context or engine.
- **F6.7-AS1** [implements F6.7-US1, F6.7-US6] — **Directive:** add `Portal.subscribe/1` and `Portal.broadcast/2` over
  `Portal.PubSub` (two facade one-liners), and add `{Phoenix.PubSub, name: Portal.PubSub}` to `Portal.Application`
  (`application.ex:33-38`). · **Precondition:** the facade has neither fn; the tree has no PubSub child; `config.exs:38`
  already names `Portal.PubSub` (no config change). · **Postcondition:** a process that calls `Portal.subscribe("t")`
  receives a message sent via `Portal.broadcast("t", msg)`. · **Invariant (INV5):** the only new child here is
  `Phoenix.PubSub`; everything below the facade is unchanged.
- **F6.7-AS2** [implements F6.7-US2] — **Directive:** add a private `broadcast/2` helper in `Portal.Catalog` and call it
  from `create_course/1` and `update_course/2` only on `{:ok, _}`. · **Precondition:** both writes return
  `{:ok, %Course{}} | {:error, %Ecto.Changeset{}}`. · **Postcondition:** a successful write emits `{:course_created, _}`
  / `{:course_updated, _}`; a failed write (an `{:error, changeset}`) emits nothing. · **Invariant (INV1):** the helper
  is the single place writes broadcast from — clients only ever learn of facts.
- **F6.7-AS3** [implements F6.7-US1, F6.7-US3] — **Directive:** EXTEND `CatalogLive`'s `connected?/1` true-branch
  (`catalog_live.ex:69-73`) with `Portal.subscribe("courses")`, and add `handle_info/2` clauses that `stream_insert/3`
  (prepend on create at `at: 0`, replace on update) — **gated**: insert only when `@query == "" or
  String.contains?(String.downcase(course.title), String.downcase(@query))`; the row renders through the same
  `course_card/1` the mount uses (`catalog_components.ex:32`). · **Precondition:** the connected branch holds only a
  `:connected?` marker; `handle_info/2` does not exist on this module. · **Postcondition:** a create in one LiveView
  process appears in another with an empty filter; a non-matching course does NOT appear under an active filter; an
  update replaces the row without a duplicate. · **Invariant (INV3, INV6):** each client patches its own DOM; no client
  holds the full list; `stream_insert` keys on the id so a re-delivery does not duplicate.
- **F6.7-AS4** [implements F6.7-US4] — **Directive:** add `PortalWeb.Presence` (new `presence.ex`, `use
  Phoenix.Presence, otp_app: :portal_web, pubsub_server: Portal.PubSub`), its child in `PortalWeb.Application`
  (`application.ex:13-16`), `track/3` on the connected mount, and a `presence_diff` handler recomputing `viewers`. ·
  **Precondition:** no `presence.ex`; `Portal.PubSub` is started by AS1 and boots first (`:portal_web` → `:portal`). ·
  **Postcondition:** connecting and disconnecting changes `viewers`; the count is correct across nodes (CRDT-backed). ·
  **Invariant (INV4, INV5):** the count is cluster-correct, not a single-node counter; the only new web child is
  `PortalWeb.Presence`.

**Framing (carry into every emitted artifact):** no gendered pronouns for agents; no perceptual or interior-state verbs
("sees"/"wants"/"feels"); no first-person narration ("we"/"I think"); include this same propagation clause in any brief
derived from this one.
