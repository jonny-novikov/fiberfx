## 1. The rung in one paragraph (as-built)

F6.5.5 makes the unstyled Portal **render the F0 dark-editorial design system** by PUBLISHING two real
course-index pages at parity with their static Fiber originals — computed-style + geometry parity, not pixels:
`:4000/` (`PortalWeb.PageController.home/2` → `page_html/home.html.heex`) reproduces `html/courses.html`
(Fiber `:8765/courses`), and `:4000/elixir` (`PageController.elixir/2` → `page_html/elixir.html.heex`)
reproduces `elixir/index.html` (Fiber `:8765/elixir`). Mechanism: two **full-document HEEx templates**, NO
root layout (`layouts: []` already renders bare); per-page **extracted committed assets**
(`priv/static/assets/{courses.css,courses.js,elixir-index.css,elixir-index.js}`) served over the EXISTING
`Plug.Static` mount — NO bundler, NO shared `app.css`, NO `CoreComponents`. Internal navigation deep links
remap to production Fiber `https://jonnify.fly.dev` (a **strangler-fig** slice — Portal serves only `/` and
`/elixir`; everything else falls back to production). The lone new web surface: `PageController.elixir/2` +
`get("/elixir", …)`. The lone `endpoint.ex` edit: **FIX A** (below).

**Why the two-page contract superseded a shared root layout.** A single root layout cannot serve both pages
at byte-parity: the two `<head>`s are irreconcilable — different titles, font subsets, and entirely different
`<style>` token sets (`courses.html` uses `--surface`/`--body`; `elixir/index.html` uses `--elixir`/
`--measure`/`.chap`). Hence per-page full-document templates and four extracted assets, not one `app.css`.

**FIX A — the F6.1 latent this rung surfaced.** `endpoint.ex`'s `Plug.Static` was set by F6.1 to
`at: "/assets"` while `static_paths/0` is the `phx.new` default written for `at: "/"`. With `at: "/assets"` the
prefix is STRIPPED → `GET /assets/courses.css` resolves to the inner `"courses.css"` (∉ `only:`) → 404.
F6.5.5 is the mount's FIRST consumer, so the latent surfaced here. Ratified 1-line fix
(`at: "/assets"` → `at: "/"`); the rest of the diff is an explanatory comment.
