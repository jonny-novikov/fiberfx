## Where the combined rung's `[RECONCILE]` threads now live

The combined `f6.8.md` opened with a `[RECONCILE]` block folding F6.5/F6.5.5/F6.6 deferrals forward. Each thread
is now owned by one child (recorded here so the supersession is auditable):

- **Components & theming / the DYNAMIC-UI styling reckoning** — **DISSOLVES in F6.8.1** ([`f6.8.1.md`](f6.8.1.md)
  RECONCILE 2). The combined rung assumed the auth LiveViews would be the first dynamic styled pages, forcing a
  root-layout + shared-token + `CoreComponents` reckoning here. F6.8.1 ports the STATIC `login.html` (per-page
  extracted CSS, like F6.5.5) — there are no dynamic auth LiveViews — so the reckoning re-defers to the first
  genuinely-dynamic styled page (the live `/courses` restyle, or the F6.9 dashboard; the grooming's "dedicated
  dynamic-UI styling pass", [`phoenix.grooming.md`](phoenix.grooming.md)). The F6.6 flash-paint deferral moves
  with it.
- **Auth — the interim `RequireUser` superseded** — **F6.8.1** (RECONCILE 3, 4, D9). The hand-rolled `RequireUser`
  plug + the bare `:current_user_id` assign evolve into `PortalWeb.UserAuth` (`fetch_current_user/2` →
  `current_user`, `require_authenticated_user/2` → redirect to `/login`, `on_mount/4` `:ensure_authenticated`).
  F6.9's `live_session` forward-ref `{PortalWeb.UserAuth, :ensure_authenticated}` rides this.
- **Context — the `Accounts` persistence choice** — **F6.8.1** (the one open contract; §The persistence fork).
  `%User{}`/`%Session{}` already exist (F6.4); F6.8.1 adds a credential + hash verify + session mint (the
  recommended thin slice: a `password_hash` on the Store-backed user + one net-new hashing dep; Option B = Ecto
  tables, no new persistence dep). It does NOT run `mix phx.gen.auth` (the entities exist; the login UI is the
  static port).
- **Deploy — the strangler-fig two-origin boundary** — **F6.8.2.** F6.5.5 made the Portal serve `/` and `/elixir`
  at parity, deep links remapped to a configurable `:deep_link_base_url`; the deploy rung states which routes the
  Portal owns vs. fall back to Fiber and sets the base for the deployed environment.
- **Identity — the unguarded concurrent branded-id mint (F6.6)** — **F6.8.2.** A multi-node deploy makes the
  same-millisecond seq-0 collision live; `Portal.ID`'s `@node = 1` ("F6.8 derives it per machine") is the deploy
  rung's to harden — it is single-node-irrelevant for F6.8.1.
- **The live `/courses` shipped PUBLIC (F6.6)** — **F6.8.1** classifies it (RECONCILE 5): browsing stays public;
  whether the inline create is gated is deferred to the dynamic-UI pass (F6.8.1 ships the gate, that pass applies
  it).
- **Endpoint bind — `runtime.exs` hardcodes `127.0.0.1` (F6.6)** — **F6.8.2.** The deploy bind moves to
  `{0, 0, 0, 0}` under release/`:prod` while keeping `127.0.0.1` for local dev.
