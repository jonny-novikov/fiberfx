# F6.5.5 · user stories

> Who the two published course-index pages serve, what they need, and how we will know it works. Connextra stories with
> Given/When/Then acceptance criteria; together they cover every deliverable in [`f6.5.5.md`](f6.5.5.md). Acceptance is the
> TEXT PROXIES + the two-origin parity gate, not screenshots — full-document envelope / remapping / clamp-spacing /
> token-fidelity / computed-style parity (the index at `:4000/` vs the Fiber baseline `:8765/courses`, and `:4000/elixir` vs
> `:8765/elixir`, `f0.roadmap.md` M5). The clamp-spacing proxy is the highest-leverage (the unspaced bug shipped invisibly
> across 204 static pages), and here it holds by construction (each CSS file is the master `<style>` relocated verbatim).
> Stories honor the [`f6.5.5.md`](f6.5.5.md) reframe: TWO proof-pages — `/` = the two-course index (`html/courses.html`),
> `/elixir` = the Elixir index (`elixir/index.html`) — each a complete-document HEEx template with NO root layout, links
> remapped to `https://jonnify.fly.dev` where Portal does not serve the route (the `/` + `/elixir` self-routes stay relative),
> each page making NO domain call; the styling ships as committed CSS + JS files over the EXISTING `/assets` mount (no bundler)
> with NO `CoreComponents`.

## F6.5.5-US0 — The two published course indexes, styled

As a **Visitor**, I want the Portal to render BOTH course-index pages in the dark-editorial design system — the two-course
index at `/` and the Elixir course index at `/elixir`, each identical to its static original — so that the Portal serves the
real course landings at the routes I expect instead of an unstyled `Portal` stub.

Acceptance criteria
- Given a request to `~p"/"`, when it renders, then the response is the `html/courses.html` document reproduced as a
  full-document HEEx template — `<!doctype html>` → `<html lang="en">` → a `<head>` carrying `<title>Courses · jonnify</title>`,
  the Google-Fonts `<link>`, and `<link rel="stylesheet" href={~p"/assets/courses.css"}>` → `<body>` with the `.topbar`, the
  `.hero.hub-hero` ("In-depth courses."), the `#series` "Choose a course" section, the `.filter-bar`, the two `.series-card`s,
  and the `<footer>` (per [`f6.5.5.md`](f6.5.5.md) D1, INV8; the master `html/courses.html:136-190`).
- Given a request to `~p"/elixir"`, when it renders, then the response is the `elixir/index.html` document reproduced as a
  full-document HEEx template — head carrying `<title>Functional Programming in Elixir — a jonnify course</title>`, the
  Google-Fonts `<link>`, and `<link rel="stylesheet" href={~p"/assets/elixir-index.css"}>` → body with the master's
  `header.site` (with its `.route-tag`, master `:319`), the hero `<h1>Functional Programming in Elixir</h1>`, the arc figure,
  the pipe figure, the seven `.chap` sections, the pager, and the 3-column `footer.site-foot` (master `:512`) (per
  [`f6.5.5.md`](f6.5.5.md) D2, INV8; the master `elixir/index.html:311-731`).
- Given each page renders, when inspected, then it is a COMPLETE DOCUMENT with NO root-layout wrap and NO doubled header/footer
  — each template carries its OWN `<head>` and its OWN single body header/footer (the masters' own); no `PortalWeb.Layouts`
  module exists (per [`f6.5.5.md`](f6.5.5.md) §How.1, INV4).
- Given `:4000`, when curled, then `/` returns a 200 whose body contains `<head`, `courses.css`, and "Choose a course"; a 200
  with the old bare `<h1>Portal</h1>` is a REGRESSION; and `/elixir` returns a 200 whose body contains `<head`,
  `elixir-index.css`, and "Functional Programming"; a 404 on `/elixir` is a REGRESSION (INV8).

INVEST — independent; testable by the two liveness curls + an envelope/body grep on each route; encodes F6.5.5-INV8 and the
D1/D2 full-document envelopes.
Priority: must · Size: 3 · Implements deliverables: F6.5.5-D1, D2.

## F6.5.5-US1 — Links resolve where they should: relative in Portal, production elsewhere

As a **Learner**, I want every course link on the two published indexes to resolve correctly — the in-Portal pages relative
(so I can navigate the demo), every other course/chapter link on production — so that both indexes are working front doors even
though the deep pages are still served by Fiber.

Acceptance criteria
- Given the rendered `/elixir` page, when its internal links are read, then EVERY deep `/elixir/…` href has been remapped to
  `https://jonnify.fly.dev/elixir/…` — the hero CTAs, the seven `.chap` "Open chapter" links, every `.mod` card link, the
  `/elixir/course/…` links, and the footer links — while the bare `/elixir` self-links (the brand, the pager, the footer
  course link) STAY relative; no un-remapped `/elixir/…` deep link survives in the body (per [`f6.5.5.md`](f6.5.5.md) D5,
  INV5; the remap rule §How.3).
- Given the rendered `/` page, when its internal links are read, then the `/course/agile-agent-workflow` `.series-card` link is
  remapped to `https://jonnify.fly.dev/course/agile-agent-workflow` (Portal does not serve `/course/…`), while the `/elixir`
  `.series-card` link and the `/` topbar brand link STAY relative (Portal serves both) (per [`f6.5.5.md`](f6.5.5.md) D5, INV5).
- Given the `/elixir` arc chapter-selector, when a chapter is selected and its "Open Fn" link is read, then that link too
  points at `https://jonnify.fly.dev/elixir/…` — the arc `CH` dataset `"route"` fields in `elixir-index.js` are remapped (D4).
- Given the parity gate, when it inspects each Phoenix page, then it asserts the deep links resolve per the remap rule (D7,
  INV7).

INVEST — independent; testable by `grep -c 'href="/elixir/'` == 0 on `/elixir` + `grep -c 'jonnify.fly.dev/course'` > 0 on `/`
+ the parity-gate link assertions; encodes the remap rule and F6.5.5-INV5.
Priority: must · Size: 2 · Implements deliverables: F6.5.5-D5, D1, D2, D4.

## F6.5.5-US2 — Each page renders identically to its static original

As an **Operator**, I want each published Phoenix page to render identically to its live static counterpart, proven by a
computed-style comparison, so that "renders in the system" is a checked claim against a real witness — for BOTH routes — not a
resemblance.

Acceptance criteria
- Given the Playwright parity gate (a new/extended `apps/e2e` spec), when run, then it visits each index at BOTH origins —
  `:4000/` vs the Fiber `http://localhost:8765/courses`, and `:4000/elixir` vs the Fiber `http://localhost:8765/elixir` — and
  asserts the computed type scale matches per page: on `/elixir`, `h1` renders > 70px and `.prose p` body > 18px on BOTH (the
  no-space-`clamp()` regression guard); on `/`, the `.hero-title` renders large on BOTH (its clamp max is `4.4rem`) — using
  selectors that EXIST on each page (`courses.html` has `.hero-title`/`.hero-sub`, NOT `.prose p`) (per [`f6.5.5.md`](f6.5.5.md)
  D7, INV7; `f0.roadmap.md` M5).
- Given the same gate, when it measures geometry, then per page the layout matches between the Fiber baseline and the Phoenix
  route: on `/`, the hero sits above the `#series` section and the two `.series-card`s share a row (desktop); on `/elixir`, the
  arc figure sits above the first `.chap` and the `.chap`/`.mod` grid is multi-column (desktop) (D7).
- Given the gate, when it reads each Phoenix page's links, then they resolve per the remap rule — on `:4000/elixir`,
  `a[href^="/elixir/"]` count is 0 and `a[href^="https://jonnify.fly.dev/elixir/"]` count > 0; on `:4000/`, the agile link is
  `https://jonnify.fly.dev/course/…` while `/elixir` + `/` stay relative (D7, INV7).

INVEST — independent; testable by the two-origin parity spec across both routes; encodes F6.5.5-INV7 and the D7 parity gate.
Priority: must · Size: 3 · Implements deliverables: F6.5.5-D7.

## F6.5.5-US3 — Each page's CSS carried once, verbatim from its master

As a **Developer**, I want each master's `<style>` block(s) to live exactly once in a committed CSS file served by the mount
that already exists, so that the design system is carried faithfully with no bundler and no drift.

Acceptance criteria
- Given `priv/static/assets/courses.css` and `priv/static/assets/elixir-index.css`, when served, then each is delivered
  verbatim at `~p"/assets/courses.css"` / `~p"/assets/elixir-index.css"` by the `Plug.Static` mount AFTER the ratified 1-line
  `at:` correction (`endpoint.ex:28-32` `at: "/assets"` → `at: "/"`, FIX A — resolving the F6.1 latent where `at: "/assets"`
  strips the prefix and the assets 404) — with NO config change and NO bundler (per [`f6.5.5.md`](f6.5.5.md) D3, D4, INV4,
  §How.2). *(check: pre-fix `curl :4000/assets/courses.css` → 404, `…/assets/assets/courses.css` → 200; post-fix → 200
  `text/css`.)*
- Given `elixir-index.css`, when compared to the `elixir/index.html` `<style>` body (`:12-308`), then it is that CSS relocated
  VERBATIM — not reformatted, not re-indented — so its `:root` block (`elixir/index.html:12-26`, == `f0.md`-D1) matches
  BYTE-FOR-BYTE: the surfaces, text, accents, `--line`, the four font stacks, `--measure: 68ch` — declared exactly once and
  never re-invented (INV2; `f0.md`-INV2). *(token-fidelity proxy.)*
- Given `courses.css`, when compared to the two `html/courses.html` `<style>` bodies (`:13-109` + `:112-132`), then it is those
  CSS blocks concatenated VERBATIM, its `:root` block (`html/courses.html:13-28`) carried byte-for-byte (the `--surface`/
  `--body`/`--shadow-*`/`--rad` tokens this page adds) (INV2).
- Given each palette, when rendered, then it is declared once in its page's single `<head>`'s linked CSS and never redeclared
  per component — per full-document page (two pages ⇒ two `:root` blocks, one per head; INV2).

INVEST — independent; testable by a verbatim diff of each CSS file against its master `<style>` body + two `curl :4000/assets/…`
checks; encodes F6.5.5-INV2 and the D3/D4 no-bundler delivery.
Priority: must · Size: 2 · Implements deliverables: F6.5.5-D3, D4.

## F6.5.5-US4 — Every clamp survives the parser

As a **Developer**, I want every `clamp()` in both design CSS files to stay correctly spaced, so that titles and body text
render at their fluid sizes instead of silently falling back to UA defaults.

Acceptance criteria
- Given `courses.css` and `elixir-index.css`, when scanned, then every `clamp()` has spaces around its `+`/`-` operators
  (`clamp(2.7rem, 1.9rem + 4.2vw, 5.1rem)`, NEVER `1.9rem+4.2vw`) — holding by construction because each file is the master
  `<style>` body relocated UNCHANGED (the build must not reformat it) (per [`f6.5.5.md`](f6.5.5.md) INV3).
- Given the clamp-spacing proxy (a parse / `CSS.supports` probe over both CSS files), when run, then it reports zero unspaced
  `clamp()` — this is the highest-leverage acceptance check (the bug shipped invisibly across 204 static pages).
- Given a rendered heading on `:4000/elixir`, when its computed `font-size` is read, then it is the fluid clamp value (> 70px
  for h1), not the 32px UA fallback (caught by the same two-origin parity gate, INV7).

INVEST — independent; testable by the clamp-spacing parse over both files + the parity gate; encodes F6.5.5-INV3.
Priority: must · Size: 1 · Implements deliverables: F6.5.5-D3, D4, D7.

## F6.5.5-US5 — Publishing over the facade, each page makes no domain call

As an **architect**, I want both published indexes to add zero boundary crossings, make no `Portal` facade call, and leave
everything below the facade byte-unchanged, so that applying the design system and publishing the indexes cannot regress the
domain.

Acceptance criteria
- Given the two templates, the four asset files, and the new `elixir/2` action, when inspected, then they name ONLY the
  `~p"/assets/…"` static routes, the in-Portal relative links (`/`, `/elixir`), and literal external/production URLs — no
  `Portal.Engine`, `Repo`, or `GenServer.call`, and each page renders from NO assigns with NO facade call (both are static
  content) (per [`f6.5.5.md`](f6.5.5.md) INV1).
- Given the change set, when reviewed, then the ONLY web files touched are the two `page_html/*.heex` templates,
  `page_controller.ex` (a new `elixir/2` + docstring), the one new router line, the 1-line `endpoint.ex` `at:` fix (FIX A),
  `page_controller_test.exs`, and the four `priv/static/assets/` files — NO file under `apps/portal` or `apps/echo_data`, no
  endpoint plug BEYOND the `at:` line, no `portal_web.ex` macro, no `page_html.ex`, and no config key is touched; the engine,
  contexts, store, ids, and `EventStore` port are byte-unchanged
  (INV6).
- Given the build references, when grepped, then NO `CoreComponents` module, NO asset bundler (esbuild/tailwind/dart_sass), and
  NO `PortalWeb.Layouts` root layout is created or referenced; the CSS + JS ship as committed static files (INV4;
  `f6.5.md`-INV8 preserved).

INVEST — independent; testable by a boundary grep + a diff-scope review; encodes F6.5.5-INV1, INV4, INV6.
Priority: must · Size: 1 · Implements deliverables: F6.5.5-D6 (and the D1/D2 templates' facade-safety).
