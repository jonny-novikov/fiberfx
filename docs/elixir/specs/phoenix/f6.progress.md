## F6.5.5 — Apply the design system (two STATIC parity pages) · BUILD-GRADE

**Slice.** The styling rung inserted between F6.5 (views, shipped) and F6.6 (LiveView, shipped) but built
AFTER both. The Portal rendered UNSTYLED (`layouts: []`, `priv/static/assets/` held only `.keep`, `/`
returned a bare `<h1>Portal</h1>`). This rung applies the F0 dark-editorial system by PUBLISHING two real
course-index pages at parity with their static Fiber originals: `:4000/` reproduces `html/courses.html` (the
two-course index, Fiber `:8765/courses`) and `:4000/elixir` reproduces `elixir/index.html` (the Elixir index,
Fiber `:8765/elixir`). Mechanism: two full-document HEEx templates, NO root layout; per-page extracted assets
(`courses.css`/`courses.js`, `elixir-index.css`/`elixir-index.js`) over `Plug.Static`; NO bundler, NO shared
`app.css`, NO `CoreComponents`. Internal deep links remapped to `https://jonnify.fly.dev` (strangler-fig);
Portal serves only `/` + `/elixir`. The lone new web surface: `PageController.elixir/2` + `get("/elixir", …)`.

**The reframe (single-page → two-page parity).** The rung was reframed twice. ORIGINAL: style the synthetic
`/courses` LiveView catalog. First reframe: ONE published page at `/` reproducing `elixir/index.html`, catalog
demoted. Operator's final correction: BOTH course-index pages, each at the route a visitor expects, each at
parity with its live static original — `/` = `html/courses.html`, `/elixir` = `elixir/index.html`. The
two-page contract SUPERSEDED the prior single-page root-layout mechanism: a single shared root layout cannot
serve both at byte-parity because the two `<head>`s are irreconcilable (different titles, font subsets, and
entirely different `<style>` token sets — `courses.html` uses `--surface`/`--body`, `elixir/index.html` uses
`--elixir`/`--measure`). Hence per-page full-document templates and four extracted assets, not one `app.css`.

**FIX A — the F6.1 latent the rung surfaced.** `endpoint.ex`'s `Plug.Static` was set by F6.1 to `at: "/assets"`
while `static_paths/0` is the `phx.new` default written for `at: "/"`. With `at: "/assets"` the prefix is
STRIPPED, so `GET /assets/courses.css` resolves to the inner path `"courses.css"` (∉ `only:`) → 404. F6.5.5 is
the mount's FIRST consumer, so the latent surfaced here. The ratified 1-line fix (`at: "/assets"` → `at: "/"`)
is the precondition for serving the extracted assets — the lone `endpoint.ex` edit.

**Team & gate.** Venus (reconcile + brief + the triad FIX-A fold) → Mars-1 (the two pages) → Mars-2 (harden +
e2e) → Apollo (verify: BUILD-GRADE, every promise MATCH). Gate: compile `--warnings-as-errors` clean (forced
full recompile, exit 0, no warnings); `TMPDIR=/tmp mix test` **38 tests, 0 failures**. The **≥100 determinism
loop was correctly NOT run** (INV8) — styling + static publishing touches no id-mint and no process state, so
the same-millisecond mint hazard the loop guards cannot fire; the liveness criterion is the two styled-index
curls + the parity gate. Visual-proxy gate: clamp-spacing parse (0 unspaced `[+-]` in both CSS files),
token-fidelity diffs CLEAN, the two-origin Playwright parity gate **11/11 across both origins**.

  `courses.css` == the two `courses.html` `<style>` bodies concatenated clean. The highest-leverage proxy (the
  204-page UA-fallback bug) is satisfied by construction, not by spot-check — the no-reformat discipline held.
- **The liveness-with-visual-dimension gate ran and the FIX-A DOUBLE proved the mount move.** The prior rung's

  `/assets/courses.css` 200s — confirming the mount genuinely moved from `at: "/assets"` to `at: "/"`, not an

- **The two-origin parity gate is real render-parity, not a claim.** 11/11 — the type-scale + geometry
