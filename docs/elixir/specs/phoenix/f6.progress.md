## F6.5.5 — Apply the design system (two STATIC parity pages) · BUILD-GRADE

(`courses.css`/`courses.js`, `elixir-index.css`/`elixir-index.js`) over `Plug.Static`; NO bundler, NO shared

**The reframe (single-page → two-page parity).** The rung was reframed twice. ORIGINAL: style the synthetic
`/courses` LiveView catalog. First reframe: ONE published page at `/` reproducing `elixir/index.html`, catalog
demoted. Operator's final correction: BOTH course-index pages, each at the route a visitor expects, each at
parity with its live static original — `/` = `html/courses.html`, `/elixir` = `elixir/index.html`. The
two-page contract SUPERSEDED the prior single-page root-layout mechanism: a single shared root layout cannot
serve both at byte-parity because the two `<head>`s are irreconcilable (different titles, font subsets, and
entirely different `<style>` token sets — `courses.html` uses `--surface`/`--body`, `elixir/index.html` uses
`--elixir`/`--measure`). Hence per-page full-document templates and four extracted assets, not one `app.css`.

**FIX A — the F6.1 latent the rung surfaced.** `endpoint.ex`'s `Plug.Static` was set by F6.1 to `at: "/assets"`
while `static_paths/0` is the `phx.new` default written for `at: "/"`. With `at: "/assets"` the prefix is
STRIPPED, so `GET /assets/courses.css` resolves to the inner path `"courses.css"` (∉ `only:`) → 404. F6.5.5 is
the mount's FIRST consumer, so the latent surfaced here. The ratified 1-line fix (`at: "/assets"` → `at: "/"`)
is the precondition for serving the extracted assets — the lone `endpoint.ex` edit.

e2e) → Apollo (verify: BUILD-GRADE, every promise MATCH). Gate: compile `--warnings-as-errors` clean (forced

#### Chapter-level lessons (distilled from the F6.5.5 close-out Mentoring Q&A — full Q&A in [`f6.5.5.progress.md`](f6.5.5.progress.md))
These generalize past F6.5.5 to every later rung:
- **Liveness is a STANDING property for an Operator-facing rung, not a transient gate.** boot→curl→kill from
  an ephemeral agent process proves CAPABILITY but leaves no artifact the Operator can re-probe — leave the
  node running (or hand the Director a durable boot command). The F6.6 "green suite, dead route" hazard, lifted
  one level. (Sharpens the F6.6 liveness guardrail in `mars.md`/`apollo.md`.)
- **When a sanctioned realization drifts the spec, sync PROSE to a deliverable-pinned realization; sync CODE
  only when the realization itself breaks an invariant.** The stale party is the one NOT pinned by an authority
  the other already cites. (F6.5.5: the prescribed 3-arg resolver outranked the incidental "ONLY in config"
  sentence — keep the code, fix the words; do not amend code to match prose and break the deliverable that
  prose served.)
- **A config value shared by a server-rendered surface and a byte-served static asset crosses the boundary
  EXACTLY once** — read once on the server, inject once for the asset to read; the runtime analog of F0-INV2
  ("declared once"), and the grep that returns "only one source" IS the proof, made an exercised claim by a
  config swap.
- **A grep/check verdict is only as trustworthy as its pattern** — a CLEAN/leak emitted on a coarse match or a
  tool error is the "a check counts only if it RUNS" failure turned inward on the verifier; read the hit and
  tighten the pattern before vouching the verdict.
- **The grade-first firewall + "a guardrail that re-fires correctly earns its place (no edit)":** mentoring
  runs on the verdict's evidence AFTER the verdict is rendered; a finding that would move the grade is a
  BLOCKING delta routed to Mars, never a softening note — so nothing grade-moving survives to the mentoring
  stage. A durable peer-def line is earned by a recurring FAILURE, never by restating a guardrail that worked.
