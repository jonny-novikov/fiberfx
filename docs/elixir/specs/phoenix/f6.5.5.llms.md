## The configurable base-url contract (D9 / R8 / AS8 — Mars-3's primary deliverable this pass)

The Operator's words: *"relative links will become with base url configurable «`https://jonnify.fly.dev`» e.g.
`/elixir/algebra` → `https://jonnify.fly.dev/elixir/algebra`"* — AND *"still static assets must be served by Portal Phoenix."*
The two-page parity slice shipped with the host as a bare literal written 22 times (1 `home.html.heex`, 20 `elixir.html.heex`, 1
`elixir-index.js` `CH` line). This pass lifts that host into application config, read ONCE, reaching both **nav-link** surfaces
from the single read — and touching ONLY navigation deep-links, NEVER the Portal's own static assets.

**The link taxonomy — the base applies to CATEGORY 4 ONLY (read this before editing any href).** Every URL in either page is
exactly one of five categories; the configurable base touches ONLY category 4. The build must NOT rewrite the others:

| # | Category | Examples | Treatment |
|---|---|---|---|
| 1 | **Portal-served routes** | `/`, `/elixir` | **LOCAL / relative** — never remapped. |
| 2 | **Portal static assets** | `~p"/assets/courses.css"`, `…/courses.js`, `…/elixir-index.css`, `…/elixir-index.js`, `favicon.ico`, any local vendored asset | **LOCAL** — `<link>`/`<script>` via `~p"/assets/…"`, served at `:4000/assets/*` by Portal's OWN `Plug.Static` (what FIX A `at: "/"` enabled). **NEVER remapped.** *(A Portal page pulling its own CSS/JS from `fly.dev` is a hollow shell + a liveness regression even if it visually matches.)* |
| 3 | **External CDN** | jsDelivr KaTeX, Google Fonts (`fonts.googleapis.com`, `fonts.gstatic.com`) | **UNTOUCHED** — already absolute external. |
| 4 | **Nav deep-links to NON-served routes** | `/elixir/algebra`, `/elixir/<ch>/<mod>`, `/elixir/course…`, `/course/agile-agent-workflow` | **THE BASE APPLIES HERE — and only here.** |
| 5 | **Same-page anchors** | `#main`, any `#…` | **UNTOUCHED.** |

Only the ~20 `elixir.html.heex` deep links, the 1 `home.html.heex` agile card, and the ~6 `elixir-index.js` arc `CH` routes are
category 4. The asset `<link>`/`<script>` tags (category 2), the `/`+`/elixir` self-links (category 1), the fonts (category 3),
and `#main` (category 5) are LEFT EXACTLY AS-BUILT. The JS gets the base VALUE for its nav routes ONLY — it must NEVER prefix an
asset fetch (the assets are server-rendered category-2 tags the JS does not touch).

**The load-bearing design decision — config crosses the server/static boundary EXACTLY ONCE.** Two surfaces need the host: the
server-rendered `<a href>` markup (HEEx, can read config at render time) and the static `elixir-index.js` arc "Open Fn"
link-builder (a `Plug.Static` file served byte-for-byte that CANNOT read Elixir config). The HEEx `<head>` injects the base ONCE
as `window.__deepLinkBase`; the static JS reads it. That single injection IS the boundary crossing.
