## The rung in one paragraph

F6.6 makes F6.5's server-rendered catalog **interactive without a reload**. `PortalWeb.CatalogLive`
(`use PortalWeb, :live_view`) mounts `@courses` from the facade and renders by reusing the F6.5
`course_card`; a two-stage mount (disconnected HTTP paint → connected socket, `connected?/1`-guarded)
keeps first paint fast and indexable; a live search box filters as the learner types; a live create form
flashes on `{:ok, _}` and shows inline `to_form(changeset)` errors on `{:error, _}`; a `stream` keeps a
large list off the socket heap. The F6.3 changeset and F6.4 context are unchanged. Spec triad:
[`f6.6.md`](f6.6.md) (body, authoritative) · [`f6.6.stories.md`](f6.6.stories.md) (Connextra + Given/When/Then,
covers every deliverable) · [`f6.6.llms.md`](f6.6.llms.md) (the agent brief Venus refreshes).

## Mode — Flat-L2 lead-team, review gates (not Workflow)

Real `Agent` spawns + SendMessage coordination, Director-in-loop between stages — **not** a deterministic
`Workflow` fan-out. The rung is sequential (Mars builds from Venus's brief; Apollo verifies Mars's tree)
with two human-grade decision points (the reconcile result, the final commit), so the value is in the
*gates*, not in parallelism. Process weight scales to **risk**: F6.6 touches one umbrella app
(`portal_web`) over an unchanged engine — moderate risk — so this is a lead-team, not the full cclin
audit ceremony. All peers Opus (LAW-2). The Director coordinates and ratifies; peers write the code
(LAW-1a); one commit at close (LAW-4).

## What reconcile already settled — and the internal STALEs it leaves

direction, `47a15f1`) already **decided** the three questions that used to be forks. The stories already
encode the decisions (US0/US1/US2). So Venus does **not** surface these as open forks — they are closed:

| Settled by `[RECONCILE]` | Decision |

| Components | reuse the **local** `PortalWeb.CatalogComponents` (`course_card`/`panel`/`input`); **no** `CoreComponents`. |

But the decisions postdate the spec body, so the body now **contradicts itself** — and that is Venus's
lag-1 reconcile work (correct the body to the folded-forward direction; the body wins only where
`[RECONCILE]` is silent):

- `f6.6.md` **D3** + the Goal say search filters "through the `Catalog` context"; **D6** + the Goal say
  `live "/catalog"`; **Scope/Out** says the facade is "unchanged." All three are STALE against the
  `[RECONCILE]` + stories. → correct them.
- The new contract the search direction requires **does not exist yet**. The as-built facade is
  `list_courses/0 · get_course!/1 · fetch_course/1 · change_course/0 · create_course/1 · lesson/1`
  (`apps/portal/lib/portal.ex:41-46`) — no search. → Venus **pins** `Portal.search_courses/1` →
  `Catalog.search_courses/1 :: [Course.t()]` (a read-only delegate, named to match the siblings). It is
  an addition the `[RECONCILE]` already ratified in principle, so Venus pins it in the brief and flags it

## Why the prompts are short (the evolved process)

F6.5's Mars prompt ran ~180 lines — the full contract plus every discipline (cite-don't-invent, the
master invariant, the `@enforce_keys` trap, the determinism gate, framing, the report format). F6.6's
Mars prompt is ~12, because all of that now lives in `mars.md`. The spawn carries only what is *new
about this rung*. That is DRY applied one level up: the "how the agent works" fact has one authority
(the agent definition), and the prompt carries only the delta. Cheap acceptance — the Given/When/Then
stories Mars + Apollo verify against — is the multiplier.
