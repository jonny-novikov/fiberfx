# F6.5 · user stories

> Who the views serve, what they need, and how we will know it works. Connextra stories with Given/When/Then
> acceptance criteria; together they cover every deliverable in [`f6.5.md`](f6.5.md).

## F6.5-US0 — One honest URL per resource

As an **architect**, I want each URL named after the resource it returns, so that the catalog and a learner's
enrollments stop colliding on `/courses`.

Acceptance criteria
- Given the catalog, when routed, then `resources "/courses", CourseController, only: [:index, :show, :new, :create]`
  serves `GET /courses`, `GET /courses/:id`, `GET /courses/new`, and `POST /courses` from `Portal.Catalog`.
- Given a learner's enrollments, when routed, then `get "/my/courses", EnrollmentController, :index` serves them under
  `pipe_through [:browser, :require_auth]`, reading `conn.assigns.current_user_id` — and `get "/courses/:user_id"` and
  `scope "/learn"` no longer exist.
- Given a successful enroll, when it redirects, then the target is `~p"/courses/#{course_id}"` (the catalog show of the
  joined course), not a `user_id`.

INVEST — independent; testable through the router pipeline; encodes F6.5-INV7.
Priority: must · Size: 2 · Implements deliverables: F6.5-D0.

## F6.5-US1 — See the catalog

As a **visitor**, I want the course list rendered as a page, so that I can see what is available and follow a link to
each course.

Acceptance criteria
- Given `@courses` (= `Portal.list_courses/0`), when the index renders, then it shows one entry per course with `:for`.
- Given a course, when rendered, then its link uses `~p"/courses/#{course.id}"` (the catalog `:show`) and its title is
  escaped.
- Given no courses, when the index renders, then an `:if={@courses == []}` empty state appears — and the template makes
  no `Repo` or context call.
- Given a course id, when `:show` is requested, then `CourseController.show/2` (`Portal.get_course!/1`) renders the
  course — so the index link and the create redirect compile under Verified Routes.

INVEST — independent; testable by rendering with and without courses and by requesting `:show`; encodes F6.5-INV1,
INV2, INV3.
Priority: must · Size: 2 · Implements deliverables: F6.5-D1, F6.5-D8.

## F6.5-US2 — A reusable course card

As a **developer**, I want a `course_card` function component, so that per-row markup is declared once and reused.

Acceptance criteria
- Given the component, when declared, then `attr :course, :map, required: true` and `attr :class, :string, default:
  ""`.
- Given the index, when it renders, then it calls `<.course_card :for={course <- @courses} course={course} />`.
- Given a missing required attr, when compiled, then a warning is raised.

INVEST — independent; testable by rendering and by compile output; encodes F6.5-INV4.
Priority: should · Size: 2 · Implements deliverables: F6.5-D2.

## F6.5-US3 — Frame content with a slot

As a **developer**, I want a slot-based `panel` wrapper, so that callers can frame arbitrary content consistently.

Acceptance criteria
- Given `panel`, when declared, then it uses `slot :inner_block`.
- Given a caller, when it wraps content, then `render_slot(@inner_block)` renders the inner content inside the frame.

INVEST — independent; small; testable by rendering; encodes F6.5-D3.
Priority: could · Size: 1 · Implements deliverables: F6.5-D3.

## F6.5-US4 — Create a course

As a **learner** (an author), I want a form to create a course, so that I can add one through the UI.

Acceptance criteria
- Given the new page, when it renders, then `@form` is `to_form(Catalog.change_course(%Course{}))` (the new wrapper +
  its `Portal` facade delegate).
- Given the form, when rendered, then it uses `<.form for={@form} action={~p"/courses"}>` and the **locally-defined**
  `<.input field={@form[:title]} label="Title" />` (no `CoreComponents` exists; `input/1` lives in
  `catalog_components.ex`).
- Given valid params, when submitted, then `Catalog.create_course/1` succeeds and the action redirects.

INVEST — independent; testable by rendering and submitting; encodes F6.5-INV6, INV8.
Priority: must · Size: 2 · Implements deliverables: F6.5-D4, F6.5-D9.

## F6.5-US5 — See my mistakes inline

As a **learner**, I want validation errors shown on the form, so that I can fix a bad submission without losing
context.

Acceptance criteria
- Given invalid params, when submitted, then the action re-renders `new.html.heex` with `to_form(changeset)`.
- Given the re-render, when shown, then the F6.3 changeset errors appear inline per field.
- Given valid params, when submitted, then no errors show and the action redirects to `~p"/courses/#{course.id}"` (the
  catalog `:show`).

INVEST — independent; testable by an invalid submission; encodes F6.5-INV5.
Priority: must · Size: 2 · Implements deliverables: F6.5-D5.

## F6.5-US6 — Rendering stays pure

As an **architect**, I want data access kept in the controller, so that templates render and never query.

Acceptance criteria
- Given the controller, when an action runs, then it performs the `Catalog` calls and assigns the result.
- Given any template, when inspected, then it reads only from assigns and names no `Repo` or context.

INVEST — independent; testable by inspection; encodes F6.5-INV1.
Priority: must · Size: 1 · Implements deliverables: F6.5-D6.
