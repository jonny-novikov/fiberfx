Branded ids

A 64-bit integer is right for a database column, but awkward in a URL, a log line, or a support ticket. A branded id dresses it for the outside: encode the integer in base62 — the alphabet 0-9A-Za-z, url-safe with no punctuation — left-pad it to eleven characters, and prepend a three-letter namespace. The result is PGE0NbWMtkosp8: self-describing, compact, and still sortable.

One Snowflake, 319545566822428714, taken through encode → brand → decode. Every step is computed on this page.

Integer to branded string Advanced: why base62, and why the order survives

Integer to branded string

Branding is reversible: encode the integer to base62, prepend the namespace, and to read it back, strip the three-letter prefix and decode the rest. Select a step to see the value it produces.

A branded id is a lossless skin over an integer: encode and prefix on the way out, strip and decode on the way in. The database keeps the 64-bit number; everyone else sees a tidy string.

Advanced: why base62, and why the order survives

Base64 is more common, but its alphabet includes +, /, and = padding — characters that need escaping in URLs and look wrong in an id. Base62 drops them, keeping only 0-9A-Za-z, so a branded id is safe in a path, a query string, a filename, or a double-click. Eleven base62 characters cover 62¹¹, which is greater than 2⁶⁴, so any Snowflake fits with room to spare; left-padding to a fixed eleven keeps every id the same length.

Fixed width is what preserves order. Because every payload is exactly eleven characters, comparing two branded ids character by character compares their integers digit by digit — so the lexical order of the strings is the numeric order of the Snowflakes, which is time order. Branding changes how an id looks, never how it sorts. The namespace prefix sits in front, shared by every id of a kind, so it never disturbs that order; it only tells a reader, and a router, what the id points at. This is the id the branded CHAMP map of F4.09 keys on, partitioned by that very prefix.

# EchoData.Base62 — alphabet "0-9A-Za-z", width 11
EchoData.Base62.encode(319545566822428714)
# => "0NbWMtkosp8"

branded = "PGE" <> "0NbWMtkosp8"        # => "PGE0NbWMtkosp8"
EchoData.Base62.decode(String.slice(branded, 3, 11))
# => {:ok, 319545566822428714}
