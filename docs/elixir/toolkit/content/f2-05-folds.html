F2 · Functional Programming

map / filter / reduce (folds)

A fold collapses a list to a single value by threading an accumulator through it, combining one element at a time. Enum.reduce is that fold — and it is the one list operation the others are built from.

F2.04 ended on the observation that sum, length, reverse, map, and filter are all the same recursion with a different combiner. This module makes the combiner the subject. Give reduce a starting value and a function of (element, accumulator), and it walks the list left to right, replacing the accumulator at each step. Change the combiner and the same machine computes a sum, a product, a maximum, a count — or builds a whole new list, which is how map and filter fall out of it.

How a fold runs

A fold starts with an accumulator and visits each element in turn. At every step it applies the combiner to the current element and the accumulator, and the result becomes the new accumulator. When the list is spent, the accumulator is the answer.

Change the combiner

The walk is fixed; the combiner and starting value are not. Swap them and the same fold computes a sum, a product, a maximum, or a count — without touching how the list is traversed.

The universal fold

The accumulator need not be a number. Let it be a list and the fold builds a list — which is exactly what map and filter do. Map appends each transformed element; filter appends only those that pass. Both are reduce with a list accumulator. Reduce can express map and filter; map and filter cannot express reduce. That is why reduce is the fundamental one.

What this lands

A fold is a starting value and a combiner threaded down a list. reduce is that fold, left to right and tail-recursive, so it carries the accumulator the F2.04 way without growing the stack. Swap the combiner and one machine yields sum, product, max, or count; give it a list accumulator and it becomes map and filter. That is the sense in which reduce is universal — it is the operation the others reduce to. The four deep dives above take map, filter, reduce, and the wider toolkit one at a time; F2.06 then turns to the functions those combiners so often are: closures that capture a value from around them.
