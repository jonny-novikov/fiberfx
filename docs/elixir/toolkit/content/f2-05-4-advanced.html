Advanced folds

Once reduce is clear, the rest of the Enum toolkit reads as folds with extra structure. Scan keeps every intermediate result; map_reduce maps and accumulates at once; flat_map, group_by, and chunk only choose a different accumulator shape.

Reduce throws away every intermediate accumulator and returns the last one. Many useful operations want those intermediates, or a second value alongside the result, or a particular grouping. Each is a small variation on the same left-to-right fold — worth recognising so you reach for the packaged one instead of rebuilding it.

scan: a running fold The toolkit

scan: a running fold

Reduce returns only its final accumulator. Scan returns all of them — one entry per element, each the accumulator so far. Folding addition with reduce gives the total; with scan it gives the running totals.

Where this goes

That completes F2.05. A fold threads an accumulator down a list; map and filter are folds with a list accumulator; reduce is the general one, taking an accumulator of any shape; and the wider Enum toolkit is folds with extra structure. The combiners these folds take are usually small functions that capture a value from their surroundings — which is exactly what F2.06 is about.
