Identifiers, Snowflake & branded ids

The anatomy of a branded id Three deep dives Advanced: the five W’s of an id

Every row in F4.04’s page registry, every user and session in the portal, needs a name. A database counter (1, 2, 3…) names them in one place, but the moment two machines mint ids at once it breaks. A Snowflake answers that: a 64-bit integer that carries its own millisecond timestamp, the worker that made it, and a per-millisecond sequence — sortable by time and unique without a central counter. A branded id wraps it for the outside world: a three-letter namespace and a base62 encoding, e.g. TSK0KHTOWnGLuC.

This is the id behind every other module’s keys: the registry of F4.04 keys %Page{} rows by one, the trie of F4.05/F4.06 hashes it, and F4.09 keys a branded CHAMP on it. The build stamp at the foot of every page in this course is one of these.

The anatomy of a branded id

A branded id is two parts glued together: a three-letter namespace that names the kind of thing (TSK for a task, USR for a user) and an eleven-character base62 payload that is a 64-bit Snowflake. The namespace lets a service route an id without decoding it; the payload, once decoded, hands back a timestamp and more. Select a part to read.

One string carries a type and a timestamped, coordination-free integer. A service can route on the first three characters and never decode the rest; a database can store the integer and sort by it.

Advanced: the five W’s of an id

A good identifier answers five questions at once. What kind of thing is it — the namespace says so. When was it made — the timestamp bits say so, to the millisecond. Where was it made — the worker bits name the machine or process. Which one within that millisecond — the sequence bits disambiguate. And why not a counter: because a counter needs one writer, and a distributed system has many. A Snowflake lets every worker mint ids on its own and still keeps them globally sortable, because time is in the high bits.

For the portal, that means a record can be sorted, paginated, and range-queried by time without a separate created_at column, and a service can shard by worker or route by namespace without ever parsing the payload. The same id is the key the page registry stores, the value the trie hashes, and the stamp printed at the foot of this page.

# EchoData mints one, the portal brands it for the wire
snowflake = EchoData.Snowflake.generate(worker_id: 7)
# => 274557032793636864  (a 64-bit integer; time-ordered)

branded   = "TSK" <> EchoData.Base62.encode(snowflake)
# => "TSK0KHTOWnGLuC"  (3-char namespace + 11-char base62)
