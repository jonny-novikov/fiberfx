Tracer bullets: a walking skeleton

A vertical slice Three deep dives

A running server and a domain model are still two separate things until one request travels the whole way between them. F5.03 fires a tracer bullet: it takes a single use case — enroll a learner — and drives it end to end through every layer at once, each layer present but minimal. A route on the thin server calls one function on the Learning context, which builds an %Enrollment{} and puts it in the store, and the handler answers. That thin vertical slice is the walking skeleton: real code, not a throwaway, and the frame every later feature drops into. Once it walks, the system grows by iterating the slice.

One use case cutting straight down the stack. Select a layer the slice passes through to see what it does — minimal, but real.

A vertical slice

The skeleton cuts vertically, not horizontally. Instead of finishing the web layer, then the whole engine, then persistence — with nothing running until the end — it takes one path through all of them and makes only that path work. Every layer is touched, none is finished. Select a layer to see the one real thing the slice does there.

A walking skeleton is the whole system in miniature: thin everywhere, finished nowhere, but alive end to end. It turns “does this design hold together” into a question you answer on day one.

Three deep dives

In order: what a tracer bullet is and how it differs from a prototype; the walking skeleton that drives one use case through every layer; and how to iterate that slice into a real system.
