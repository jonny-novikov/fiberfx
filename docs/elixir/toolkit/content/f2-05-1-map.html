F2.05 · part 1 of 4

map

Map applies one function to every element and collects the results. The output has the same length as the input and the same order — one element in, one element out. It transforms contents without touching shape.

Because each element is handled independently, map says nothing about order of effects and everything about the transformation. That independence is also why two maps in a row collapse into one: doing f then g to every element is the same as doing g(f(x)) once.

One function, every element

Hand map a function and it runs that function over each element, building a new list of the results. The input is untouched; the output is the same size. Swap the function and the same call transforms differently.

F2.05 · reduce

Map is reduce with a list accumulator that appends each transformed element.

Map changes every element the same way and keeps the list’s size and order. The function is the only thing that varies.

Chained maps fuse

Map one function, then map another, and you have walked the list twice. But mapping f then g gives the same result as mapping the single function x → g(f(x)) once. The two passes fuse into one.

Compare two passes against the fused map.

F1.03 · composition

Mapping a composition equals composing the maps — one walk instead of two.
