Chapter F1 · nine lessons

Algebra.

The first crossing. Before a single line of Elixir, one thing becomes plain: the algebra you already know — functions, substitution, composition — is functional programming.

Each lesson pairs one algebraic idea with the Elixir it becomes: a function is a first-class value, substitution is referential transparency, composition is the pipe, and the higher-order operators Σ and Π are a fold. Read in order, the chapter is a dictionary you already half-know — nine lessons that turn quiet intuition into the foundation everything after it rests on.

The arc of the chapter

Nine lessons in four movements. Foundations fix what a function, a substitution, and a composition are. Structure adds immutable values, collections, and recursion. The operators generalise sums and products into folds and solve equations by their shape. The lab plots and composes functions you can drag.

Select a stop on the path to see what it covers.

The chapter is a single line of reasoning: settle what a function is, learn the operators that act on it, then compose those operators into programs.

A function is a mapping

A function sends each input to exactly one output — that is the entire definition. Pick a rule and watch the whole mapping redraw at once, with nothing mutated and every input carrying exactly one arrow. In Elixir the same idea is written fn x -> … end.

F1 · Algebra

A function f : A → B is a rule sending each element of the domain to one element of the range.

→

Elixir

A first-class value: f = fn x -> x + 2 end — pass it, return it, compose it.

Every input has exactly one arrow. Switch the rule and the mapping redraws whole — the mathematical meaning and the Elixir value are the same object.

The algebra you already speak

Functional programming did not invent its ideas — it inherited them from algebra. Map the vocabulary once and Elixir stops looking foreign.

algebraElixir / FP
a function f : A → B→a function (first-class)
substitution→referential transparency
composition f ∘ g→the pipe |>
x is a fixed value→immutable data
Σ, Π over a set→reduce / fold
recursive definition→recursion (no loops)
solving by structure→pattern matching

None of this is a metaphor. An Elixir pipeline is a composition of functions; an Enum.reduce is a finite sum written generally; pattern matching solves an equation by its shape. F1 makes each correspondence explicit.
