F3 · Data & shape · module 5

Shaping data with structs

The collections walked in F3.04 were maps — loose bags of keys and values. That works until a value needs a guaranteed shape: a learner is always an id, an email, and a role, never something else. Elixir gives three containers for key-and-value data — the map, the keyword list, and the struct — and the skill is knowing which one a situation calls for.

The portal's Portal.Accounts.User is the running example. The same learner data fits all three shapes; this module shows what each one guarantees and gives up, then takes the struct apart over three deep dives.

Three shapes, one record Three deep dives

Three shapes, one record

A map takes any keys and lets you add or drop them at runtime. A keyword list is an ordered list of {atom, value} pairs whose keys may repeat — the shape options are passed in. A struct fixes its key set at compile time and stamps the value with its module. Choose a shape and watch the same learner data take its form, and which properties hold.

No shape is best everywhere: a map for open-ended data, a keyword list for ordered options, a struct when a value must always carry the same fields. A struct is the only one of the three that a compiler can check.
