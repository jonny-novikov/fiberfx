## {eg-4-alternatives} Alternatives

### V-1 — backpressure mechanism: multiple-lanes-per-vid vs bounded-in-flight-per-vid

The brief (eg4-risks) names two satisfying options for S-7 isolation; recording the road not taken.
- CHOSEN: bounded-in-flight-per-vid. A per-Volume counter capped at N admitted-but-unfinished commands; overflow → Err{unavailable}. Rationale: the per-vid command LANE (egraft:cmd:{vol}) already gives structural isolation (volA's flood is a different channel than volB), so the only remaining need is to BOUND volA's in-flight memory under a burst — a counter does that with zero added lanes/threads. Simplest thing that isolates + bounds.
- NOT CHOSEN: multiple-lanes-per-vid (shard each Volume's commands across K lanes for intra-Volume parallelism). Heavier (K× the subscriptions, a lane-assignment scheme) and solves a problem eg.4 doesn't have — intra-Volume command parallelism is not a stated need, and the engine commits are serialized per-Volume anyway (OCC snapshot-level). Layerable later if a single Volume's throughput ever needs it; deferred.
Both satisfy S-7's "a full {vol} lane never blocks a different {vol}"; the counter is the reductive choice.

## {eg-4-learnings} Learnings

### L-1 — bilrost wire format pinned by hand (the FeedBlob client-side peek)

The client needs the LSN from the OPAQUE eg.3 FeedEvent blob for its replay cursor, but adding a full bilrost decoder to echo_store would be heavyweight. Decoded the eg.3 fixture by hand to write a minimal field-reader (EchoStore.GraftBackend.FeedBlob): bilrost keys are DELTA-encoded — key byte = (field_delta << 2) | wiretype, where each key ADVANCES the running field number by delta (NOT absolute like protobuf). wiretype 0 = varint (LEB128, 7 bits/byte, high-bit continuation); wiretype 1 = length-delimited (len varint then bytes). FeedEvent emits fields 1..4 ascending so deltas are all 1. The reader walks fields accumulating field_num=>value, returns field 1 (branded string) + field 3 (lsn varint), stays blind to log_id/ts. Pinned against the SAME 51-byte eg.3 fixture (feed_blob_test.exs). KEY PROPERTY: the blob stays opaque end-to-end (forwarded whole as {:graft_feed,branded,lsn,blob}); the peek only reads 2 fixed fields, so a future additive proto field doesn't break the forward — only the cursor-peek would need updating. This is the "opaque blob composes" contract realized concretely.

L-2 — clippy --all-targets vs the workspace lints catch test-code warnings too. The workspace sets redundant_clone/uninlined_format_args/doc_markdown = "warn". My test files tripped redundant_clone (vid.clone() at last-use) and the autofix then tripped redundant_field_names (vid: vid). Plain `cargo clippy -p X` (lib only) MISSES these; must run --all-targets to see test-code warnings. The eg.3 "new code adds 0 warnings" posture covers test code. Cleared all → plain clippy --workspace clean.

### L-2 — a prebuilt-binary determinism loop and a concurrent `cargo test` on the same target are mutually exclusive. My first determinism loop (running the prebuilt round_trip binary in a serial loop) FAILED at iter 29 — but only because I was simultaneously running `cargo test --test round_trip` for a mutation, which RELINKS the binary under target/debug/deps/ mid-loop, so the loop invoked a transiently-replaced/locked executable. The clean re-run in ISOLATION (no other cargo touching the workspace) was 50/50 + 50/50 = 100/100. PRECEPT for the determinism gate on a Rust rung: complete all rebuild-requiring work (mutations, clippy --fix, edits) FIRST, then run the determinism loop alone against a freshly-built pristine binary with no concurrent cargo process in the workspace. A loop "failure" under concurrent builds is a harness artifact, not a determinism defect — re-run isolated before believing it.

### L-3 — "tested in isolation" is not "wired in": the backpressure cap (Backpressure::admit) is fully unit-tested AND ships as dead code on eg.4's path (grep: admit() called nowhere in session.rs/dispatch.rs). The S-7 test constructs the limiter directly, so a green S-7 suite does NOT prove the dispatch enforces the cap — only that the cap, IF called, behaves. The criterion is still satisfied (structural per-Volume-lane isolation carries "without blocking other Volumes"), but a green "mechanism" test can mask an unwired mechanism. PRECEPT for the evaluator: for any safety/limit mechanism, grep that the PRODUCTION call site consults it, don't trust a unit test that instantiates it locally. Here it's a deferred-by-design wiring (the live binding in eg.5/eg.6 is the consumer), made explicit in the verdict + a recommended eg.5 deliverable — but the gap was invisible to a tests-pass reading. (Mutation M4 surfaced a sibling: the err_kind_of not_found arm passes only because nothing asserts it.)
