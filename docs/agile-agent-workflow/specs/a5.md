## Goal

After A5, the reader can **write an agent brief (`.llms.md`) and run a Claude agent to build an increment**: a
references-first brief (the sources to read, numbered testable requirements, the execution topology, the agent stories,
and a comprehensive implementation prompt), and the practice of briefing, supervising, and reviewing the agent. The
reader leaves able to brief the engine chapter and make a first full pass from spec to running code (the A5.8 workshop).

## Rationale (5W)

- **Why** — a spec (A4) defines *what and why and done*, but a Claude agent needs a brief that tells it *how to build*:
  the exact sources to read, the runtime shape, the build order, and the proof gates. Writing for an agent differs from
  writing for a person (links first, prose second, every reference exact); and running an agent well — knowing when to
  let it run and when to intervene — is a learned practice. This chapter is where the course's thesis becomes concrete.
- **What** — a teaching chapter on the agent brief and the implementation workflow: the `llms.txt` convention (A5.1),
  references and requirements (A5.2), execution topology (A5.3), agent stories (A5.4), the comprehensive implementation
  prompt (A5.5), running Claude agents well (A5.6), the thesis — Pragmatic Programming with Claude Agents (A5.7), and a
  workshop briefing the agent for Portal (A5.8).
- **Who** — the **learner** (writes a brief, runs an agent), the **developer** (briefs a rung of their own project),
  and the **Claude Author** (both: reads briefs to build, and is taught here how a brief must read to be runnable).
- **When** — Part V, after the spec layer (A4) and before reliability (A6) and the production exemplar (A7); the brief
  wraps the spec A4 defines and produces the increment A6 hardens.
- **Where** — served at `/course/agile-agent-workflow/brief`; specs under `docs/agile-agent-workflow/specs/`; the worked
  examples are the Portal's real agent briefs (`docs/elixir/specs/phoenix/f6.N.llms.md`) and the prompts
  (`f6.N.prompt.md`), plus Anthropic's engineering guidance on building effective agents.

## Scope

- **In:** the agent brief's anatomy (references, requirements, topology, agent stories, the implementation prompt) and
  the practice of running an agent (brief, supervise, review, intervene); landed on the Portal's real briefs.
- **Out:** the spec's content (A4), reliability and correctness techniques (A6), and the full zero-to-production
  exemplar (A7). A5 briefs and builds one increment; it does not define it, harden it, or run the whole program.

## Invariants

- Every page lands a **principle → Portal practice**, reuses the design system, and carries the segmented route-tag,
  the canonical footer, and real Sources links.
- Every brief/prompt example is **verbatim** from a real Portal artifact (`docs/elixir/specs/phoenix/f6.N.llms.md` /
  `f6.N.prompt.md`); no requirement, topology, or Portal API is invented (only `Portal.ID` and the real F6 surfaces).
- The agent is an **implementer of well-specified work** — never anthropomorphised; the brief fixes every decision the
  spec has already fixed and leaves none to the agent that the Operator must own.
