## What A3 teaches

Planning delivery as **thin, robust increments** in a `roadmap.md` — the layer that plans *how we deliver* a backlog,
over the spec layer that defines *what we build*. It is where the Agile Manifesto, Extreme Programming, and Continuous
Delivery meet the agent workflow. A3 enters with the A2 backlog (a value ladder of stories) and leaves the reader able
to write a chapter roadmap: milestones, the iteration loop, thin-but-robust slices, and the program-level roadmap of
roadmaps.

## Grounding (no-invent — the real exemplar)

The worked example is the Portal's real web chapter, **F6 (Phoenix)**, and its real `phoenix.roadmap.md`
(`docs/elixir/specs/phoenix/phoenix.roadmap.md`): nine rungs `f6.1…f6.9`, three milestones (ship the catalog · make it
live · ship to users), each rung a line that points at a spec and defines no behaviour, under the master invariant
"the web layer calls only the `Portal` facade and renders only `%Portal.Error{}`." Quote it verbatim; invent no rung.
Cite `/elixir/phoenix` (the built chapter) and its sub-chapters (`/elixir/phoenix/{lifecycle,routing,ecto,contexts,
heex,liveview,pubsub,deployment,live-dashboard}` — all resolve 200) where a rung is implemented. The only Portal API
to name freely is `Portal.ID.generate/1`/`Portal.ID.decode/1`; the real F6 surfaces (`Portal` facade, `%Portal.Error{}`,
`CatalogLive`, `Portal.search_courses/1`) may be cited **as they appear in `docs/elixir/specs/`**.

## Conventions every A3 page holds

- **Copy the design system verbatim** from a built model: hub ← `html/agile-agent-workflow/why/two-layers/index.html`;
  lesson ← `…/why/two-layers/spec.html`. Reuse the stamp `TSK0Ng9hnHJgW0`. Accent: elixir-purple (`.ex`).
- **Segmented clickable route-tag** (`.rsep`/`.rcur`; `/course/agile-agent-workflow` one segment; `roadmap` links to
  the chapter). **Canonical 3-column `.foot-cols` footer**.
- **References → Sources = ≥3 real vetted links** from the course-home registry — for A3 the apt set is Continuous
  Delivery (`https://continuousdelivery.com/`), Extreme Programming Explained
  (`https://www.oreilly.com/library/view/extreme-programming-explained/0201616416/`), The Pragmatic Programmer
  (`https://pragprog.com/titles/tpp20/the-pragmatic-programmer-20th-anniversary-edition/`). Never fabricate a URL.
- **Two interactives per lesson** (hero + main, distinct moves; a hub carries ≥1 framing interactive) — real
  computation over a fixed dataset, live `.geo-readout` (aria-live), pure functions, degrades without JS, honours
  `prefers-reduced-motion`, no browser storage. Close concepts with a `.bridge` (principle → Portal practice) + `.take`.
- **Voice** — no first person, no exclamation/emoji, none of {just, simply, obviously, effortless, magical,
  revolutionary, blazing}. "how we deliver" / "what we build" / "who does the work" are the established house-style
  layer/question names — use them freely.
- **Pager (model convention):** a module hub's pager is `prev` = the chapter landing `/roadmap`, `next` = the module's
  own first dive; dive pagers chain hub → dive1 → dive2 → dive3 → back to hub. Crumbs/`Related` must resolve.
- **md-first:** write each page's md source under `docs/agile-agent-workflow/content/roadmap/<module>/`.
