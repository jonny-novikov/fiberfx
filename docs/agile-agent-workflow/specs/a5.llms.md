## What A5 teaches

How a Claude Author turns a spec into working code via a machine-readable brief, and the practice of running an agent
well — the heart of *Pragmatic Programming with Claude Agents*. A5 enters with the A4 spec and leaves the reader able
to write an `.llms.md` brief (references, requirements, topology, agent stories, the implementation prompt) and brief,
supervise, and review a Claude agent through a first full pass from spec to running code.

## Grounding (no-invent — the real exemplar)

The worked examples are the Portal's real agent briefs and prompts under `docs/elixir/specs/`: a brief
`phoenix/f6.N.llms.md` (references-first, requirements, execution topology, agent stories) and a prompt
`phoenix/f6.N.prompt.md` (the comprehensive implementation prompt an agent runs). Quote them **verbatim**; invent no
requirement, topology, or surface. The only Portal API to name freely is `Portal.ID.generate/1`/`decode/1`; the real F6
surfaces may be cited as they appear in source. A5.6/A5.7 draw on Anthropic's engineering guidance (building effective
agents; Claude Code best practices). This course's own briefs (`a3.llms.md`, `a4.llms.md`, this file) are a
self-referential exemplar — A5 can show its own brief as a worked `.llms.md`.
