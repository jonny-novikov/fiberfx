### T-4 — The smallest-change build order (the topology Mars assembles, not a pile of snippets)

The cutover makes "acting as another player" unrepresentable by replacing the trusted input at TWO ingresses (the HTTP plug + the socket connect) with a verified one, both reading the SAME pure verifier. Build order, smallest-change-first, each step compiles green before the next:
1. `Codemojex.InitData` (PURE, new `lib/codemojex/init_data.ex`): `verify(init_data_query_string, token, opts) :: {:ok, %{tg_user_id, user, auth_date, raw}} | {:error, reason}`. Parses the query string, strips `hash`+`signature`, builds the data-check-string, derives the WebApp secret, `:crypto.hash_equals` the lower-hex HMAC, checks `auth_date` freshness (opts max-age). reason ∈ {:missing, :malformed, :bad_hash, :stale, :no_token}. NO HTTP, NO Repo. Independently fixture-testable.
2. resolve-or-create [VenusPG-specified contract]: `Codemojex.resolve_player_by_tg/2` (game.ex facade) → `Wallet.resolve_or_create(tg_user_id, attrs)` — INSERT … ON CONFLICT (tg_user_id) DO NOTHING then read, OR get-then-insert-rescue-unique; the UNIQUE index is the concurrency backstop. Returns `{:ok, plr}`. [precise SQL/migration = cm.4.postgres.design.md].
3. `CodemojexWeb.Auth` plug (new `lib/codemojex_web/auth.ex`): pull initData from a header (`x-telegram-init-data`) or `params["initData"]`; the dev/test posture (F5) — when `trust_supplied_player` is on AND a `player`/`tg_user_id` is supplied, assign it WITHOUT a signature; else `InitData.verify` → resolve-or-create → `assign(conn,:player,plr)`; on any error `send_resp(conn,401,…) |> halt()`. The token = `Codemojex.Bot.token/0`.
4. `router.ex`: add `pipeline :auth`; pipe the 5 player-acting routes (join/guess/history/buy_keys/convert) through `:api`+`:auth`. create_player → F3 (keep behind the posture / route under :api only, prod creates via resolve-or-create).

6. `user_socket.ex`: `connect(params,socket,_)` → verify initData from `params["initData"]` (same posture) → `assign(socket,:player,plr)` + `id(socket)→ "player_socket:"<>plr`; reject (`:error`) on bad/missing.

GROUNDED (the surfaces the revision cites):
- EchoStore.Table THREE coherence modes (table.ex:22-24): :none · :broadcast (versioned app-level lane — "a message about a name", the BCS law literal) · :tracking (RESP3 CLIENT TRACKING). put/3 MINTS the version (table.ex:90); put/4 takes a caller version (table.ex:97); fetch/3 (table.ex:63) = L1 ets lookup → coalesced L2 fill → loader; invalidate/3 (table.ex:174) = DEL L2 + :ets.delete (the logout); the KIND GATE in-table (gate/2 ~table.ex:495 per the redis-patterns dive) refuses a wrong-namespace id with ZERO keys on the wire. Sliding TTL = a re-put with a fresh version + PX (the redis-patterns ttl-expiry dive); L1 jittered expiry + a :sweep.
- THE MUTABLE-SESSION COHERENCE FORK (NEW, load-bearing): the two existing caches are coherence: :none ONLY because their entities are IMMUTABLE for life (Codemojex.Tables moduledoc: a game's secret, an emoji set).
