## {codemojex-admin-learnings} Learnings

### L-1

L ‚Äî the SPA-vs-island precedent split (codemojex-node frontend craft). A standalone `@codemojex` console (economy, dashboard) is an SPA: `index.html` ‚Üí `src/main.tsx` `createRoot`, `@mercury/ui`+`@mercury/effector` source aliases (THREE `../` to `mercury/packages`; two resolve into `codemojex/packages`), default `dist/` build ‚Äî the `economy` app is the structural template. `@codemojex/game` is NOT that template: it is a library-mount island (`src/index.tsx` exports `mount(el,props,bridge)`, React bundled, content-hashed `game-[hash].js`, dynamic-imported by a Phoenix hook). Mirror `economy` for a console's skeleton; mirror `game` ONLY for its `channel/model.ts` (`createChannel` + `sample`) pubsub pattern. Applied on admin.5 (dashboard Shell): the Director brief named `game` as the template; the reconcile corrected it to economy-for-skeleton / game-for-channel.
