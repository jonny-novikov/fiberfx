# msh — specs

This directory holds the per-rung spec triads (`<rung>.md` + `.stories.md` + `.llms.md`) for the `msh`
toolchain, authored in REVERSE mode (the code is canonical; each triad is derived from it and grounds every
surface at its `file:line` — [the reverse playbook](../../../aaw/aaw.reverse.md)).

**The rung ladder is defined in [`../msh.roadmap.md`](../msh.roadmap.md)** — the as-built ladder msh.0…msh.6
(shipped) plus the deferred msh.P2 (Phase 2) and msh.FX (the frontmatter-`type` fix, pending the Operator
ruling on the fork in [`../msh.design.md`](../msh.design.md) §8).

**Per-rung triads are DEFERRED to follow-on rungs.** The design canon ([`../msh.design.md`](../msh.design.md)),
the roadmap ([`../msh.roadmap.md`](../msh.roadmap.md)), the dashboard ([`../msh.progress.md`](../msh.progress.md)),
the feature catalog ([`../msh.features.md`](../msh.features.md)), the testing view
([`../msh.testing.md`](../msh.testing.md)), and the references ([`../msh.references.md`](../msh.references.md))
specify the as-built surface today; the rung triads are authored when each subsystem is taken through the
reverse fan-out (survey → ladder → triad → adversarial grounding verification). This directory is empty until
then.
