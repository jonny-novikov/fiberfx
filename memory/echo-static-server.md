`go/echo-static` (module `github.com/fiberfx/echo-static`) = a standalone Echo v5 server that serves the repo's **`html/` directory AS-IS, byte-for-byte** — the **parity** counterpart to [[echo-courses-program]]. Where echo-courses *rebuilds* the courses index from a catalog (so byte-parity is a fragile property it must prove), echo-static *serves the actual files*, so parity is true by construction and verification collapses to `curl == file`. Built 2026-06-22 via a fanned-out Workflow (build → parallel content/links/url-convention/regression verify → remediate-until-clean; 10 agents, 153/154 checks green) to **urgently serve the new `html/echo-persistence/` course** (a complete 25-page course on EchoMQ/echo_store persistence: foundations champ-accept-tier/durability-spectrum/persistence-concepts · local-store cubdb/mvcc-time-travel · overview — content already authored on disk, it just needed serving).

**The one engineering problem = the URL convention** (parity is otherwise free): the published site advertises **clean, extensionless, directory-indexed** URLs (`/redis-patterns/queues/the-reliable-queue`) while the bytes live at nested `…/the-reliable-queue.html`. The resolver does **try-files**: for a request path P (cleaned absolute, leading slash trimmed, empty→"."), serve the first existing regular file under `HTML_DIR` among `[P, P+".html", P+"/index.html"]`; bare `/`→index.html; a miss serves `html/error/404.html` with status 404. Path cleaning forces absolute before trimming so `..` cannot escape root.

**Echo v5 grounding gotchas (NO-INVENT, found during the build — load-bearing):** (1) the catch-all MUST be `e.RouteNotFound("/*", h)`, **NOT** `e.GET("/*", h)` — only RouteNotFound fires for single-segment site paths. (2) `c.File`/`e.Static` serve through `e.Filesystem` = `os.DirFS(".")`, which **rejects absolute / `..` fs.FS paths** — so the handler uses `http.ServeFile` for hits and writes the 404-page bytes **manually** with an explicit 404 status (http.ServeFile forces 200). Same v5 idioms as echo-courses otherwise: `*echo.Context`, graceful shutdown via `signal.NotifyContext` + `echo.StartConfig{Address,GracefulTimeout}.Start(ctx,e)` (no `e.Shutdown`), `replace github.com/labstack/echo/v5 => ../echo`, build `GOWORK=off`, not a go.work member.

**Run it:** `cd go/echo-static && make start` (background, nohup + `bin/server.pid`, idempotent; the start/stop/restart/status targets mirror echo-courses' Makefile). Env: `ADDR` default `:1330` (echo-courses uses `:1323`), `HTML_DIR` default `../../html` (resolves to repo `html/` from the module dir; override to absolute or `html` from repo root). `make stop`/`restart`/`status` as expected.

**Parity status:** the whole `html/` tree serves (`/`, `/redis-patterns`, `/bcs`, `/echomq`, `/fsharp`, `/art`, `/mesh`, `/agile-agent-workflow` + `/course/agile-agent-workflow` alias, `/sitemap.xml` `/robots.txt` `/llms.txt`) + the new `/echo-persistence` (25/25 byte-identical, 106/106 internal links resolve). **Known divergence:** `/elixir` 404s because the Elixir course is NOT under `html/` — it lives at the **repo-root `elixir/` tree** (the jonnify Dockerfile COPYs `elixir/` separately). Full published-site parity would require serving `elixir/` too (a second root / fallback) — NOT done (the brief was "serve html/ as it is"). Also: `html/echo-persistence/` now has a **home-page course tile** in `html/index.html` (added 2026-06-22 — a 6th `.series-card`, `data-tags="persistence"`, accent `#d68f2f`, a bespoke storage-tier SVG icon; every course counter bumped 5→6 + a new "Persistence" filter track, hero/section/meta/footer prose woven; verified live on :1330, `/echo-persistence`→200). **Still not in `html/sitemap.xml`** (the remaining discoverability half) and the echo-courses catalog-rebuild server still doesn't list it — follow-ups if wanted (the index edit is uncommitted; coordinate with the index echo-courses rebuilds).
