`/Users/jonny/dev/jonnify/mercury` = a **React + Vite + Effector pnpm monorepo design system** (Node 22, pnpm 10.17). Distinct from the echo_mq/BCS Elixir stack that shares the `jonnify` git root — git root is `jonnify` (the PARENT), so scope commits with a `mercury/...` pathspec.

**Packages (resolve from SOURCE via vite alias + tsconfig `paths` — apps need NO prebuild):**
- `packages/mercury-ui` (`@mercury/ui`): token-driven, className-based (`.mx-*`) React 18 components, ~23 exports (Button, Input/Textarea/Search, Select, Checkbox/Radio/Switch/Segmented/Slider, Chip/Tag/Badge/Avatar, Card, Alert/Progress, Tabs, Modal/Tooltip, Table, AuthCode, Icon). Presentational, light theme default (`.dark-theme` class on root for dark), fonts DM Sans/Mono/Serif via `--font-primary/secondary/display`. Build: `vite build && tsc` → `dist/mercury-ui.{js,css}` + `index.d.ts`. Tokens in `src/styles/tokens.css` (`rgb(var(--token))` triplets).
- `packages/mercury-effector` (`@mercury/effector`): the state plug — `useTheme/setTheme/toggleTheme/initTheme`, `toast`/`<Toaster>` (renders Mercury `Alert`s), `createForm` (Effector-backed `useField`/`useForm`). Components stay presentational; effector lives outside React.

**THE RULE (Operator-stated):** ready-to-use components live ONLY in `packages/mercury-ui`; `apps/*` (showcase, catalogue, docs, echomq) just COMPOSE them + the effector plug — never reimplement, never house reusable components in an app.

**`static/`** = design-tool exports (custom `x-dc`/`DCLogic` runtime, raw `.mx-*` markup + `mercury.css`/`tokens.css`) = the design source of truth (`showcase.html`, `catalogue.html`, `react-showcase.html`, Svelte variants, screens).
