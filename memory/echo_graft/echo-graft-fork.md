**Ground truths that diverge from the spec/integration sketch (confirmed against the pinned source — do not re-derive):**
- Remote storage is Apache **opendal** (`RemoteConfig::{Memory,Fs,S3Compatible}` over `opendal::Operator`), NOT `object_store`. The commit fence already exists (`RemoteErr::precondition_failed` ⇒ `opendal::ErrorKind::ConditionNotMatch`); `S3Compatible` already reads `AWS_ENDPOINT_URL` → **eg.2 Tigris is mostly config**, not new structure.
- Local commit OCC is **snapshot-version level, NOT page-level** (`fjall_storage::commit` → `is_latest_snapshot`): a 2nd concurrent commit from the same base aborts with `LogicalErr::VolumeConcurrentWrite` **even on disjoint pages**. So spec acceptance #4 ("disjoint concurrent → both succeed") was reframed to the engine's real behavior; disjoint-merge is a *remote-sync* property (eg.2/eg.3).
- macOS build patch: `reqwest::tcp_user_timeout` is Linux/Android-only → cfg-gated in `remote.rs` (the one FORK.md build patch).
- **No `echo_graft_remote` crate** — remote is a module, not an extractable trait (would force a core/remote/runtime split the rung never required).
