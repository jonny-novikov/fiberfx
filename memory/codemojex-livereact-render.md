**Dev gotchas (load-bearing, all bit this rung):**
- **`mix` does NOT load `.env`** (no Dotenvy in the repo). Boot dev with the env: `cd echo/apps/codemojex && set -a && source ../../.env && set +a && TMPDIR=/tmp MIX_ENV=dev mix phx.server`. `echo/.env` carries `CODEMOJI_BOT_TOKEN` (+ DATABASE_*/AWS/Tigris); it's gitignored.
- **Bot token dev wiring**: `Codemojex.Bot.token/0` reads `config :codemojex, Codemojex.Telegram, token:` first — that was `:prod`-only in `runtime.exs`, so dev fell back to the demo hello-bot YAML and resolved nil. Added a `:dev` block arming it from `CODEMOJI_BOT_TOKEN` (OUTBOUND only; inbound polling stays off in dev — prod owns the `@codemoji_bot` webhook, a dev getUpdates poller would conflict). The `ECHO_BOT_HELLO_TOKEN` boot warning is a *separate* benign echo_bot demo bot.
- **`check_origin`**: dev `url` host is `localhost`, so a `127.0.0.1` WS Origin fails the check → the LiveView/channel socket never connects (page renders but is dead/non-interactive). Fixed with `check_origin: false` in `dev.exs` (prod keeps its `runtime.exs` allowlist). Use `http://localhost:4000`.

**E2E harness `node/codemojex-e2e/`** (Playwright, standalone — `codemoji-*` workspace glob doesn't capture `codemojex-e2e`). `lib/initData.ts` forges a valid WebApp `initData` (HMAC mirrors `Codemojex.InitData.verify/3`: `secret=HMAC("WebAppData",token)`, `hash=HMAC(secret,data_check_string)` excluding hash+signature, sorted, `\n`-joined) → sets the `tg_init` cookie → real handshake, no bypass. 5 stories (welcome, auth-gate redirect, authenticated lobby, enter-room→board shell, axe a11y), screenshots+trace+video in the HTML report (`npm run report`). Run needs the dev server up + the bot token (env or `CODEMOJEX_ENV_FILE`). Setup runbook = `echo/docs/codemojex/edge-bucket-setup.md`. **Board vite build FIXED 2026-06-28 (commit `e0fd3305`):** `vite.config.ts` had `input: "react/index.tsx"` — a BARE specifier that Rollup resolved as the npm `react` package's `./index.tsx` export (the entry dir name collides with the pkg) → `Missing "./index.tsx" specifier in "react"`.
