**The codemojex admin control plane** = `@codemojex/admin` at `mercury/codemojex/apps/admin/` ‚Äî a **Fastify + Drizzle + Valkey backend API** (NOT a Mercury/React frontend), one of four apps in the `mercury/codemojex/` Node/TS monorepo (siblings: `economy`, `game`, `game-tauri`; packages `db`/`domain`/`types`). Charter (its own `server.ts`): "an operator surface for browsing rooms, active games, and players, plus light management. Postgres (via `@codemojex/db`) is the system of record; ValKey (:6390) is read for the live board; the server never scores." Schema-driven via `@fastify/type-provider-typebox` (one TypeBox schema ‚Üí static type + validator + serializer); the game **secret is stripped at the wire** (`gameCols`/`GameSummary` list only public columns). Boots solo (`main.ts` ‚Üí `start`) and clustered (`cluster.ts` ‚Üí `@echo/cluster` `runCluster`). As-built read plane: `/health`, `/rooms`(+`:id`,`PATCH :id/status`), `/games`(+`:id`), `/players`(+`:id`). **`mercury/codemojex/{packages,apps}` (except `apps/game`, its own nested workspace) are MEMBERS of the `mercury/` pnpm workspace** ‚Äî D-3 repointed the glob to `codemojex/{packages,apps}/*`, so `pnpm --filter @codemojex/<app>` resolves from `mercury/` and deps are installed (verified on admin.5; the earlier "own workspace / matches nothing" note was pre-D-3 and is corrected).
