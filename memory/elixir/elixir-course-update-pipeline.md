The user wants `docs/elixir/elixir-progress.md` to be the trigger that updates the `/elixir` course. The update is split into two tiers by cost/determinism (see [[jonnify-cms-toolchain]]):

- **Monitor (always-on, cheap):** the bash watcher `docs/elixir/references/watch_refs.sh` polls `elixir-progress.md` mtime + new `elixir/**/*.html` and runs the deterministic `promote.py` (declare built-but-planned modules/subpages → sync contents cards → rebuild cms). It CANNOT call the Workflow tool.
- **Update (on signal, multi-agent):** the Workflow `elixir-refs-rollout` (`.claude/workflows/elixir-refs-rollout.js`) — bibliography-driven References rollout (sources verbatim from `docs/elixir/kb/elixir-references.md`, keyed by module) + independent adversarial verify. **Invoke by baking the page list into the script's `PAGES` and running via `scriptPath` (or an inline script).** `name`+`args` does NOT reliably inject `args` in this harness — it silently falls back to `DEFAULT_PAGES` (a 4-page arg list once ran the 22 defaults, wasting ~2.3M tokens). The insert step is re-entrant (single-block enforced), so a double-run is wasteful but harmless.

**Run sequence (hybrid — scout inline, then pipeline):** pause watcher → `promote.py --rebuild` → surgically sync any stale contents cards to the manifest (e.g. flip `<a class="mod">`/`pill built` ↔ `<div class="mod is-quiet">`/`pill planned`) → scout pages lacking `id="refsTitle"` → `Workflow({name:'elixir-refs-rollout', args:[…]})` → `cms audit` + batch `cms check` → re-baseline + restart watcher.

References are NOT one of the 9 Apollo gates, so a page is A+ without one; the rollout is an enhancement, not a defect fix. **F3 completed 2026-05-31 (22/22 A+); F4.01 (Lists) integrated same day.** F3.03 `/elixir/language/modules` has no page on disk — kept as a non-linking planned card; author it before linking. **Chapter promotion (learned via F4.01).** When a chapter's FIRST module arrives, the chapter route (`/elixir/<chapter>`) stays forbidden until the CHAPTER status is flipped `planned → live` (AllowedRoutes admits a chapter route only when `Linkable(chapter.Status)`), so every subpage that links up to its chapter shows `REGRESSION / FAIL:links`. `promote.py` now auto-detects this (landing resolves + ≥1 built module → flip chapter) AND always rebuilds `bin/cms` on `--rebuild` (previously it returned early on "no drift", leaving a hand-edited manifest uncompiled). The nav cards in the hand-edited `elixir/course/index.html` are synced surgically (flip `<div class="mod is-quiet">`/`pill planned` ↔ `<a class="mod">`/`pill built`, and the chapter head's planned-pill ↔ `<a class="chap-link">Open chapter →</a>`); `sync_contents.py` regenerates `elixir/index.html` only.
