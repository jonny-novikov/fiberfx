The **F10 "bot" chapter** (`docs/elixir/specs/bot/`) was reframed from "a Portal Telegram bot" to **`echo_bot` — a standalone, first-party multibot ENGINE** in the echo umbrella. Direction (Operator):
- **Wrap-then-own a VENDORED FORK of ex_gram** (NOT a hex dep, NOT hand-rolled): F10.1 copies ex_gram's Telegram-API code into `apps/echo_bot/vendor/ex_gram/` with a `README.md` (provenance/license) + `CLAUDE.md` (owned-fork directive: agents Read/Write/modify directly, NO upstream PR to ex_gram main — the Operator IS ex_gram team, maintainer-tasked for next-gen modules). `EchoBot.Platform.Telegram` WRAPS the vendored copy; later rungs replace internals behind the unchanged port (F10.6). Core reaches vendored code ONLY via the adapter (an invariant).
- **Platform-abstracted:** an `EchoBot.Platform` behaviour/port; `EchoBot.Platform.Telegram` is adapter #1; more platforms slot in without editing the core.
- **YAML v1.0 config (approved, versioned, plug-and-play):** 5 keys — `version` ("1.0", read first), `name`, `platform`, `token_env` (env-var NAME not the token), `handler`. Drop a YAML + handler → a bot.
- Lives as **`apps/echo_bot`** — its own `EchoBot.Application`/`EchoBot.Supervisor`. `echo/mix.exs` is `apps_path: "apps"` so apps auto-discover (no root mix.exs edit; umbrella `config/config.exs` may gain an `echo_bot` block — umbrella-level, never portal code). The umbrella already has `bandit`/`plug`/`websock`/`websock_adapter`/`phoenix`/`phoenix_pubsub` in `mix.lock`.
- **Games slice (F10.3+):** a NEW app **`apps/echo_games`** = server-authoritative backend for **"Эмоджи: Найди пару"** (Russian-targeted; rewrite of `html/game.html` — 3 levels, combo/penalty/time-bonus/stars), the frontend rewritten in **Svelte 5 (Runes) + Vite**, served by a **NEW** Elixir web server (decision matrix → recommend **B: a minimal `Plug.Router` on Bandit + WebSock**, NOT `portal_web`), with **WebSocket** live state and a **Telegram-payments** seam opened (monetized games; full flow F10.8).
