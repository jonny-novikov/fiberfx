`apps/jonnify-cms` is a Go **Cobra** content toolchain for the `/elixir` "Functional Programming in Elixir" course. It is **not** mentioned in `CLAUDE.md` (which scopes itself to the jonnify server and lists only the older out-of-scope modules).

**What it is:** a reconciliation engine over **three sources of truth** — (1) the **manifest** in `docs/elixir/kb/build_page.py` (declared status; ported verbatim into `internal/manifest`), (2) the **filesystem** (built pages under `/elixir`), (3) the **nine Apollo A+ gates** (ported into `internal/apollo` from `build_page.py`). `build_page.py` is the Python source-of-truth for the manifest, the gates, and the branded Snowflake stamp.

**Commands:** `cms readiness` (the headline: READY/DRIFT-PROMOTE/IN-PROGRESS/REGRESSION/BROKEN/PLANNED per module), `cms audit [--fix]` (broken links; `--fix` renames slug-mismatch orphans), `cms check FILES…` (the nine gates), `cms graph` (dot/mermaid/json), `cms manifest`/`routes`, `cms stamp mint|decode`, `cms build` (inert — the `content/` fragments `build_page.py` consumes are NOT in the repo). Build with **`GOWORK=off`** (repo convention); Cobra v1.10.2 is cached offline. The route→file resolver mirrors the server's `serveDirTree` exactly.

**Companion artifacts:** the writer agent `.claude/skills/elixir-technical-writer/` (SKILL.md + 7 references); the fan-out harness `.claude/workflows/elixir-author.js` (authors planned modules to `apps/jonnify-cms/drafts/`, gated by `cms check` — does NOT write the live tree).

**References pipeline (`docs/elixir/`):** `kb/elixir-references.md` is the curated per-module bibliography, generated by `kb/_gen_refs_md.py` (which does `import build_page`; writes in-repo now). A `deep-research` pass verified it and applied `http→https` + host/relabel fixes. `references/fetch_refs.py` mirrors every source **offline** into `references/files/` (gitignored, ~16 MB, 111/113; 2 paywalled book-catalog pages excluded) plus `manifest.json` + `INDEX.md`; it has a paren-aware parser and a Wayback fallback. `references/watch_refs.sh` is a polling daemon: on a **new** `elixir/*.html` page it syncs the bibliography+mirror and (with `REFS_AI=1`) spawns a **scoped `claude -p`** to deep-research + integrate a gated References block. The **user must start it themselves** (`REFS_AI=1 docs/elixir/references/watch_refs.sh start`) — Claude Code's safety classifier blocks the assistant from auto-launching an agent-spawning daemon.
