F3 · The Elixir Language

Values, types & IEx

An Elixir program is built from a small set of immutable value types, and every value belongs to exactly one of them. IEx is the shell where each value is entered, evaluated, and named by its type.

F1 and F2 taught the functional shape of the language — mappings, folds, composition. This chapter turns to the concrete data those functions move. Seven literal types cover most Elixir code, and one shell function, i/1, reports the type of any value. Pattern matching (F3.02), structs (F3.05), and protocols (F3.06) all rest on knowing what a value is.

The seven literal types

Elixir evaluates expressions to values. A value has a single type: a set of values that share a representation and a set of operations. Seven literal forms cover the data most code handles, and each has a literal syntax that produces a value of one type.

Two pairs are worth separating at the outset. An integer and a float are different types — 1 is an integer, 1.0 is a float — even when they compare equal. A boolean is not a separate type either: true and false are the atoms :true and :false, so is_atom(true) holds.

integer
an exact whole number of arbitrary size, e.g. 42.
float
a 64-bit IEEE 754 number, e.g. 3.14; distinct from integer.
atom
a constant whose name is its value, e.g. :ok; booleans and nil are atoms.
binary
a sequence of bytes; a double-quoted string is a UTF-8 binary, e.g. "hi".
list
an ordered, variable-length linked list, e.g. [1, 2, 3].
tuple
a fixed-size ordered group held contiguously, e.g. {:ok, 200}.
map
a set of key–value associations, e.g. %{id: 7}.

Inspect a value in IEx

IEx (Interactive Elixir) is the read–eval–print loop started with the iex command. A value typed at the prompt is evaluated and its result printed. The helper i/1 reports a value's type and details; the is_* guards answer a type membership question with true or false.

Choose a literal below. The inspector runs the real type rule and reports what i/1 and the matching guard return for that value.

Algebra

A type is a set of values; a value belongs to exactly one such set. Membership is a predicate that answers yes or no — the idea from F1.05, sets and their membership test.

→

Elixir

i/1 names the set a value sits in; each is_* guard is that set's membership predicate, returning true or false.

One value maps to one type, and IEx reports that mapping for any literal on the spot.

One value, one type

The type a value carries is fixed at the moment of evaluation and travels with the value, not with the name bound to it. Rebinding a name to a value of another type is allowed — the earlier value is unchanged, and the type test follows whatever value is held now.

The boundary cases earn their place. 1 == 1.0 is true because == compares numeric value, yet 1 === 1.0 is false because === also compares type. The strict operator is the honest read of "same value and same type".

iex> 1 == 1.0
# => true     (== compares numeric value)
iex> 1 === 1.0
# => false    (=== also compares type: integer vs float)
iex> is_integer(1.0)
# => false
iex> is_atom(true)
# => true     (booleans are the atoms :true and :false)

F2 · immutability

A value never changes; rebinding a name points it at a different value. The type is a property of the value, so it cannot drift underneath a name.

→

Elixir

x = 1 then x = "one" rebinds x; the integer 1 still exists and is still an integer wherever it is held.

Type belongs to the value; === is the test that respects both value and type.

What this lands

This module fixes the vocabulary the rest of F3 builds on: the literal value types, the shell that evaluates them, and the guards that test them.

  • Seven literal types — integer, float, atom, binary, list, tuple, map — cover most Elixir data.
  • IEx evaluates an expression and prints its value; i/1 reports the value's type.
  • Each is_* guard is a membership predicate for one type, returning a boolean.
  • An integer and a float are different types; === compares value and type, == compares value alone.
  • Booleans and nil are atoms, not a separate type.
